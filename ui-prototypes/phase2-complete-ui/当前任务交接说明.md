## 4. 当前用户最新要求

用户最新要求是：

```text
资源生成使用流程页面跳转的形式不要用现在这样的，重新设计个性化的学习路径
```

需要理解为两个任务：

### 4.1 资源生成流程要重做

当前 `资源工作台` 页面是单页内步骤条 + 过程区。

用户不满意，要求改为：

```text
流程页面跳转的形式
```

也就是说资源生成不应只是一个页面里的 stepper，而应该像多步骤流程：

```text
确认学习上下文
-> 选择资源类型
-> 配置生成要求
-> 生成过程页
-> 结果预览页
-> 保存/开始学习页
```

在 HTML 原型里可以用同一个 section 内的多个流程视图模拟页面跳转，也可以增加 `generate-context`、`generate-type`、`generate-config` 等内部视图，但一级导航仍只显示“资源工作台”。

关键要求：

1. 每一步都像独立流程页。
2. 有上一步、下一步。
3. 当前步骤主任务清晰。
4. 不要把所有内容堆在同一屏。
5. 生成结果要能保存到资源库、开始学习、进入练习。

### 4.2 个性化学习路径要重做

当前 `学习路径` 页面仍然偏知识图谱和节点详情。

用户要求：

```text
重新设计个性化的学习路径
```

下一版路径页要更像为学生定制的学习路线，而不是普通流程图。

建议方向：

1. 顶部展示学生当前目标和学习策略。
2. 路径不是普通节点图，而是“个性化路线图”。
3. 每个节点显示：
   - 当前状态
   - 推荐原因
   - 达标条件
   - 推荐资源
   - 练习入口
   - 是否被错题或画像影响
4. 要能看到路径为什么变化，例如：

```text
因为返回值掌握度下降，模块导入暂缓。
```

5. 路径页需要回答：

```text
我现在在哪？
为什么这样安排？
什么时候能进入下一步？
每个节点该怎么学？
```
