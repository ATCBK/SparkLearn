## 1. 文档目的

本文档是《学而思》个性化学习智能体系统的后端设计文档，基于已定稿的技术产品方案 V2、数据设计文档和动态数据设计文档，聚焦回答以下问题：

第一，后端项目结构如何组织，模块边界如何划分。

第二，每个 API 接口的输入输出、业务逻辑和错误处理如何定义。

第三，多智能体（CrewAI）如何与 FastAPI 集成，Agent 编排的具体代码结构是什么。

第四，知识库构建、RAG 检索、流式输出的具体实现方案。

第五，数据存储层（SQLite + JSON/JSONL + ChromaDB）的具体操作封装。

本文档作为后端开发的唯一依据。每一条设计都必须可直接对应到代码实现，不允许出现"待定""后续完善"等模糊表述。

### 2.2 层间调用规则

API 层只调用 Service 层，不直接调用 Agent 或 Storage 层。

Service 层编排业务流程：读数据 → 调用 Agent → 写数据 → 返回结果。

Agent 层只接收 AgentContext 和 LLM 调用能力，不直接读写数据库或文件。

Storage 层提供统一接口，上层不感知底层是 SQLite 还是 JSON 文件。

## 4. 数据存储层设计

### 4.1 SQLite 表设计（models.py）

```python
from sqlalchemy import Column, Integer, String, Float, Text, DateTime
from sqlalchemy.orm import DeclarativeBase
from datetime import datetime

class Base(DeclarativeBase):
    pass

class Student(Base):
    __tablename__ = "students"
    id = Column(Integer, primary_key=True, autoincrement=True)
    user_id = Column(String(64), unique=True, nullable=False, index=True)
    name = Column(String(64), default="")
    major = Column(String(128), default="")
    grade = Column(String(32), default="")
    course_id = Column(String(64), default="python_basic")
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

class Profile(Base):
    __tablename__ = "profiles"
    id = Column(Integer, primary_key=True, autoincrement=True)
    user_id = Column(String(64), nullable=False, index=True)
    goal = Column(Text, default="[]")                # JSON 数组
    knowledge_level = Column(String(64), default="")
    weak_points = Column(Text, default="[]")         # JSON 数组
    learning_preference = Column(Text, default="[]")  # JSON 数组
    cognitive_style = Column(String(64), default="")
    daily_time = Column(Integer, default=60)         # 分钟
    practical_ability = Column(String(128), default="")
    current_stage = Column(String(64), default="")
    version = Column(Integer, default=1)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

class MasteryRecord(Base):
    __tablename__ = "mastery_records"
    id = Column(Integer, primary_key=True, autoincrement=True)
    user_id = Column(String(64), nullable=False, index=True)
    knowledge_point_id = Column(String(64), nullable=False)
    knowledge_point_name = Column(String(128), default="")
    score = Column(Float, default=0.0)               # 0-1
    chapter = Column(String(64), default="")
    last_updated = Column(DateTime, default=datetime.utcnow)

class ContributionDay(Base):
    __tablename__ = "contribution_days"
    id = Column(Integer, primary_key=True, autoincrement=True)
    user_id = Column(String(64), nullable=False, index=True)
    date = Column(String(10), nullable=False)        # YYYY-MM-DD
    count = Column(Integer, default=0)               # 学习次数
```

### 4.2 SQLite 操作封装（sqlite_repo.py）

提供以下方法，所有方法均为 async：

| 方法 | 说明 |
|------|------|
| `get_student(user_id) -> Student \| None` | 查询学生 |
| `upsert_student(user_id, **kwargs)` | 插入或更新学生 |
| `get_profile(user_id) -> Profile \| None` | 查询画像 |
| `upsert_profile(user_id, **kwargs)` | 插入或更新画像，自增 version |
| `get_mastery_records(user_id) -> list[MasteryRecord]` | 查询所有掌握度 |
| `upsert_mastery_record(user_id, kp_id, score, chapter)` | 更新单个知识点掌握度 |
| `get_contribution_data(user_id) -> list[ContributionDay]` | 查询打卡数据 |
| `upsert_contribution_day(user_id, date, count)` | 更新单个打卡日 |
| `record_learning_event(user_id, event_type, payload)` | 记录学习事件到 JSONL |

### 4.3 JSONL 文件操作封装（file_repo.py）

| 方法 | 说明 |
|------|------|
| `append_learning_event(user_id, event: dict)` | 追加一条学习事件 |
| `read_learning_events(user_id, limit=100) -> list[dict]` | 读取最近 N 条事件 |
| `read_json(path) -> dict` | 读取 JSON 文件 |
| `write_json(path, data)` | 写入 JSON 文件（全量覆盖） |
| `read_quiz_records(user_id) -> list[dict]` | 读取练习记录 |
| `write_quiz_records(user_id, records)` | 写入练习记录 |
| `read_resource_usage(user_id) -> list[dict]` | 读取资源使用记录 |
| `append_resource_usage(user_id, record)` | 追加一条资源使用记录 |

### 4.4 数据库初始化

在 `main.py` 的 `startup` 事件中执行：

1. 调用 `Base.metadata.create_all(engine)` 创建所有表。
2. 检查 `students` 表中是否存在 `single_user`，不存在则插入默认学生记录。
3. 检查 `data/users/single_user/` 目录是否存在，不存在则创建。

## 5. API 接口设计

### 5.1 统一响应格式

```python
from pydantic import BaseModel
from typing import TypeVar, Generic, Optional

T = TypeVar("T")

class ApiResponse(BaseModel, Generic[T]):
    success: bool
    data: Optional[T] = None
    error: Optional[str] = None

def ok(data: T) -> ApiResponse[T]:
    return ApiResponse(success=True, data=data)

def fail(msg: str) -> ApiResponse:
    return ApiResponse(success=False, error=msg)
```

### 5.2 学生画像接口（profile.py）

#### POST /api/profile/initiate

启动对话式建档。

请求：
```json
{}
```

响应：
```json
{
  "success": true,
  "data": {
    "session_id": "sess_001",
    "first_question": "你好！我是学而思 AI 学习助手。请问你的专业和年级是什么？",
    "round": 1,
    "total_rounds": 3
  }
}
```

业务逻辑：
1. 生成 `session_id`，存入内存字典（不持久化，重启即丢失）。
2. 调用画像 Agent 生成第一轮对话问题。
3. 返回第一个问题和会话标识。

#### POST /api/profile/chat

建档对话交互（支持流式）。

请求：
```json
{
  "session_id": "sess_001",
  "message": "我是计算机科学专业大二学生"
}
```

响应（流式，SSE）：
```
data: {"type": "text", "content": "了解"}
data: {"type": "text", "content": "了，"}
data: {"type": "text", "content": "那你的学习目标是什么？"}
data: {"type": "done", "round": 2, "total_rounds": 3}
```

业务逻辑：
1. 从内存中取 `session_id` 对应的对话历史。
2. 追加用户消息到历史。
3. 调用画像 Agent 生成下一轮对话问题和画像特征抽取。
4. 第三轮结束后，画像 Agent 输出结构化画像 JSON，写入 SQLite `profiles` 表。
5. 返回流式响应。

错误处理：
- `session_id` 不存在 → 400 + "会话已过期"
- Agent 调用超时（30s）→ 504 + "AI 服务暂时不可用"
- 画像解析失败 → 500 + "画像生成失败，请重试"

#### GET /api/profile

获取当前学生画像。

响应：
```json
{
  "success": true,
  "data": {
    "user_id": "single_user",
    "goal": ["期末提分", "竞赛准备"],
    "knowledge_level": "有一定基础",
    "weak_points": ["函数", "面向对象"],
    "learning_preference": ["视觉型", "实践型"],
    "cognitive_style": "归纳型",
    "daily_time": 60,
    "practical_ability": "能独立完成小项目",
    "current_stage": "函数与模块",
    "version": 5
  }
}
```

业务逻辑：
1. 从 SQLite `profiles` 表读取 `single_user` 的画像。
2. 不存在则返回 404。

#### PUT /api/profile

手动更新画像字段（管理端或画像修正）。

请求：
```json
{
  "weak_points": ["函数", "面向对象", "装饰器"],
  "current_stage": "面向对象"

```

响应：
```json
{
```

1. 合并字段到 SQLite `profiles` 表，自增 version。
2. 追加一条 `profile_updated` 事件到 `learning_events.jsonl`。

### 5.4 资源接口（resource.py）

#### GET /api/resources

获取资源列表。

查询参数：
- `type`（可选）：资源类型过滤
- `limit`（可选，默认20）
- `offset`（可选，默认0）

响应：
```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "res_001",
        "title": "变量与数据类型详解",
        "type": "document",
        "status": "completed",
        "created_at": "2026-04-15",
        "preview_path": "/resources/res_001/preview.html"
      }
    ],
    "total": 6
  }
}
```

业务逻辑：
1. 读取 `resources_index.json`。
2. 按 `type` 过滤，按 `created_at` 倒序排列。
3. 返回分页结果。

#### POST /api/resources/generate

请求生成资源（触发多智能体协同）。

请求：
```json
{
  "type": "document",
  "knowledge_points": ["函数"],
  "prompt": "生成一份关于 Python 函数的讲解文档"
}
```

响应（流式 SSE）：
```
data: {"type": "progress", "stage": "画像分析", "percent": 20}
data: {"type": "progress", "stage": "路径匹配", "percent": 40}
data: {"type": "progress", "stage": "内容生成", "percent": 70}
data: {"type": "progress", "stage": "内容审核", "percent": 90}
data: {"type": "done", "resource_id": "res_20260418_001"}
```

业务逻辑（串行 + 并行混合编排）：
1. 加载 AgentContext（画像 + 掌握度 + 知识库上下文）。
2. 画像 Agent 串行执行，确认学生状态（复用缓存画像，不重新对话）。
3. 路径规划 Agent 串行执行，确定资源范围和难度。
4. 资源生成 Agent 并行执行，生成内容。
5. 审核 Agent 串行执行，校验内容。
6. 写入资源目录 + `meta.json` + 更新 `resources_index.json`。
7. 全程推送进度事件。

错误处理：
- 审核 Agent 判定不合规 → 标记 `status: "review_failed"`，记录审核意见，返回给前端。
- 生成超时（120s）→ 标记 `status: "failed"`，返回错误提示。

#### GET /api/resources/{resource_id}

获取资源详情。

响应：

业务逻辑：

## 6. 多智能体架构实现

### 6.1 AgentContext 统一上下文（context.py）

所有 Agent 执行前，由编排器构建统一的 AgentContext：

```python
@dataclass
class AgentContext:
    user_id: str
    profile: dict                    # 从 SQLite 读取
    mastery: list[dict]              # 从 SQLite 读取
    current_stage: str               # 从 profile 读取
    knowledge_context: list[dict]    # 从 RAG 检索获取
    chat_history: list[dict]         # 对话历史（答疑场景）
    task_hint: str                   # 编排器的任务提示
```

加载逻辑：
1. `profile` 和 `mastery` 从 SQLite `profiles` 和 `mastery_records` 表读取。
2. `knowledge_context` 由调用方根据具体场景指定关键词后从 RAG 检索获取。
3. `chat_history` 仅答疑场景需要。

### 6.2 编排器（orchestrator.py）

编排器负责调度 Agent 链路，提供三种预设编排模式：

**资源生成模式**：
```
画像Agent(串行) → 路径规划Agent(串行) → [资源生成Agent + 练习Agent](并行) → 审核Agent(串行)
```

**辅导答疑模式**：
```
RAG检索(先行) → 答疑Agent(串行) → 审核Agent(串行)
```

**评估模式**：
```
评估Agent(串行) → 审核Agent(串行)
```

实现方式：
```python
class Orchestrator:
    def __init__(self):
        self.profile_agent = ProfileAgent()
        self.path_agent = PathAgent()
        self.resource_agent = ResourceAgent()
        self.quiz_agent = QuizAgent()
        self.evaluation_agent = EvaluationAgent()
        self.review_agent = ReviewAgent()
        self.tutor_agent = TutorAgent()

    async def generate_resource(self, ctx: AgentContext) -> dict:
        # 步骤1: 画像确认
        profile_result = await self.profile_agent.run(ctx)

        # 步骤2: 路径匹配
        path_result = await self.path_agent.run(ctx)

        # 步骤3: 并行生成
        import asyncio
        resource_coro = self.resource_agent.run(ctx)
        quiz_coro = self.quiz_agent.run(ctx)
        resource_result, quiz_result = await asyncio.gather(resource_coro, quiz_coro)

        # 步骤4: 审核
        review_result = await self.review_agent.run({
            "resource": resource_result,
            "quiz": quiz_result
        })

        return review_result

    async def tutor_chat(self, ctx: AgentContext) -> AsyncGenerator[str, None]:
        answer_stream = self.tutor_agent.run_stream(ctx)
        async for chunk in answer_stream:
            reviewed = await self.review_agent.run_quick(chunk)
            if reviewed.approved:
                yield reviewed.content
            else:
                yield "我暂时无法确认这个回答的准确性。"

    async def evaluate(self, ctx: AgentContext) -> dict:
        eval_result = await self.evaluation_agent.run(ctx)
        review_result = await self.review_agent.run({"evaluation": eval_result})
        return review_result
```

### 6.3 各 Agent 实现规范

每个 Agent 文件遵循统一结构：

1. 定义 Agent 角色描述（CrewAI Agent 构造函数参数）。
2. 定义 Agent 任务 Prompt 模板。
3. 实现 `run(ctx)` 方法：构建 Prompt → 调用星火 API → 解析输出 → 返回结构化结果。
4. 实现 `run_stream(ctx)` 方法（仅答疑 Agent）：流式调用星火 API → yield 文本片段。

**画像 Agent**（profile_agent.py）：
- 角色：学习画像分析师
- 目标：从对话中抽取学生特征并更新画像
- Prompt 关键内容：提取专业、年级、目标、薄弱点、偏好、认知风格、每日时长、实操能力
- 输出格式：JSON 对象，严格匹配 Profile 表字段
- 容错：抽取失败时保留原文，标记为"待确认"

**路径规划 Agent**（path_agent.py）：
- 角色：学习路径规划师
- 目标：根据画像和掌握度计算阶段目标与章节顺序
- Prompt 关键内容：当前掌握度、薄弱点、学习阶段、课程知识树
- 输出格式：stages 列表 + knowledge_tree 节点状态
- 容错：输出非法格式时默认按知识库原始顺序排列

**资源生成 Agent**（resource_agent.py）：
- 角色：学习资源创作者
- 目标：根据画像参数和知识范围生成内容类资源
- Prompt 关键内容：知识点、学生水平、学习偏好、资源类型、难度等级
- 输出格式：Markdown 文本（document/reading/code 类型）
- 容错：生成失败时返回模板资源，标记低置信

**练习与案例 Agent**（quiz_agent.py）：
- 角色：练习命题专家
- 目标：生成分层练习题和错题变式题
- Prompt 关键内容：知识点、薄弱点、难度层次（基础/进阶/挑战）
- 输出格式：JSON 数组，每题包含 type/content/options/correctAnswer/explanation
- 容错：题目格式校验失败时丢弃该题

**评估 Agent**（evaluation_agent.py）：
- 角色：学习效果评估师
- 目标：分析学习行为数据，生成评估结论
- Prompt 关键内容：学习事件列表、练习记录、掌握度数据
- 输出格式：evaluation_snapshot 结构 + report_view_model 结构
- 容错：数据不足时仅做有限评估，标注"数据不足"

**审核 Agent**（review_agent.py）：
- 角色：内容审核员
- 目标：事实校验、安全过滤、格式标准化、低置信拦截
- Prompt 关键内容：待审核内容 + 知识库参考
- 输出格式：{ approved: bool, issues: [...], corrected_content: ... }
- 容错：审核超时时默认放行但标记"未经审核"

**答疑 Agent**（tutor_agent.py）：
- 角色：智能辅导老师
- 目标：基于知识库上下文回答学生问题
- Prompt 关键内容：学生问题、知识库上下文、对话历史、答疑模式
- 输出格式：Markdown 文本（流式输出）
- 容错：知识库上下文不足时提示学生补充信息

### 6.4 Agent 超时与降级策略

所有 Agent 调用设置 30 秒超时（`asyncio.wait_for`）。

超时策略：
- 画像 Agent 超时 → 使用当前缓存画像
- 路径规划 Agent 超时 → 使用默认知识库顺序
- 资源生成 Agent 超时 → 返回模板资源，标记 `status: "fallback"`
- 练习 Agent 超时 → 返回预设题库题目
- 评估 Agent 超时 → 返回上次评估快照
- 审核 Agent 超时 → 放行但标记 `review_status: "skipped"`
- 答疑 Agent 超时 → 返回"抱歉，请稍后重试"

重试策略：仅对网络类错误重试 1 次，间隔 2 秒。

### 7.2 混合检索（retriever.py）

```python
class HybridRetriever:
    def __init__(self, chroma_store, keyword_index):
        self.chroma = chroma_store
        self.keywords = keyword_index

    async def search(self, query: str, top_k: int = 5, student_profile: dict = None) -> list[dict]:
        # 第一层：关键词精确匹配
        keyword_hits = self._keyword_search(query)

        # 第二层：ChromaDB 语义检索
        semantic_hits = await self.chroma.query(query, top_k=top_k * 2)

        # 第三层：标签过滤（根据画像）
        if student_profile:
            semantic_hits = self._filter_by_profile(semantic_hits, student_profile)

        # 合并去重，关键词匹配结果优先
        merged = self._merge_results(keyword_hits, semantic_hits, top_k)

        return merged
```

### 7.3 Embedding 方案

使用讯飞星火 Text Embedding API（`wss://spark-api.xf-yun.com/v1/embedding`）。

批量 embedding：每批最多 16 条，每条不超过 512 token。

缓存：已 embedding 的文本计算 hash 值，重复文本不重新调用 API。

### 8.1 讯飞星火 WebSocket 流式调用

```python
async def stream_spark(prompt: str, system_prompt: str = "") -> AsyncGenerator[str, None]:
    """
    通过 WebSocket 连接讯飞星火 API，流式返回生成内容。
    """
    ws_url = generate_auth_url(settings.SPARK_API_URL, ...)
    async with websockets.connect(ws_url) as ws:
        await ws.send(build_request(prompt, system_prompt))
        while True:
            response = await ws.recv()
            data = json.loads(response)
            if data["header"]["code"] != 0:
                raise SparkAPIError(data["header"]["message"])
            content = data["payload"]["choices"]["text"][0]["content"]
            if content:
                yield content
            if data["header"]["status"] == 2:  # 结束
                break
```

### 11.2 全局异常处理

```python
@app.exception_handler(Exception)
async def global_handler(request, exc):
    logger.error(f"Unhandled: {exc}", exc_info=True)
    return JSONResponse(status_code=500, content={"success": False, "error": "服务暂时不可用"})
```

所有异常不得向客户端暴露堆栈信息或内部实现细节。

### 11.3 Agent 错误传播

Agent 失败不直接暴露给用户。Service 层捕获 Agent 异常后，根据降级策略返回兜底内容，同时在服务端日志中记录完整错误信息。
