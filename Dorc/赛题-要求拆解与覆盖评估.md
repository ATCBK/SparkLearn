## 1. 赛题要求逐点拆解（任务点一一对应）

### 1.1 基本功能需求 1：对话式学习画像自主构建

**赛题要求原意**
- 通过自然语言对话自动抽取特征。
- 画像维度不少于 6 个。
- 支持“随学随新”的动态更新。

**任务拆解**
1. 建立画像对话引导流程（首问、追问、多轮结束判定）。
2. 实现画像结构化抽取（目标、基础、弱项、偏好、认知风格、学习时长等）。
3. 画像版本化存储（JSON + SQLite）。
4. 提供画像查询与编辑 API。
5. 把练习/辅导行为回流到画像更新。
6. 建立画像质量校验（字段完整性、冲突检测）。

---

### 1.2 基本功能需求 2：多智能体协同的资源生成

**赛题要求原意**
- 必须体现“多智能体架构”。
- 按学生画像生成个性化资源。
- 至少 5 种类型资源（文档、PPT、题目、阅读、视频/动画、代码案例等）。

**任务拆解**
1. 设计多智能体角色（如 Planner、Teacher、Quizer、Reviewer、MediaAgent）。
2. 定义 Agent 间协作协议（输入输出、上下文共享、失败重试）。
3. 建立统一资源生成编排器（按类型路由到不同 Agent）。
4. 打通 5+ 资源类型生产链路。
5. 每类资源建立质量约束与可追踪元数据（来源、版本、生成时间）。
6. 资源生成过程可视化（进度、状态、日志）。

---

### 1.3 基本功能需求 3：个性化学习路径规划与资源推送

**赛题要求原意**
- 基于画像、进度、掌握度、偏好进行动态规划。
- 给出明确学习顺序与步骤。
- 推送文档、视频、题库、实操案例等多类型资源。

**任务拆解**
1. 定义路径图谱模型（阶段、知识点、依赖关系）。
2. 构建动态调度策略（按掌握度/错题/行为实时调整）。
3. 建立推荐引擎（多源特征融合：画像+行为+评估）。
4. 支持资源推送与解释（为什么推荐）。
5. 路径与推荐联动闭环（评估结果回写策略）。

---

### 1.4 基本功能需求 4（加分）：智能辅导

**赛题要求原意**
- 即时答疑。
- 多模态解答（文字、图解、短视频等）。
- 针对性学习引导。

**任务拆解**
1. 聊天会话管理（新建、切换、删除、历史回看）。
2. 角色系统（角色创建/编辑/启用）。
3. 流式问答输出与中断恢复。
4. 文件上传并纳入上下文。
5. 多模态回复扩展（图解卡片、视频讲解链接/生成）。

---

### 1.5 基本功能需求 5（加分）：学习效果评估

**赛题要求原意**
- 跟踪学习行为、练习测试、资源反馈。
- 多维度评估。
- 评估结果反向驱动计划和推荐持续优化。

**任务拆解**
1. 建立学习事件采集规范（答题、资源使用、对话行为）。
2. 建立掌握度模型（知识点级别）。
3. 输出评估报告（趋势、弱项、建议）。
4. 把评估结果回写路径/推荐策略。
5. 提供评估刷新与版本追踪。

---

### 1.6 非功能性需求拆解

#### A. 交互体验
1. 流式输出统一规范（SSE/WebSocket）。
2. Markdown 渲染与卡片化展示。
3. 空态、错态、加载态统一设计。
4. 关键页面无明显 UI/交互错误。

#### B. 开源/工具合规
1. 第三方框架与开源组件清单。
2. 协议与来源说明文档。
3. AI Coding 工具使用说明。

#### C. 防幻觉与内容安全
1. 提示词层安全规则。
2. 输出后置审核（敏感词、事实风险、引用一致性）。
3. 高风险答案降级/拦截/重写策略。

#### D. 性能与稳定性
1. 核心链路响应 SLA。
2. 资源生成进度追踪。
3. 异常可观测性（错误码、日志、链路 ID）。
4. 边界场景（空数据、并发、超时）测试。

## 3. 当前系统“完全没有做”的遗漏点（0→1 必补）

> 这里仅列“当前基本没有落地”的项，不含“已做但不完善”。

1. **明确可运行的多智能体编排框架（强制项）**
- 当前更像单模型多接口调用，缺少可审计的 Agent 协同链路与职责拆分证据。

2. **防幻觉与内容安全过滤机制（强制项）**
- 尚未形成“输入约束 + 输出审查 + 风险降级”的完整安全闭环。

3. **多模态教学视频/动画真实生成链路（核心资源类型项）**
- 目前已有视频页面/占位内容，但“按画像即时生成视频/动画并纳入推荐”未真正闭环。

4. **开源与工具合规文档显式标注（非功能强要求）**
- 缺少集中式“组件来源 + 协议 + AI 工具使用说明”文档落点（可答辩展示）。

5. **赛事交付物中的演示视频（7 分钟）与答辩级 PPT 成果稿**
- 项目实现存在，但“可提交版演示资产”尚未完成定稿。

---

## 4. 当前系统“已做但需加强”的关键项（避免误判为未做）

1. 画像系统已存在，但动态抽取与持续更新深度不足。  
2. 资源生成已覆盖多类型入口，但“多智能体协作证据 + 多模态深度”不足。  
3. 路径与推送功能已可用，但个性化策略仍偏静态。  
4. 智能辅导功能较完整（会话/角色/历史/流式/上传），是当前优势模块。  
5. 评估与掌握度已有框架，但“评估反驱策略”仍需增强。  

---

## 5. 建议的落地顺序（按比赛得分效率）

1. 先补多智能体编排与证据化日志（影响“功能实现及技术要求”大头分）。  
2. 同步补安全过滤与防幻觉（避免答辩被一票否决风险）。  
3. 打通视频/动画资源真实生成链路（提升“创新价值与实用性”）。  
4. 固化文档、PPT、演示视频（直接对应提交要求与文档分）。
