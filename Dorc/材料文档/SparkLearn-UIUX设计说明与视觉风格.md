# SparkLearn UI/UX 设计说明与视觉风格

> **版本**: v1.0 · 2026-05-29
> **适用的项目**: SparkLearn 学而思个性化学习多智能体平台
> **适用端**: Web 桌面端（🔲 手机端适配进行中）
> **技术栈**: Next.js 16 (App Router) + Tailwind CSS + Lucide Icons + TypeScript
> **设计风格**: 浅蓝大厂科技风 + Apple Pure 纯净克制

---

## 1. 设计理念

✅

SparkLearn 的 UI/UX 设计建立在一个根本性的信念之上：**学习工具本身不应该消耗学习者的认知资源**。传统教育类产品常犯的错误是"信息过载"——仪表盘塞满数字、按钮、图表和通知，用户需要先学会使用工具本身，才能开始学习。SparkLearn 的界面设计哲学恰恰相反：界面应该安静地退居幕后，让学习者的心智完全投入到学习内容上。这一理念源自 Apple 设计思想的核心理念——"真正的简约不是没有杂乱可添加，而是没有多余可删减"，我们追求的不是极简主义的冰冷感，而是一种温暖的克制，即足够的信息引导但绝不过量。

以学习效率为核心的设计标准贯穿所有页面。每个页面的信息层级都严格遵循三层模型：L1 导航层告诉用户"在哪"、L2 概览层呈现"整体状态"、L3 内容层承载"实际操作"，而 L4 详情层（弹窗、抽屉、展开面板）仅通过用户主动交互触发。任何同屏页面可见内容不超过 L1-L3，以最大程度降低认知负荷。首屏设计遵循"3 秒原则"——用户打开页面 3 秒内能够理解当前所处位置、整体状态和下一步行动方向。

学习闭环的可视化呈现是 SparkLearn 区别于传统 LMS 产品的关键差异点。平台完整覆盖 画像构建 → 路径规划 → 资源生成 → 学习互动 → 练习评测 → 反馈报告 → 画像更新 的七个环节，每个环节之间都有明确的状态反馈和因果连接。在 UI 层面，闭环的表达方式包括：状态标签的颜色编码系统（已完成绿色/进行中蓝色/待完成灰色）、进度环的 SVG 填充动画、知识点掌握度的渐变色条（绿→橙→红）、以及在报告页面中展示"本次评估已更新画像、路径已重新规划"的回流状态提示。页面之间必须能够解释清楚四个问题：当前信息从哪里来、用户点击后去哪里、操作结果保存到哪里、结果如何影响下一轮学习。

多端一致性是当前设计的核心原则之一。Web 桌面端（≥1200px）为完整布局：左侧 236px 固定导航侧栏 + 主内容区（max-width: 1200px）。🔲 手机端适配正在进行中，当前已定义的适配策略包括：平板端（768px-1199px）侧栏折叠为 64px 图标栏配合单列内容布局；手机端（<768px）侧栏隐藏为汉堡菜单触发，底部需要保留 6 个移动端 Tab（工作台/画像/路径/资源/练习/论坛）作为主要导航方式。统计卡片栅格从桌面 4 列 -> 平板 2 列 -> 手机 1 列递增收缩，内容网格从桌面双列退化为移动端单列堆叠，确保信息层级在任何屏幕宽度下保持一致。

## 2. 视觉风格

✅

### 色彩系统

SparkLearn 的色彩系统源自 `globals.css` 中定义的 `@theme inline` 变量体系，并结合设计迭代最终锁定为"浅蓝大厂科技风"方向。整体色彩以白色和冰蓝色为主，配合少量低饱和蓝紫辅助色，营造专业、智能、可信且温暖的视觉感受。

**主色调**：

| 用途 | CSS 变量 | 色值 | 应用场景 |
|------|----------|------|----------|
| 页面背景 | `--color-bg` | `#f6f8fb` | 全局页面背景，柔和浅灰蓝 |
| 卡片背景 | `--color-bg-card` | `#ffffff` | 所有功能卡片的默认背景 |
| 主文字 | `--color-ink` | `#111827` | 标题、正文主文字，深灰蓝 |
| 次要文字 | `--color-ink-secondary` | `#52627b` | 描述文字、副标题、辅助说明 |
| 三级文字 | `--color-ink-tertiary` | `#6b7280` | 辅助文字、时间戳、元数据 |
| 前置文字 | `--color-ink-disabled` | `#9ca3af` | 占位符、禁用态文字 |
| 主操作蓝 | `--color-blue` | `#2563eb` | 按钮、链接、导航激活态、图标高亮 |
| 深蓝 | `--color-blue-dark` | `#1d4ed8` | 按钮 hover 态、强调型文字 |
| 蓝色浅底 | `--color-blue-light` | `#eff6ff` | 选中态背景、图标底块 |
| 悬浮色 | `--color-bg-hover` | `#f3f4f6` | 导航项 hover、骨架屏、分隔区 |
| 分隔线 | `--color-line` | `#e5e7eb` | 卡片边框、列表分割、输入框描边 |

**功能色（语义色板）**：

| 语义 | 主色/变量 | 浅底色 | 应用场景 |
|------|-----------|--------|----------|
| 成功/绿色 | `--color-success / --color-green` `#16a34a` | `--color-green-light / #eafaf0` | 完成状态、正确答案、正向指标、高置信度标识 
| 警告/橙色 | `--color-warning / --color-orange` `#f59e0b` | `--color-orange-light / #fff7e8` | 需要注意、中等掌握度、中置信度标识 
| 错误/红色 | `--color-danger / --color-red` `#ef4444` | `--color-red-light / #fff0f0` | 错误状态、低掌握度、低置信度标识 
| 紫色 | `--color-purple / #8b5cf6` | `--color-purple-light / #f3efff` | AI 功能标识、图表/分析、思维导图 |
| 青色 | `--color-teal / #68a8f8` | — | 知识库、数据库、搜索、资讯功能 |

**置信度三色圆圈系统**是 SparkLearn 特有的功能色应用模式。在 AI 对话和辅导场景中，每条 AI 回复右上角展示置信度标识圆圈，绿色代表高置信度（score ≥ 0.8）、黄色代表中置信度（0.5 ≤ score < 0.8）、红色代表低置信度（score < 0.5）。圆圈内部显示一个小勾（高）、感叹号（中）或问号（低），悬浮时展示置信度原因码和详细验证说明。这套置信度标识系统实现了"防幻觉控制器"的 UI 可视化，让学习者能够直观判断 AI 回复的可靠程度。

**色彩使用规则**：

- 面积比：白色/浅灰 85%，蓝色主色 10%，其他强调色 5%
- 文字对比度：主文字在白色背景上对比度 ≥ 12:1（满足 WCAG AAA 标准）
- 渐变仅用于 Hero 区域标题高亮和特色卡片装饰，不作为功能性颜色
- 不在同一页面使用超过 4 种颜色的图标底块
- 禁止大面积黑色背景、赛博朋克霓虹色、高饱和撞色以及传统后台管理系统的厚重灰色

### 字体系统

平台使用两种 Google Fonts 通过 Next.js 的 `next/font/google` 进行加载，并在 `layout.tsx` 中以 CSS 变量形式注入。DM Sans 用于英文和数字（字重 400/500/600/700/800），Noto Sans SC 用于中文（字重 300 到 900 全覆盖）。字体备用回退链为 `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`，确保在字体加载失败时仍有合格的中英文回退方案。

**字号梯度**（定义于 globals.css 的 `@theme inline`）：

| CSS 变量 | 字号 | 用途 |
|----------|------|------|
| `--text-display` | 34px | 页面 Hero 标题 |
| `--text-h1` | 26px | 卡片标题、特色卡片标题 |
| `--text-h2` | 18px | 模块标题、章节标题 |
| `--text-h3` | 15px | 卡片内标题、面板标题 |
| `--text-body` | 14px | 正文、列表项、按钮文字 |
| `--text-small` / `--text-caption` | 13px | 导航项、标签、辅助说明 |
| `--text-micro` | 12px | 徽标、极小标注、时间戳 |

正文行高统一为 1.5，标题行高约为 1.1 到 1.4，确保在中文混排时视觉舒适。代码块和 Markdown 渲染中的代码部分使用系统默认等宽字体（`monospace`），保证代码片段可读性和格式对齐。

### 间距与圆角规范

间距系统基于 4px 基数，具体通过 Tailwind 的工具类表达（`gap-3` = 12px、`gap-4` = 16px、`gap-6` = 24px 等），同时辅以 CSS 变量定义的标准化间距 Token。卡片内边距通常为 20px-24px，区段间距为 24px-32px，页面主内容区的水平 padding 为 32px-52px。导航侧栏宽度固定 236px，主内容区最大宽度为 1200px 向左对齐，右侧留有弹性空间避免大屏幕内容过度拉伸。

**圆角系统**（定义于 globals.css 的 CSS 变量）：

| CSS 变量 | 值 | 应用场景 |
|----------|-----|----------|
| `--radius-sm` | 8px | 小按钮、输入框内标识、标签 |
| `--radius-md` | 10px | 中等按钮、列表项、小型容器 |
| `--radius-lg` | 12px | 通用卡片、面板、对话框 |
| `--radius-xl` | 14px | 特色卡片、Hero 卡片、主功能卡片 |
| `--radius-pill` | 999px | 标签胶囊、阶段徽章、头像、进度环背景 |

**阴影系统**（定义于 globals.css 的 CSS 变量）：

| CSS 变量 | 值 | 应用场景 |
|----------|-----|----------|
| `--shadow-sm` | `0 2px 10px rgba(15,23,42,.035)` | 默认卡片阴影 |
| `--shadow-md` | `0 4px 16px rgba(15,23,42,.04)` | 悬浮态卡片 |
| `--shadow-lg` | `0 10px 28px rgba(15,23,42,.08)` | 弹窗、下拉面板 |

所有阴影使用黑色透明度而非彩色，保持 Apple 式的"无色感"阴影风格。卡片的悬浮态通过 `translateY(-3px)` + `--shadow-md` 的组合营造物理纸张抬起的 Z 轴空间隐喻。

## 3. 页面结构

### 一级导航体系

✅ SparkLearn 的一级导航体系按照学习闭环的流程逻辑组织为七大功能模块：

**学习工作台**（`/`）—— 首页入口，提供今日学习任务总览、统计数据卡片、学习热力图和资源推荐。作为学习闭环的启动页，用户在 3 秒内能了解自身学习状态和今日优先任务。

**学习路径**（`/path`）—— 知识图谱 DAG 可视化展示页面。通过阶段步骤指示器和知识树形结构展示学习者的完整知识地图，每个节点标注掌握度状态（已完成/当前/待学习），并提供资源推荐、练习和 AI 解释入口。

**资源中心**（`/generate`）—— 资源生成与资源库的统一入口。六宫格类型选择器 + 需求描述输入区 + 已生成资源列表的左右分栏结构，支持六类资源（课程讲解文档/PPT/思维导图/练习题/拓展阅读材料/代码实操案例）的 AI 生成、在线预览和一键保存。

**练习评测**（`/practice`）—— 个性化题目练习页面，包含练习区、错题本、收藏题目和练习记录四个子模块。支持即时判题、答案解析、错题回流和掌握度回写。

**学习画像**（`/profile`）—— 学习者画像展示与编辑页面，展示知识掌握度、薄弱点、学习偏好、认知风格等画像数据，提供对话式更新画像入口和手动编辑功能。

**智能辅导**（`/tutor`）—— AI 对话辅导页面，支持多角色/多专家协作，含角色管理、对话流、工坊讨论事件流三个核心区域。

**学习空间**（`/agent`）—— 学习伙伴宠物系统，即将承载学习伙伴（Agent Pet）的收养、成长和互动功能。

**其他功能入口**：学习论坛（`/forum`）、学习报告（`/report`）、知识库（`/knowledge`）、视频资源（`/video`）、MCP 插件商店（`/tutor/mcp-store`）及教师端（`/teacher`）作为二级或独立模块存在，不占用一级导航的主 Tab 位置。

### 侧边栏布局

学生端的侧边栏结构定义于 `navigation.tsx` 中，通过 `PAGE_META` 对象维护所有路由的页面元数据（所属分组、页面标题、是否为主 Tab）。侧边栏固定宽度 236px（`w-[236px]`），使用白色背景配合 `border-r border-[#e8edf5]` 的浅灰蓝右侧边框，整体效果干净、通透。侧边栏顶部展示 SparkLearn 品牌标识区域（带蓝色图标底块 + 产品名称 + 副标题 "个性化学习平台"），中间为按功能分组的导航项列表，底部展示用户信息卡片和退出登录入口。

导航项的激活态使用 `bg-[#EEF5FF]` 蓝色浅底搭配 `text-[#2563EB]` 蓝色文字，左侧显示 3px 蓝色竖线指示条；非激活态为灰色文字 `text-[#52627B]`，hover 时切换为浅灰背景 `bg-[#F3F4F6]`。每个导航项由线性图标（20px）+ 文字标签组成，使用 `rounded-lg`（8px 圆角）确保与整体卡片风格一致。

教师端侧边栏（`teacher/layout.tsx`）采用类似结构但导航项不同：工作台、学生管理、干预中心、通知分发、AI 助手、学习报告六项，底部额外增加了"学习广场"和"打开关联大屏"两个快捷入口。教师端主内容区使用教师端背景图（`/gongzuotai-bg.png`）作为固定背景，顶部有 56px 高度的 Sticky 头部栏，包含当前页面名称和教师身份标识。

移动端导航（`MOBILE_TABS`）定义了 6 个底部 Tab：工作台（`Home`）、画像（`User`）、路径（`Map`）、资源（`BookOpen`）、练习（`CheckSquare`）、论坛（`MessageSquare`），使用 Lucide 图标作为视觉标识，各 Tab 通过 `href` 属性决定对应的路由路径。

### 面包屑导航规则

面包屑导航遵循"应用名称 / 模块名称 / 子页面名称"的三级嵌套格式。以 `PAGE_META` 中定义的 `group` 字段为模块分组依据：`group` 为 "SparkLearn" 的一级页面（如学习工作台、学习画像、学习路径等）只显示两级面包屑（`SparkLearn / 页面标题`）；二级页面（如错题本、收藏题目、个人设置等）显示三级面包屑（`SparkLearn / 分组名 / 页面标题`）。面包屑的最后一层为当前页面标题，使用 `text-[#64748b]` 灰色显示，前置层级为可点击链接，使用 `text-[#2563eb]` 蓝色。教师端的面包屑前缀固定为 "教师工作台"，并以 `NAV.find` 匹配当前路由的导航标签作为第二级。

### 🔲 手机端响应式布局方案

手机端响应式布局是当前开发中的待办事项。设计方案已经明确：底部 6 个 Tab 导航（`MOBILE_TABS`）替代桌面端侧边栏作为主要导航方式，侧边栏通过汉堡菜单触发 overlay 展示。所有统计卡片从桌面 4 列退化为平板 2 列再退化为手机 1 列全宽堆叠；双列内容网格（`1fr 360px`）退化为单列，右侧辅助信息栏下移到主体内容下方。主内容区的 `padding` 从 40px-52px 缩减为 16px-20px，确保在小屏幕上不浪费空间。字体层级在手机端保持不变但标题字号可适度缩减（Display 34px → 28px）。globals.css 中已针对 `max-width: 767px` 定义了 `writing-mode` 和 `text-orientation` 的适配规则，确保中文文字在小屏幕上不会出现竖排问题。

## 4. 核心页面设计说明

### 学习工作台（首页）

学习工作台是学习者进入平台后的第一眼页面，设计目标是让用户在 3 秒内完成三个认知判断：今天的学习状态如何、今天最优先做什么、做完之后会影响什么。页面采用 Hero 区域的 34px 大号标题开场，配合个性化问候语和学习连续天数标签（绿色药丸形 `Streak` 徽章），在视觉锚点上建立情感连接。Hero 下方是 Featured Card 特色卡片，使用深色渐变背景 + 径向蓝色光晕，展示当前学习路径节点、环形进度图（SVG `stroke-dashoffset` 动画）和阶段步骤指示器，在白色为主的页面中制造强烈的视觉焦点。

四个统计数据卡片等宽并列（学习时长、任务完成率、练习正确率、连续学习天数），每张卡片带对应颜色的顶部色条——蓝色/绿色/橙色/紫色，快速建立色彩编码系统，让用户一眼感知各维度的状态。卡片右侧展示一个大数字和趋势指示，左侧为对应颜色的图标。四个指标覆盖了学习的投入量、执行力、理解度和坚持度四个维度，不超过 Miller's Law 的 7±2 认知上限。

页面主内容区采用 `1fr 360px` 双列布局，左列承载今日任务列表（带勾选完成态的交互清单）和个性化资源推荐卡片，右列固定 360px 展示辅助信息：当前学习阶段、画像标签胶囊、知识掌握度进度条和 GitHub 风格的贡献热力图（记录每日学习次数）。热力图单元格 hover 时显示具体日期和学习次数的 Tooltip。所有内容卡片支持骨架屏加载状态（ `DashboardSkeleton` 组件），在 API 数据返回前提供平滑的用户过渡体验。

### 智能辅导页

智能辅导页是平台最核心的 AI 交互入口，采用三栏布局设计：左侧 280px 角色/知识库设置面板、中间弹性宽度的对话流区域、右侧 320px 工坊事件流面板。左侧面板顶部展示当前选择的 AI 辅导角色（如"Python 导师"），下方可切换角色列表，每个角色包含名称、人设、背景、风格指南和规则五项配置。角色卡片支持创建、编辑和删除操作，新建角色通过弹出 Modal 对话框完成表单填写。面板底部是知识库文件管理区，支持拖拽上传文件到角色关联的知识库。

中间对话流区域是页面的主体，实现了完整的 AI 对话交互闭环。用户消息右对齐（浅蓝色背景气泡 `bg-blue-light`），AI 回复左对齐（白色背景气泡），每条 AI 消息右上角显示置信度三色圆圈标识。对话内容支持 Markdown 渲染（`react-markdown` + `remark-gfm`），代码块使用等宽字体和浅灰背景。✅ 多模态图片融入对话：AI 可以在回复中嵌入 Base64 编码的图片，实现"AI 生成的图片直接显示在对话流中"的多模态交互效果。底部输入栏固定在对话区底部，包含文本输入框、文件上传按钮、语音输入切换按钮（`Mic`/`MicOff`）和发送按钮。输入栏上方是快捷建议按钮行，提供当前上下文相关的推荐问题。

右侧工坊事件流面板展示多 Agent 协作讨论的实时过程。启用工坊模式后（选择多个角色），系统会逐轮展示各 Agent 的发言内容（标注 Agent 名称和角色类型），采用类似"多人在线讨论"的视觉形式——每条事件带彩色角色标识头像 + 半透明对话气泡。讨论结束后自动生成结构化总结卡片，包含"核心结论"、"分步行动清单"和"可追问问题"三个区段，用户可将总结内容采纳为 AI 助手的提示词草稿。

### 学习路径页

学习路径页的核心是知识点的 DAG 可视化图谱，将学习者的知识状态映射为有向无环图中的节点和边。页面采用左侧路径图 + 右侧节点详情面板的左右分栏布局。左侧路径图中，每个节点用不同颜色表示掌握状态：蓝色（`completed` 已完成）、蓝色实心高亮（`current` 当前正在学习）、灰色（`pending` 待学习）。节点之间的有向边（箭头连接线）表示前置依赖关系，让学习者直观理解"必须掌握 A 才能学习 B"的知识结构。

右侧详情面板展示当前选中节点的深度信息：掌握度评分（0-100 数字 + 渐变色进度条）、知识点薄弱来源（来自练习错题、报告评估或 AI 诊断）、节点的达标条件说明、推荐学习动作（查看资源、生成资源、做练习、请教 AI 助手）。面板底部是路径调整记录时间线，展示因错题回流或画像更新导致的路径变化。🔲 关联个性化路径功能正在进行——即每个节点的状态不仅反映掌握度，还动态关联学习者画像中的薄弱点和学习目标，自动调整节点的优先级和推荐策略。

页面顶部为阶段步骤指示器，用横向胶囊形进度条展示学习阶段（基础阶段 → 进阶阶段 → 实战阶段等），当前阶段高亮为蓝色，已完成阶段打绿色勾，未完成阶段显示为灰色虚线框。每个阶段标签下方标注关键知识点和预计学习时长，点击可切换阶段视图。路径图支持缩放和拖拽，在大规模知识点图中，用户可以通过搜索框快速定位特定知识点节点并自动居中到视口。

### 资源生成页

✅ 资源生成页实现了完整的 AI 资源生产闭环，采用"六步生成 SOP + 资源类型选择 + 配置参数"的流程化设计。页面顶部是六宫格资源类型选择器，每个类型使用大尺寸卡片（带图标 + 图标底块 + 类型名称 + 简短描述）：课程讲解文档（`FileText` + 蓝色）、PPT 课件（`Presentation` + 橙色）、思维导图（`Brain` + 紫色）、练习题（`CheckSquare` + 橙色）、拓展阅读材料（`BookOpen` + 绿色）、代码实操案例（`Code` + 蓝色）。选中态卡片显示蓝色描边 + 浅蓝背景 + 右上角勾选标识。

类型选择器下方是需求描述输入区域，包含一个大尺寸文本输入框（多行自然语言描述）和知识点标签快捷选择器（从当前学习上下文中自动提取的标签，如 "Python / 函数 / 闭包"）。✅ 固定模板选择功能已实现——用户可在输入描述的同时从预设模板库中选择符合当前资源类型的结构化模板，模板展开显示各节标题和占位说明，点击选择后自动填充到需求描述中。输入框右侧是"生成"按钮，使用蓝色渐变主按钮样式，点击后触发 AI 生成流程。

生成过程通过六步进度指示器展示：分析学习画像 → 读取路径节点 → 生成内容大纲 → AI 内容创作 → 安全检查 → 生成完成。每个步骤在不同阶段显示不同状态：待处理（灰色圆点）、处理中（蓝色旋转动画）、已完成（绿色勾选）。生成过程中，生成状态栏显示实时的日志输出，让学习者了解 AI 正在做什么，增加生成过程的透明度和可信度。生成完成后，左侧"已生成资源列表"自动追加新生成的资源项（带类型图标和生成状态标签），右侧预览面板自动展开展示生成内容，底部提供"下载"和"保存到资源中心"两个操作按钮。

### 资源库页

资源库页承载所有已保存和已生成学习资源的集中管理和检索功能，采用搜索筛选栏 + 左侧资源列表 + 右侧详情预览的邮件客户端式布局。顶部搜索栏为大型输入框，支持按资源名称全文搜索，搜索框带 `Search` 图标和 placeholder 提示。搜索框下方是类型筛选标签行，使用胶囊形按钮实现单选/多选筛选：全部、讲义、PPT、思维导图、练习题、阅读材料、代码案例、视频资源。

左侧资源列表以表格式卡片排列展示每项资源：左侧为对应类型的彩色图标底块（36px），中间为资源名称（14px 粗体）、类型标签胶囊、来源标识（"AI 生成"/"教师上传"），右侧为学习进度条（百分比数字 + 浅色进度背景条）和创建时间。列表项 hover 时轻微高亮，点击后右侧预览面板展示资源详情。支持的资源操作包括：继续学习（跳转资源详情或播放器）、在线预览（面板内渲染）、下载源文件、删除和"学完去练习"快捷入口。

右侧资源详情预览面板（固定 360px）根据资源类型渲染不同内容：文档类型使用 Markdown 渲染；PPT 类型使用 Schema 驱动的组件树渲染（基于 `PptDeck` 类型定义的幻灯片结构）；视频类型显示 HTML5 播放器；思维导图使用树形组件展开。预览面板底部展示关联知识点标签列表和推荐动作按钮。空状态时面板展示浅蓝色文件夹插画 + "选择左侧资源查看详情"引导文案。

### 练习评测页

✅ 练习评测页实现了"个性化题目展示 + 即时判题 + 答案解析 + 错题本/收藏"的完整练习闭环。页面采用 Tab 切换结构（4 个标签：练习 Tab、错题本、收藏题目、练习记录），默认停留于练习 Tab。练习区使用 70% 宽度的主内容区承载题目，采用题干 + 选项区域的纵向布局：单选题选项为圆形单选按钮（`radio` 样式），多选题为方形复选框（`checkbox` 样式），选中态用蓝色背景高亮，正确答案用绿色边框 + 绿色勾标识，错误答案用红色边框标识。

提交判题后，页面自动展开答案解析区，显示正确答案、详细解析文档（支持 Markdown 渲染）和错误原因分析（错因归类为"知识点不熟"、"审题不清"、"计算错误"等）。解析区下方自动展示补弱资源推荐卡片（1-3 个推荐资源，带推荐理由），学习者可一键跳转到对应资源页面开始学习。题目区域右上角有心形收藏按钮和错题标识。

错题本 Tab 以红色图标底块（`AlertCircle` + 红色）区分每条错题，列表展示错题内容摘要、错误次数、错误时间以及对应知识点标签。每条错题可展开查看完整题目、自己的错误答案和正确答案的对比，以及重新练习的入口。收藏题目 Tab 结构类似，使用橙色星形图标（`Star`）标识。练习记录 Tab 以时间线形式展示历史练习记录，每条记录包含练习时间、题目数量、正确率环形图和掌握度变化指示。所有 Tab 支持空状态处理，无数据时显示浅蓝色插图 + 引导文案。

### 学习画像页

✅ 学习画像页以中心 3D 学习角色形象为核心，周围环绕注释卡片（Annotated Cards）的方式展示学习者的全面画像数据。中心头像区域使用圆形轨道装饰 + 淡蓝色光圈和学习图标点缀，增强科技感和个性化陪伴感。头像下方展示学习者姓名、年级、专业和当前学习阶段标签，以及"上次更新于 XX"的画像更新时间戳。

周围注释卡片分别展示：学习目标（卡片选择式可视化展示，如"期末提分"、"竞赛准备"等）、当前薄弱点（红色和橙色标签的多选标签云）、已掌握知识点（绿色标签列表，带掌握度百分比）、学习偏好（包含学习风格、日常学习时间段的图标 + 文字描述）和认知风格标签。每张注释卡片用虚线连接线指向中心头像，形成"解剖注释"的视觉效果，类似 Apple 产品发布会的功能介绍图。

页面右侧提供画像操作面板，包含"手动编辑画像"按钮（弹出编辑表单弹窗）、"对话式更新画像"入口（唤起 AI 对话面板，通过自然语言描述更新画像字段）和"查看更新记录"时间线面板。更新记录以时间倒序展示每条画像变更的来源（练习结果、AI 对话、手动修改）、变更字段和变更前的值。底部区域展示能力雷达图（6 个维度的蛛网图：知识掌握、实践能力、逻辑思维、创新能力、表达沟通、自主学习）和最近学习过的资源列表（点击可继续学习）。编辑入口使用紫色 `Sparkles` 图标标识 AI 能力。

### MCP 插件商店页

✅ MCP 插件商店页实现了 Model Context Protocol 插件的发现、配置和管理功能。页面主体为插件的卡片网格列表（3 列自适应布局），每个插件卡片包含：插件名称（15px 粗体）、插件描述（两行截断 + 悬浮全文 Tooltip）、来源标签（系统内置 `system` 或用户自定义 `user`）、传输协议标签（`stdio` 或 `http` 胶囊标识）、启用/禁用开关（Toggle 组件）以及状态指示器（在线绿色圆点/离线灰色圆点/错误红色圆点）。

卡片右下角是"配置"按钮（齿轮图标），点击触发右侧配置抽屉（Drawer）从侧边滑入。配置抽屉包含：传输协议选择（stdio/HTTP 单选）、端点 URL 输入框（HTTP 协议时显示）、启动命令输入框（stdio 协议时显示）、命令行参数列表（带 + - 增删按钮的动态参数项）、环境变量 JSON 文本区域、各阶段超时时间设置、以及底部"测试连接"和"保存配置"操作按钮。✅ 工具弹窗已实现——已连接的 MCP 服务可展开工具列表弹窗，展示该服务提供的所有 MCP 工具名称、描述和参数说明，方便用户了解插件的实际能力。

页面顶部为操作栏：搜索框（按插件名称搜索）、来源筛选下拉框（全部/系统/用户）、新建插件按钮（蓝色主按钮，触发创建表单弹窗）。空状态时展示浅蓝色拼图插画 + "暂无可用的 MCP 插件，点击右上角创建第一个插件"的引导文案。

### 教师大屏

✅ 教师大屏是教师端的功能模块集合，通过独立的教师侧边栏布局（`teacher/layout.tsx`）承载多个子页面。Dashboard 数据看板页展示全平台学生的聚合学习数据：顶部四个大数字指标卡片（在学人数、活跃率、平均完成率、干预预警数），使用大型彩色图标底块（44px 圆形）和大号数字（32px 粗体）。中间为学习数据可视化区域：折线图展示近 7/30 天活跃人数趋势、柱状图展示各阶段学生分布、环形图展示知识点整体掌握度分布（高/中/低 三段占比）。底部为实时预警列表，展示需要教师干预的学生名单（标注预警原因、薄弱知识点、最近学习状态）。

Broadcast 广播页实现教师向指定学生群体发送通知和学习材料的功能。页面采用上下分栏布局：上部分为广播编辑区，包含标题输入框、目标选择器（全体学生/指定学生列表多选）、富文本内容编辑器以及材料文件上传区域（支持多文件上传并展示文件列表）。下部分为已发送广播列表，按发布时间倒序排列，每条广播展示标题、目标范围、发送时间和送达人数，支持查看详情和删除操作。

Students 学生管理页展示全平台学生的列表视图，每行包含学生姓名、年级、专业、当前学习阶段、最近活跃时间和学习进度百分比。列表上方为筛选栏（按年级/专业/阶段筛选）和搜索框（按姓名搜索）。点击学生行展开详情面板，展示该学生的画像摘要、学习路径进度、最近练习记录和教师干预建议按钮。

### 学习社区（论坛）

学习社区（`/forum`）是学习者之间交流学习经验、分享资源和提问解答的社区平台。页面采用传统的论坛帖子列表布局：顶部为操作栏，包含搜索框、标签筛选和"发布帖子"主按钮（蓝色）。帖子列表以卡片形式展示每篇帖子：左侧为帖子的热度统计（喜欢数 + 评论数 + 收藏数 + 浏览数的纵向数字组合），中间为帖子标题（可点击进入详情）、内容摘要（2 行截断）、标签胶囊列表（如 "Python"、"函数"、"新手提问"），右侧为作者头像 + 用户名 + 发布时间。帖子卡片 hover 时背景微亮，暗示可点击交互。

帖子详情页（`/forum/[postId]`）展示完整帖子内容和评论区。帖子正文支持 Markdown 渲染和附件下载。评论区采用嵌套回复结构，每条评论显示用户身份、内容、时间戳和回复按钮。回复框位于页面底部，支持基本的文本输入和附件上传。发帖页（`/forum/new`）提供标题输入框、标签多选器、Markdown 编辑器（带预览 Tab）和文件上传区域。"我的帖子"页面（`/forum/my`）展示当前用户发布的全部帖子列表，支持编辑和删除操作。

### 🔲 Admin 端页面

Admin 端管理页面是平台的后台管理系统，目前处于规划阶段。设计方向将继承教师端的侧边栏布局框架，但导航项会调整为面向系统管理员的元管理功能：用户管理（学生/教师账号的 CRUD、角色权限分配）、系统配置（模型参数、API 密钥管理）、知识库全局管理（公共知识库的上传与索引监控）、运营数据分析（全平台使用统计、学习行为趋势）、审核中心（论坛内容审核、资源质量审核）和日志管理（系统错误日志、API 调用记录）。Admin 端的视觉风格将保持与主平台一致的浅蓝白色系统，但在数据表格和配置表单区域会增加信息密度，以适应管理员的高效操作需求。

### 🔲 趣味性/游戏化交互元素（待定）

平台规划了一系列趣味性和游戏化元素来增强学习动力的持续性，具体实现方案仍在探讨中。首要引入的是学习伙伴（Agent Pet）系统，学习者可选择一只 3D 动漫风格的宠物作为学习伴侣（可选角色包括灵狐、智宝、芯助、星猫等 8 种），宠物具有等级（Lv.）、经验值（XP）、亲密度等属性，这些属性与学习行为关联——完成任务 +XP、连续学习 +亲密度、做错题但坚持复习 +亲密度。宠物展示在独立的学习空间页面（`/agent`），包含当前伙伴展示区（带发光圆台和柔和光效）、属性面板（学习专注、知识探索、思维敏捷、坚持毅力、幸运值五项）、宠物图鉴收集和每日任务奖励四个区域。

其他可能引入的游戏化元素包括：成就徽章系统（完成特定里程碑后解锁的 Badge 标识）、学习打卡连续天数奖励、知识点掌握的星星评分机制、以及排行榜/学习小组的轻社交竞争机制。所有游戏化设计遵循"轻量、不喧宾夺主、服务于学习动机而非替代学习本身"的原则，严格避免重度的 RPG 化、金币商城氪金感、以及会分散学习注意力的过度特效和动画。

## 5. 交互规范

### 流式输出响应（SSE 打字机效果）

✅ 平台全面采用 SSE（Server-Sent Events）流式传输来构建 AI 生成内容的实时输出体验。前端使用 `useStreaming` 自定义 Hook（位于 `frontend/src/lib/hooks/useStreaming.ts`）来管理流式响应的生命周期：通过 `fetch` API 的 `ReadableStream` 读取器逐块接收后端 SSE 推送的文本数据，使用 `TextDecoder` 解码后以增量方式拼接（`setText(prev => prev + chunk)`），在界面上形成逐字出现的"打字机效果"。流式输出应用于智能辅导对话（AI 回复逐字呈现）、资源生成过程日志（实时展示 AI 处理进度）、AI 学习报告生成（总结内容逐步呈现）以及视频脚本润色结果展示等场景。

流式输出过程中 UI 的配套设计包括：光标闪烁指示器（`TypewriterLoader` 组件，显示跳动的三个点 + "AI 正在思考..."字样）、流式内容区域的自动滚动（`scrollIntoView({ behavior: 'smooth' })`确保最新内容始终可见）、以及流式结束后的完成态标识（光标消失 + 内容区域的微动画过渡）。用户可以在流式输出过程中随时暂停（Pause 按钮），暂停后内容保持已输出部分不变，用户可阅读已完成的内容而不受持续打字机效果干扰。

### 骨架屏加载状态

✅ 骨架屏（Skeleton）加载状态通过 `Skeleton` 组件实现，组件内部使用 `animate-pulse` CSS 动画 + `bg-bg-hover` 浅灰背景色 + `rounded-[12px]` 统一圆角，模拟内容加载过程中的占位效果。通用 `Skeleton` 组件通过 `className` 属性配置不同尺寸（`h-4 w-32` 模拟文字行、`h-48 w-full` 模拟大卡片等）。

专用骨架屏组件 `DashboardSkeleton` 复刻了首页学习工作台的真实布局结构：Hero 区域（3 条不同宽度的文字骨骼线）→ Featured Card 大卡片 → 4 列统计卡片行 → `1fr 360px` 双列内容网格（含任务列表骨骼和推荐卡片骨骼）。这种基于真实布局的骨架屏设计确保了从加载态到内容态的视觉连续性，用户在数据加载期间能预先感知内容的布局结构，减少等待焦虑。资源列表页、论坛页等也采用类似的骨架屏策略，每个页面的骨架屏结构与其实际内容布局一一对应。

### 错误状态处理

✅ 错误状态处理通过 `ErrorState` 组件（位于 `frontend/src/components/ui/ErrorState.tsx`）统一管理，支持三种错误类型：服务器错误（`server`，图标 ⚠️、"服务器暂时无法响应"）、网络错误（`network`，图标 📶、"网络连接异常"）和通用错误（`generic`，图标 ❗、"发生了意外错误"）。每种错误类型有预设的标题和描述文案，同时支持通过 `title` 和 `description` props 自定义覆盖。组件采用垂直居中布局：大号图标圆形背景 → 错误标题（`text-h3`） → 错误描述文（`text-ink-secondary`，最大宽度 520px） → 蓝色"重试"按钮（仅在提供 `onRetry` 回调时显示）。

`ErrorState` 在智能辅导页、视频播放页等多个页面中被使用，当 API 调用失败、网络断开或服务器返回异常时展示，替代了传统的"白屏"或浏览器原生错误提示。在辅导对话页中，当消息发送失败时，错误状态出现在消息列表末尾（而非替换整个页面），用户可以选择重试发送该条消息或忽略继续对话，体现了"优雅降级"的错误处理策略。

### 空状态设计

✅ 空状态通过 `EmptyState` 组件统一处理，在新用户首次进入某页面或数据为空时展示。组件包含三个元素：一个柔和色彩的插画图标区域、一句引导性文案标题、以及一个操作按钮（如"创建第一个"、"去生成"、"去上传"等）。空状态不显示赤裸裸的"暂无数据"字样，而是用正向的引导语言告诉用户可以做什么。例如：知识库页空状态为浅蓝色文件夹插画 + "还没有上传任何资料，点击上方上传按钮添加你的第一个学习资料"；MCP 插件商店空状态为拼图插画 + "暂无可用的 MCP 插件，点击右上角创建第一个插件"；资源生成页的空状态为 "描述你想要的学习资源，AI 将为你量身生成内容"。

### 置信度标识 UI

✅ 置信度标识 UI 是 SparkLearn 防幻觉系统在界面上最直观的表现形式。在每条 AI 回复的右上角显示一个三色圆圈，通过颜色快速传达 AI 对该回答的可信度评估。圆圈内部根据置信度级别动态切换图标：高置信度（`level: 'high'`，score ≥ 0.8）—— 绿色圆圈 + 内部白色勾形图标 + 色值 `#16a34a`，代表 AI 对该回答的准确性有较高把握；中置信度（`level: 'medium'`，0.5 ≤ score < 0.8）—— 黄色圆圈 + 内部白色感叹号图标 + 色值 `#f59e0b`，表示回答可能存在不确定性，建议进一步验证；低置信度（`level: 'low'`，score < 0.5）—— 红色圆圈 + 内部白色问号图标 + 色值 `#ef4444`，警示用户该回答可能不准确，务必核实后再采纳。

鼠标悬浮在置信度圆圈上时，会弹出 Tooltip 卡片，展示置信度的详细验证信息：置信度评分（数字）、置信度原因码列表（如 "source_verification_failed"、"domain_boundary_crossed" 等）、以及知识库验证状态说明。这套 UI 通过 `ConfidenceInfo` 类型（定义于 `types.ts`）驱动，包含 `score`、`level`、`color`、`label`、`message` 和 `reasonCodes` 六个字段，由后端防幻觉控制器在每次生成回答时同步返回。

### 🔲 细节优化：字体大小调整

字体大小调整功能是规划中的无障碍增强项。当前平台的字体层级已经完全通过 CSS 变量体系化定义（`--text-display` 至 `--text-micro`），未来将支持用户在设置页面选择"小/中/大"三档字号偏好，通过修改根 CSS 变量的比例系数（0.9x / 1.0x / 1.15x）来全局调节所有文字的显示大小，同时保持各个字体层级之间的比例关系和行高的一致性。这一功能对视觉障碍用户和老年学习者尤为重要。

## 6. 多模态资源展示

### Markdown 渲染

✅ 平台在多个场景中使用 Markdown 作为 AI 生成内容的标准格式，前端使用 `react-markdown` 库配合 `remark-gfm` 插件进行渲染。Markdown 渲染应用于智能辅导对话中的 AI 回复内容（包括标题层级、列表、代码块、表格、引用块等）、全局 AI 助手的回答面板、资源生成内容的预览、以及论坛帖子的正文展示。代码块通过 `react-markdown` 的内置代码渲染支持语法高亮，代码区域使用等宽字体（`monospace`）和浅灰色背景（`bg-[#f6f8fa]`），包含代码语言标签和复制按钮，确保编程教学场景下代码片段的清晰可读性和可复用性。表格渲染带斑马纹行背景色区分，引用块使用左侧蓝色竖线和浅蓝背景以示区分。

### PPT 预览

✅ PPT 预览功能通过 Schema 驱动的组件树实现。后端返回 `PptDeck` 类型的结构化 JSON 数据，包含 `deck_id`、`theme` 主题、`title` 标题和 `slides` 幻灯片数组。每张幻灯片（`PptSlide`）由 `layout` 字段（`cover` 封面 / `bullets` 要点列表 / `process` 流程步骤 / `summary` 总结）决定渲染模式。封面幻灯片渲染为大号标题 + 副标题的居中布局；要点列表幻灯片渲染为带序号的 bullet point 列表（每项有 `step` 数字编号）；流程幻灯片渲染为步骤指示器组件，展示分步流程图；总结幻灯片渲染为要点汇总列表和叙述文字。

所有 PPT 渲染均在右侧预览面板中以组件形式呈现，支持幻灯片之间的翻页切换（上一页/下一页按钮 + 幻灯片页码指示器）。幻灯片渲染采用白色背景模拟投影效果，每张幻灯片内的文本层级严格遵循 `title` → `subtitle` → `bullets` → `narration` 的信息层级递减关系。PPT Schema 数据同时被资源生成管线中的 PPT 课件生成引擎所消费，实现从 Prompt 描述到结构化 Schema 再到可视化呈现的完整链路。

### 视频播放器

✅ 视频播放器基于 HTML5 `<video>` 原生播放器构建，实现沉浸式学习体验。播放器采用 16:9 宽高比，居中放置在视频页面的主要内容区域。播放控件包括：播放/暂停、音量调节、播放进度条（可拖拽）、全屏切换、播放速度调节（0.5x / 1x / 1.5x / 2x）以及画质选择。播放器使用深色半透明控件叠加层，在不干扰视频画面的同时提供完整的操作能力。

字幕功能通过 `<track>` 元素加载 WebVTT 格式的字幕文件（由 `subtitleUrl` 字段指定），支持字幕的开启/关闭切换。时间线导航功能通过 `timelineUrl` 字段获取视频章节时间线数据，以右侧面板的形式展示视频的章节分段列表，每段显示标题、预览图和起始时间，点击可快速跳转到对应章节位置。视频还支持 `sceneUrl` 场景描述数据的加载，用于在教学场景中标注关键知识点出现的时间节点。视频生成管线支持 AI 自动生成字幕、章节分段和场景标注，实现"视频 + AI 自动笔记"的集成学习体验。

### AI 图片生成展示

✅ AI 图片生成展示支持通过 Base64 编码字符串内联图片到对话流中。在智能辅导和多 Agent 工坊对话场景中，AI 生成的图片（如数据图表、概念示意图、流程图等）以 Base64 编码格式嵌入 AI 回复的消息体内，前端自动识别并渲染为 `<img>` 标签。图片展示采用响应式宽度（`max-width: 100%`），在代码块以外独立成行显示，确保不同尺寸的图片都能在对话气泡内合理呈现。图片带有轻微的圆角和阴影效果，与对话气泡的视觉风格保持一致。大尺寸图片支持点击放大查看，在图片状态下使用 Modal 浮层展示全尺寸图片。这套 Base64 内联机制避免了图片文件的外部托管和 URL 管理问题，让 AI 生成的多模态内容可以直接在对话上下文中显示，不需要学习者跳转到外部链接查看。
