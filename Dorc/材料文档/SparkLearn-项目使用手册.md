# SparkLearn 项目使用手册

> 适用版本：v3.0+ | 最后更新：2026-05-29

## 1. 快速开始 ✅

本章指导新用户在 10 分钟内完成 SparkLearn 的首次启动和使用体验。从环境准备到打开首页，每一步都可独立执行。

**第一步：环境准备**。确保你的 Windows 10/11 电脑已安装以下软件：Python 3.12+（在终端执行 `python --version` 验证）、Node.js 20+（执行 `node --version` 验证）、Git（执行 `git --version` 验证）。如果尚未安装，请分别从 python.org、nodejs.org 和 git-scm.com 下载安装包进行安装。安装 Node.js 时会自动安装 npm 包管理器。

**第二步：克隆项目并安装依赖**。打开终端（建议使用 Windows Terminal 或 PowerShell），执行 `git clone <仓库地址> D:\Project_building\SparkLearn` 将项目克隆到本地。克隆完成后，分别安装后端和前端依赖。后端：`cd D:\Project_building\SparkLearn\backend` 然后 `pip install -r requirements.txt`，安装完成后执行 `python -m playwright install chromium` 下载 Playwright 浏览器。前端：`cd D:\Project_building\SparkLearn\frontend` 然后 `npm install`。两个安装命令可以在不同终端窗口中并行执行以节省时间。

**第三步：配置环境变量**。在项目根目录 `D:\Project_building\SparkLearn\` 下创建 `.env` 文件。最低要求配置讯飞星火大模型的凭据：`spark_app_id`、`spark_api_key` 和 `spark_api_secret`（从讯飞开放平台控制台获取）。详细的配置模板和每项说明请参考《SparkLearn 部署文档》第 3 章。如果暂时没有 API 凭据，可以先将 `use_mock_data=true` 设为启用，系统将使用内置模拟数据运行。

**第四步：启动服务**。打开两个终端窗口分别启动后端和前端。后端：`cd D:\Project_building\SparkLearn\backend` 执行 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload`，看到 `Uvicorn running on http://127.0.0.1:8000` 表示启动成功。前端：`cd D:\Project_building\SparkLearn\frontend` 执行 `npm run dev`，看到 `Ready in Xs` 表示启动成功。

**第五步：访问应用**。打开浏览器访问 `http://localhost:3000`，你将看到 SparkLearn 的学习工作台首页，包含学习概览卡片、热力图和今日推荐。侧边栏列出所有功能入口。至此，你已经成功启动了 SparkLearn。

## 2. 使用模式说明 🔲

SparkLearn 支持两种使用模式，满足不同用户的学习习惯和场景需求。

**平滑使用模式（探索式自由使用）** 🔲：此模式为规划中功能。用户无需设定明确的学习目标和路径，可以自由浏览知识图谱、随机生成学习资源、与 AI 导师随意对话、尝试不同类型的练习。系统会默默记录用户的每一次交互，逐渐构建隐式学习画像，在适当的时候提供个性化推荐。这种模式降低了平台的使用门槛，特别适合有好奇心但尚未形成明确学习规划的用户。当前开发优先级中，该模式排在结构化流程之后。

**目标导向的结构化学习流程** ✅：这是当前平台的主推模式，也是功能最完整的模式。用户的典型旅程为：(1) 首次进入平台时填写学习画像问卷（学习目标、当前水平、薄弱点、偏好、可用时间）；(2) 系统基于画像生成个性化学习路径，展示知识图谱和阶段规划；(3) 在路径中选择知识点，系统推荐对应的学习资源；(4) 使用 AI 资源生成器创建针对性学习材料（文档、练习、思维导图等）；(5) 进入智能辅导模块与 AI 导师对话，深入学习具体知识；(6) 通过练习评测检验学习效果，错题自动加入错题本；(7) 查看学习报告了解阶段学习情况，系统基于报告更新画像和路径。这个闭环设计确保学习者每一步都有明确的方向和可衡量的产出。

## 3. 各功能模块使用说明 ✅

### 3.1 学习工作台 ✅

学习工作台是登录后的默认首页，提供学习状态的全局概览和快捷入口。页面顶部为导航栏，左侧为功能侧边栏，中央主区域展示学习仪表盘。

**学习概览卡片**：页面顶部展示四张统计卡片——总学习时长（小时）、任务完成率（百分比）、练习正确率（百分比）和连续学习天数。这些数据从后端 `/api/learning/dashboard` 接口获取，基于用户的所有学习记录聚合计算。卡片数据每完成一项学习活动后自动更新。

**学习热力图**：概览卡片下方是类似 GitHub 贡献图的学习热力图，每个格子代表一天，颜色深浅表示当天学习任务的数量。数据来源于 `contribution_days` 表，记录每日的学习操作次数。热力图直观展示学习规律性和持续性，帮助用户自我监督。

**今日推荐**：热力图下方展示系统根据用户画像和当前学习阶段推荐的今日学习资源。推荐算法综合考虑画像中的薄弱知识点和路径规划中的当前阶段节点。每条推荐包含资源类型图标、标题、推荐理由和操作按钮（开始学习/查看详情）。推荐资源点击后跳转到对应功能页面。

**快捷操作**：页面右上角提供"开始学习"按钮，点击后快速跳转到学习路径页面；"每日一句"卡片展示 AI 生成的学习激励语，通过 `/api/learning/daily-quote` 接口获取。侧边栏底部显示用户头像和名称，点击可进入个人设置页。

### 3.2 学习画像 ✅

学习画像是 SparkLearn 个性化推荐和路径规划的基石，系统通过画像了解用户的学习目标、水平、薄弱点和偏好。支持三种建档方式。

**问卷建档（Onboarding）**：新用户首次进入平台时的引导式画像建立。问卷分为五个步骤：(1) 选择学习目标（如"期末提分""技能提升""兴趣学习"等，可多选）；(2) 自评当前水平（"零基础""有一些基础""中等水平""较为熟练"）；(3) 标记薄弱知识点（如"函数""几何""语法"等，可多选）；(4) 选择学习偏好（"实践型""理论型""视觉型""听觉型"等）；(5) 设置每日可用学习时长（"15分钟以下""15-30分钟""30-60分钟""60分钟以上"）。填写完成后点击"开始学习之旅"，系统通过 `/api/profile/onboarding` 接口保存画像并跳转到学习工作台。

**手动编辑画像**：在侧边栏点击"学习画像"进入画像页面。页面展示当前画像的完整信息，包括：学习目标标签列表、知识水平等级、薄弱知识点标签云、学习偏好类型、认知风格、每日学习时长和当前学习阶段。每个字段右侧都有编辑按钮，点击后弹出编辑面板修改相应内容。修改通过 `/api/profile` 的 PATCH 请求提交。

**查看版本历史**：画像支持版本管理（`profiles` 表的 `version` 字段）。每当画像发生实质性变更时版本号自动递增。在画像页面底部点击"版本历史"可以查看历次画像的快照，了解画像随学习进程的演变。这在复盘学习效果和验证 AI 推荐的准确性时非常有用。

**设置页面**：侧边栏 → 学习画像 → 设置（或直接访问 `/profile/settings`）。设置页面包含用户基本信息（姓名、邮箱、专业、年级）的编辑功能，以及对平台行为的偏好设置。

### 3.3 学习路径 ✅+🔲

学习路径模块展示 AI 为用户规划的个性化学习知识图谱和阶段计划，从宏观到微观指导学习进程。

**知识图谱浏览** ✅：页面主体为交互式知识图谱可视化。节点代表具体知识点，颜色表示掌握状态：绿色（已掌握）、蓝色（学习中）、灰色（待学习）。节点之间有连线表示前置依赖关系——必须先掌握前置节点才能解锁后续节点。点击任意节点弹出详情面板，展示节点名称、当前掌握度百分比、前置依赖节点列表、学习内容清单和推荐资源类型。图谱支持缩放、拖拽和平移操作。

**阶段切换** ✅：知识图谱上方是学习阶段横向导航条。阶段按顺序排列（如"基础入门""进阶提升""精通应用"），已完成阶段标记为绿色对勾，当前阶段高亮显示，未开始阶段为灰色。点击任意已解锁阶段，知识图谱自动过滤展示该阶段的知识点。阶段切换时系统更新用户画像中的 `current_stage` 字段。

**路径规划功能** ✅+🔲：在路径页面顶部输入框中输入学习目标（如"掌握 Python 数据分析"），点击"生成路径"，系统调用 AI 生成个性化的学习建议和资源推荐。返回结果包含：路径 ID、目标描述、分阶段建议列表（每个建议含文本描述和详细说明）、推荐资源列表（含资源标题、标签和链接）、学习阶段划分（阶段标题、描述和包含的知识节点）。生成的学习路径可保存并导出。🔲 标记表示 AI 路径规划的节点级建议细化功能仍在优化中。

**节点详情与建议** ✅：点击知识图谱中的任一节点，可查看 AI 针对该节点的学习建议。`/api/path/node-advice` 接口返回：节点掌握度评估、学习建议、通俗解释、下一步行动计划、推荐学习内容和推荐资源列表。这些建议基于用户画像和当前节点状态个性化生成。

### 3.4 资源生成 ✅

AI 资源生成器是 SparkLearn 的核心学习材料生产工具，支持 6 种资源类型的流式生成。

**选择资源类型**：在侧边栏点击"资源生成"进入生成页面。页面展示 6 种资源类型的卡片：文档（长文讲解）、思维导图（树形知识结构）、练习题（选择题/填空题/解答题）、阅读材料（拓展阅读文章）、代码示例（可运行的代码片段）和 PPT（演示文稿）。点击任意卡片选中类型，卡片高亮并显示类型说明。

**输入提示词**：类型选择下方是提示词输入框。输入你想要学习的主题或知识点，如"Python 装饰器原理""牛顿第二定律""光合作用过程"。提示词要具体明确，包含关键词，避免过于宽泛（如"数学"这样的提示词会使生成结果不够聚焦）。输入框下方实时显示字数统计。

**查看生成进度**：点击"开始生成"按钮后，页面切换到生成进度视图。系统通过 SSE（Server-Sent Events）流式推送生成状态：连接建立 → AI 开始处理 → 正在生成内容 → 内容逐段渲染 → 生成完成。进度条实时展示完成百分比，右侧面板流式渲染已生成的内容。不同类型的资源有不同的进度展示方式：文档显示 Markdown 逐段渲染，思维导图显示节点逐步扩展，PPT 显示大纲转幻灯片进度。

**预览与保存**：生成完成后自动跳转到预览页面。可以查看完整的生成内容（文档支持 Markdown 渲染、思维导图支持交互式展开/折叠、代码示例支持语法高亮）。预览页面提供以下操作：复制内容、重新生成（使用相同提示词）、调整参数后重新生成、保存到资源库。保存时自动添加生成时间戳和资源类型标签。

### 3.5 资源库 ✅

资源库集中管理所有已生成和收藏的学习资源，支持浏览、筛选和导出操作。

**浏览资源**：在侧边栏点击"资源库"进入列表页面。资源以卡片网格布局展示，每张卡片显示：资源标题、类型标签（文档/PPT/思维导图/练习/阅读/代码/视频）、生成日期和状态（已完成/生成中/失败）。支持分页加载，默认每页 20 条资源。卡片点击进入详情页查看完整内容。

**筛选功能**：列表顶部提供多种筛选条件：(1) 按资源类型筛选（点击类型标签即筛选，支持多选）；(2) 按状态筛选（已完成/生成中/失败）；(3) 按时间排序（最新优先/最旧优先）；(4) 关键词搜索（在标题和内容中模糊匹配）。筛选条件组合使用可快速定位到目标资源。

**导出功能**：在资源详情页点击"导出"按钮，支持以下导出格式：文档类资源导出为 Markdown 文件（`.md`）、PPT 类资源导出为 PPTX 文件（通过讯飞智文接口）、思维导图导出为 JSON 结构数据、练习资源导出为可打印的试卷格式。导出文件通过浏览器下载到本地。视频资源支持下载 MP4 文件、音频文件（MP3）、字幕文件（SRT/VTT）和场景时间轴 JSON。

**资源删除**：在列表页和详情页均提供删除按钮。删除操作不可逆，系统会弹出确认对话框防止误删。删除资源时同步清理关联的存储文件（如生成产物、上传附件等）。

### 3.6 智能辅导 ✅

智能辅导是 SparkLearn 中最复杂的交互模块，提供 AI 导师对话、知识库辅助、置信度评估、图片生成和工坊协作等能力。

**选择导师角色**：进入辅导页面后，首先选择一个导师角色。系统预设了多种角色模板（如"耐心导师""严厉教练""学霸同学"等），每个角色定义了 persona（人格描述）、background（背景设定）、style_guide（话术风格）和 rules（行为规则）。点击"角色管理"进入角色编辑页面，可以创建自定义角色、编辑现有角色或删除角色。角色切换后对话风格和回答方式会相应改变。

**创建对话与发送消息**：选择角色后点击"新建对话"，输入对话标题（可选，默认使用第一条消息的前几个字）。进入对话界面后，在底部输入框中输入问题，按 Enter 或点击发送按钮。AI 回复通过 SSE 流式展示，文字逐字出现在屏幕上，实现类似打字的自然效果。对话界面支持 Markdown 渲染（标题、列表、代码块、表格）和数学公式。

**上传文件与知识库选择**：在输入框旁点击"附件"图标可上传文件（支持 PDF、DOCX、TXT、图片等格式）。上传的文件会与当前对话关联，AI 可以读取文件内容并结合上下文回答问题。点击"知识库"图标可选择已索引的知识库文档，AI 将基于选中的文档内容进行检索增强生成（RAG），回答中会标注信息来源和引用。

**置信度查看**：每条 AI 回复下方有置信度指示器，以彩色圆点和等级标签展示：绿色表示高置信度（模型自评答案可靠）、黄色表示中等置信度、红色表示低置信度（答案可能不准确，建议进一步验证）。点击置信度指示器展开详细信息，包括置信度分数（0-1）、评分等级、评估说明和具体原因代码。这体现了系统的防幻觉机制，帮助用户识别可能需要查证的内容。

**图片生成**：在对话中使用特定的提示词格式或通过工具栏的"生成图片"按钮触发 AI 图片生成能力。生成的图片嵌入在对话流中，支持点击放大查看和保存到本地。该功能依赖后端与 AI 模型的图片生成接口集成。

**工坊模式**：工坊（Workshop）是一种多角色协作学习模式。在工坊中，系统创建多个 AI 角色（如"分析者""提问者""总结者"），每个角色从不同角度参与讨论和知识构建。工坊流程包括三个阶段：画像分析（分析学习者的知识水平和盲点）、多角色讨论（不同角色轮流发言辩论）、综合总结（整合各方观点形成最终结论）。工坊页面以时间线形式展示各角色的发言，用不同颜色区分角色，支持实时流式渲染。进入工坊需要先在辅导页面选择"工坊"模式而非"知识问答"模式。

### 3.7 练习评测 ✅

练习评测模块提供 AI 生成和题库练习两种模式，支持自动判题、错题管理和收藏功能。

**开始练习**：点击侧边栏"练习评测"进入练习页面。页面展示题目列表，每道题显示题号、题目类型图标（单选/多选/填空）、题目内容和选项。通过 `/api/quiz` 接口获取题目，系统会根据用户的画像和薄弱知识点个性化出题。用户逐题作答，选择或填写答案后点击"下一题"，也可以使用键盘快捷键（1/2/3/4 选择选项，Enter 确认）。

**提交答案与判题反馈**：完成全部题目后点击"提交"按钮，系统通过 `/api/quiz/submit` 接口逐题判题。判题结果以卡片列表形式展示：正确题目显示绿色对勾和题目知识点标签，错误题目显示红色叉号、你的答案、正确答案和详细的题解说明。判题由 AI 驱动，除了判断对错外还提供每个选项的解析和知识点关联。

**错题本**：侧边栏 → 练习评测 → 错题本（或 `/practice/mistakes`）。错题本自动收集所有答错的题目，展示题目内容、错误答案、正确答案和解析。支持以下操作：(1) 重新作答（答对后自动从错题本删除）；(2) 查看详细解析；(3) 按知识点标签筛选错题；(4) 按时间排序（最近错题优先）。错题本是复习的利器，建议定期回顾和清空。

**收藏题**：侧边栏 → 练习评测 → 收藏题（或 `/practice/favorites`）。在做题过程中，遇到有价值的题目可点击题目旁的心形图标加入收藏。收藏夹展示所有已收藏题目，支持按知识点标签筛选。收藏题可以导出为错题本合集或试卷格式进行打印复习。

**练习记录**：侧边栏 → 练习评测 → 练习记录（或 `/practice/records`）。记录页面展示历次练习的统计信息：练习时间、题目数量、正确率、用时等。支持按时间范围筛选，以列表和时间轴两种视图展示。点击任意记录可回顾当次练习的完整题目和作答情况。

### 3.8 学习报告 ✅

学习报告模块提供阶段学习情况的 AI 总结和数据可视化分析。

**周期选择**：进入报告页面后，顶部是报告周期选择器。支持按周、月、自定义时间段查看报告。选择不同周期后，页面数据自动刷新。报告基于该时间段内的所有学习活动记录生成。

**数据可视化**：报告页面的上半部分是数据图表区域，包括：(1) 每日学习时长柱状图（展示该周期内每天投入的学习时间）；(2) 学习时间分布饼图（按学习活动类型拆分为辅导对话、练习评测、资源阅读、视频观看等类别）；(3) 知识点掌握度雷达图（展示各薄弱知识点的当前掌握程度变化，可对比周期前后的差异）；(4) 学习任务完成进度条（展示该周期内计划任务与实际完成任务的对比）。所有图表使用 ECharts 5.6.0 渲染，支持鼠标悬停查看详细数据。

**AI 学习总结**：图表区域下方是 AI 生成的学习总结文本。总结内容包括：学习概况（总时长、完成率、正确率）、各知识点掌握度变化分析、学习习惯和规律性评价、下一阶段学习建议。AI 总结通过 `/api/report?period=weekly` 等接口获取，由星火大模型根据学习数据和画像生成。

**导出与分享**：报告页面右上角提供导出按钮，支持将当前报告导出为 PDF 文件或分享链接。导出的 PDF 包含完整的图表和 AI 总结文本，适合打印或存档。分享链接可以发送给教师或学习伙伴。

### 3.9 MCP 插件商店 ✅

MCP（Model Context Protocol）插件商店是 SparkLearn 能力扩展的平台，允许用户添加和管理外部工具服务。

**浏览系统服务**：进入 MCP 商店页面（侧边栏 → 智能辅导 → MCP 商店，或 `/tutor/mcp-store`），页面展示所有已注册的 MCP 服务列表。系统预置了若干常用服务（如文件系统服务、网页抓取服务等），标记为 `source: system`。用户添加的服务标记为 `source: user`。每个服务卡片显示名称、描述、来源类型、传输模式（stdio/HTTP）、当前启用状态和最近一次连接测试状态（在线/离线/未知）。

**添加自定义服务**：点击页面右上角"添加服务"按钮打开配置表单。填写以下字段：(1) 服务名称（必填，便于识别）；(2) 描述（可选，说明服务功能）；(3) 传输模式：选择 `stdio`（通过命令行启动 MCP 服务进程）或 `http`（连接到已有 HTTP 端点）；(4) 对于 stdio 模式，填写命令和参数（JSON 数组格式，如 `["npx", "-y", "@modelcontextprotocol/server-filesystem"]`）和环境变量（JSON 对象格式）；(5) 对于 HTTP 模式，填写端点 URL；(6) 可选配置：启用状态、启动超时时间（ms）、工具调用超时时间（ms）、长时间任务超时时间（ms）。填写完成后点击"保存"。

**测试连接与启停**：每个服务卡片提供三个操作按钮。(1) "测试连接"按钮：系统尝试连接 MCP 服务并返回连接结果（成功/失败及错误信息），连接状态更新到服务卡片的在线状态指示器上；(2) "启用/禁用"开关：控制服务是否对辅导对话和 Agent 宠物可用，禁用的服务不会出现在工具列表中；(3) "删除"按钮：移除服务注册并释放相关资源。服务的连接测试结果和最后测试时间记录在数据库中。

**查看工具列表**：点击服务卡片展开可查看该 MCP 服务提供的工具列表。每个工具显示名称、描述和参数定义。工具列表通过 `/api/mcp/services/{id}/tools` 接口从 MCP 服务端获取，反映服务的实时能力。在辅导对话中，AI 可以根据用户问题自动选择并调用合适的 MCP 工具。

### 3.10 Agent 宠物 ✅

Agent 宠物是一个拟人化的学习助手，能执行搜索、总结、对比和推荐四类任务，拥有等级成长系统。

**领养宠物**：首次进入 Agent 宠物页面（侧边栏 → Agent 宠物，或 `/agent`），系统提示"你还没有学习伙伴"。点击"领养宠物"按钮进入领养流程：(1) 为宠物命名（1-10 个字符，支持中英文和数字）；(2) 选择宠物外观（狐狸 fox、猫头鹰 owl、机器人 robot、猫 cat、龙 dragon、企鹅 penguin、兔子 bunny、熊猫 panda）；(3) 选择性格（简洁 concise、详细 verbose、鼓励式 encouraging）。性格影响宠物回答的风格和字数。完成后点击"确认领养"，通过 `/api/agent/pet` POST 接口创建宠物。

**分配任务**：领养宠物后，页面展示宠物的基本信息卡片（名称、外观、等级、经验值进度条、已解锁技能列表）。在任务区域选择任务类型：(1) 搜索（search）：让宠物搜索网页上的学习资源，返回资源标题、摘要、URL 和来源；(2) 总结（summarize）：输入文本或 URL，宠物提取关键信息并生成结构化摘要（主题、要点列表、结论）；(3) 对比（compare）：输入一个概念或问题，宠物从不同角度提供对比解释；(4) 推荐（recommend）：让宠物根据你的学习情况推荐适合的学习资源。选择类型后输入任务描述（1-200 字符），点击"执行任务"。注意：某些任务类型需要宠物达到特定等级解锁（Lv.1 解锁搜索、Lv.2 解锁总结、Lv.3 解锁对比、Lv.4 解锁推荐）。

**任务执行与结果查看**：任务创建后，页面展示实时执行步骤日志（如"开始处理任务""正在搜索网页...""正在提取内容...""正在生成结果...""任务完成!"）。任务状态有四个阶段：pending（等待执行）、running（执行中）、completed（完成）、failed（失败）。任务完成后，结果以卡片形式展示：(1) 搜索结果展示列表，每个条目包含标题、摘要、来源网站和链接；(2) 总结结果展示主题、关键要点列表和结论；(3) 对比结果展示多个来源的解释和对比总结；(4) 推荐结果展示推荐资源及其推荐理由。对结果可提交反馈（有用/无用）或收藏到知识库。

**升级与成长**：每次完成任务可获得经验值（搜索 +10、总结 +20、对比 +25、推荐 +15）。经验值累计推动宠物升级（Lv.1-Lv.5，对应阈值 0/100/300/600/1000）。每升一级解锁新的任务类型能力。宠物信息卡上的经验值进度条实时更新，升级时展示升级动画和祝贺提示。

### 3.11 教师端 ✅

教师端是一个独立的功能子系统，面向教师用户提供班级管理、教学大屏、消息广播和资料管理能力。

**大屏看板**：教师端核心功能，通过 `/teacher/dashboard` 访问（或在学生端侧边栏 → 教师工具 → 教师大屏）。大屏使用 ECharts 渲染丰富的数据可视化图表，展示班级整体学习情况：学生活跃度排行、知识点掌握度热力图、学习时长分布、任务完成率趋势等。数据来源于班级 9 名学生的学习记录（1 名真实用户 + 8 名模拟学生）。当后端服务未启动时，大屏自动降级为演示数据模式，所有图表仍可正常展示。大屏还包含一个 AI 教学助手聊天窗口，教师可以咨询教学策略和学生分析建议。

**发送广播**：教师可通过 `/teacher/broadcast` 向学生发送广播消息。创建广播时填写标题和正文内容，选择目标群体（全部学生或指定学生），可选附加教学资料文件。广播消息会出现在学生端"我的消息"或通知中心。广播记录页面展示历史广播列表，包含发布时间、目标群体和阅读状态统计。

**管理教学资料**：教师可以上传和管理教学资料（课件 PPT、参考文档、练习卷等）。上传的资料可以关联到特定的广播消息中发送给学生。资料管理页面展示所有已上传文件，支持浏览、下载和删除操作。上传通过 `/api/teacher/materials` 接口。

**学生管理**：教师可查看班级学生列表（`/teacher/students`），点击学生姓名进入详情页（`/teacher/students/[id]`），查看该学生的学习画像、学习进度、近期活动记录和掌握度雷达图。这为教师提供了数据驱动的个性化辅导依据。

**其他教师功能**：教师端还包含 AI 教学助手（`/teacher/ai`）、教学干预建议（`/teacher/interventions`）和学习报告汇总（`/teacher/reports`）等功能模块。

### 3.12 学习社区 ✅

学习社区是一个面向所有学习者的交流平台，支持发布帖子、回复评论、点赞收藏等社交功能。

**浏览与发布帖子**：社区页面（侧边栏 → 学习社区）以帖子列表形式展示，每条帖子显示标题、作者、标签、发布时间、点赞数、评论数和收藏数。支持按标签筛选、按时间或热度排序。点击"发布新帖"进入编辑器（`/forum/new`），填写标题（必填）、正文内容（支持 Markdown 格式）、选择标签（如"学习心得""问题求助""资源分享""组队学习"等），可选上传附件文件（图片、文档等）。发布后帖子出现在列表顶部。

**回复与互动**：点击帖子标题进入详情页（`/forum/[postId]`），展示帖子完整内容和评论区。评论支持楼中楼回复，每条评论显示用户名、时间和内容。帖子作者和评论者会收到互动通知。帖子支持点赞（红色心形图标）和收藏（星形图标），用户的点赞和收藏记录可在个人中心查看。

**学习广场**：学习广场（`/plaza`）是社区的子版块系统，按主题分为多个版块：(1) 经验分享（`/plaza/experience-share`）：分享学习经验和心得的专区；(2) 问答求助（`/plaza/qa`）：提问和解答学习问题；(3) 资源分享（`/plaza/resource-share`）：分享优质学习资源的专区；(4) 组队学习（`/plaza/team-study`）：寻找学习伙伴和组队项目。在广场首页可以看到各版块的最新热帖。

**个人社区主页**：在社区中可以查看自己的历史发帖（`/plaza/history`）、发布过的评论（`/plaza/my-comments`）和点赞过的帖子（`/plaza/my-likes`），方便回顾和交流管理。

## 4. 常见问题 FAQ ✅

本章汇总用户在使用 SparkLearn 过程中最常遇到的问题及解决方案。

**Q1: 导师角色切换后对话风格没有变化？**
A: 角色切换仅对新创建的对话生效，现有对话的上下文和角色关联在创建时即锁定。如需在新角色下对话，请创建新的对话会话并选择目标角色。每个对话的侧边栏顶部显示当前关联的角色名称。如果已创建对话但在对话中切换了角色，该对话仍沿用旧角色——这是预期行为，目的是保持对话风格的一致性。

**Q2: 资源生成失败或返回空内容？**
A: 资源生成依赖星辰Agent开发平台的 Bot 配置。排查步骤：(1) 确认 `.env` 中 `coze_api_token` 和对应类型的 `coze_bot_id_resource_*` 已正确填写；(2) 登录星辰Agent开发平台控制台确认 Bot 处于"已发布"状态（草稿状态的 Bot 无法通过 API 调用）；(3) 检查 Bot 的 API 调用配额是否已用尽；(4) 查看后端终端日志中是否有来自星辰Agent开发平台的具体错误信息；(5) 尝试切换到 `use_mock_data=true` 使用模拟资源数据进行前端展示验证。

**Q3: 知识库文件上传后显示"索引失败"？**
A: 知识库文件处理流程为：上传 → `pending` → `processing` → `indexed`（索引成功）或 `failed`（索引失败）。失败原因排查：(1) 文件格式不受支持（当前支持 PDF、DOCX、TXT）或文件损坏无法解析；(2) Embedding API 配置不正确（检查 `.env` 中的 `xfyun_emb_*` 系列配置）；(3) 文件过大导致处理超时（`xfyun_emb_timeout_sec` 默认 20 秒，大文件可能不够）；(4) 可在知识库文件列表中点击文件旁的"重新索引"按钮重新触发处理。每次索引操作都会记录处理状态和错误信息。

**Q4: MCP 服务添加后测试连接失败？**
A: MCP 连接失败的常见原因：(1) stdio 模式：确认命令路径正确，MCP 服务端程序已安装（如 `npx` 安装的 npm 包需确保 Node.js 环境正常），环境变量配置正确；(2) HTTP 模式：确认端点 URL 可访问，从本机浏览器或 curl 测试连通性；(3) 启动超时时间（`startup_timeout_ms`）设置过短，MCP 服务端启动较慢时可增大此值；(4) 查看后端日志获取详细的错误堆栈信息；(5) 确保 MCP 服务端程序版本与 SparkLearn 的 MCP 协议版本兼容。

**Q5: 视频生成注意事项** ✅：视频生成是一个耗时操作（从数十秒到数分钟），需要耐心等待。视频生成流程包括：AI 脚本编写 → TTS 语音合成 → HTML 幻灯片渲染。注意事项：(1) TTS 语音合成需要配置讯飞 TTS 凭据（`.env` 中的 `xf_tts_*` 系列），否则视频只有画面没有配音 🔲（TTS 音色选择和切换的稳定性仍在优化中）；(2) 视频 AI 模式（`video_ai_provider=openai_compatible`）需要配置可用的 AI API key，如果 AI 不可用，设置 `video_ai_fallback_enabled=true` 可自动降级为 HTML 幻灯片本地渲染模式；(3) 生成中的视频可以在列表页面查看进度，完成后支持下载 MP4 视频文件、音频文件和字幕文件；(4) 视频生成器支持多种输出格式，可在生成前选择期望的成品类型。

**Q6: 数据库连接问题** ✅：SparkLearn 默认使用 SQLite 文件数据库，无需额外安装数据库服务。但需注意：(1) 确保 `backend/data/db/` 目录存在且可写；(2) SQLite 不支持高并发写入，多进程同时写入可能导致 `database is locked` 错误——开发环境中单用户模式不会触发此问题；(3) 当需要迁移到 PolarDB PostgreSQL 时，需修改 `.env` 中的数据库连接字符串并更新 `backend/app/db.py` 的数据库连接层代码（从 `sqlite3` 切换到 `asyncpg` 或 `psycopg2`）。PolarDB 提供更高的并发能力和云端数据持久化，适合多用户部署场景。

**Q7: Agent 宠物任务一直显示"执行中"不完成？**
A: 可能的原因和处理：(1) 任务正在通过浏览器搜索互联网，搜索过程需要 20-40 秒，请耐心等待；(2) 如果超过 2 分钟仍未完成，检查 Playwright 浏览器是否已安装（`python -m playwright install chromium`）；(3) 检查网络连接是否正常（浏览器搜索需要访问搜索引擎）；(4) 如果 `.env` 中 `NANOBOT_PET_ENABLED=true`，确认 Nanobot 服务是否正常运行（访问 `http://127.0.0.1:8900/health`）；(5) 如果 Nanobot 不可用，系统会自动回退到内置 LLM 模式执行任务；(6) 如果所有方式都失败，任务状态会变为 `failed` 并显示错误信息。可查看后端日志获取详细错误信息。

**Q8: 练习评测的 AI 判题不准确？**
A: AI 判题由星火大模型驱动的 NLP 理解能力完成，对于开放性问题（如编程题、论述题）的判题可能存在偏差。建议：对于填空题（fill_blank），AI 会进行语义相似度匹配而非精确字符串对比，因此语义等价但表述不同的答案也可能被判为正确；对于编程题，AI 关注逻辑正确性而非语法精确性；如果发现明显的判题错误，可通过题目旁的"反馈"按钮提交问题，帮助改进判题逻辑。

**Q9: 如何查看和管理系统的 API 调用量？**
A: 当前版本未内置 API 调用量统计面板。可以通过以下方式了解各服务的用量：(1) 讯飞星火大模型：登录讯飞开放平台控制台 → 我的应用 → 用量统计；(2) 星辰Agent开发平台：登录星辰平台控制台 → Bot 管理 → API 调用记录；(3) 讯飞 TTS/Embedding/智文：各自对应服务的控制台查看用量；(4) OpenAI/兼容接口：对应提供商的控制面板。各 AI 服务的费用和配额是独立计算的，请注意合理使用。🔲 Admin 管理端规划中将包含统一的 API 配额监控面板。

**Q10: 前端页面访问 404 或白屏？**
A: 请确认：(1) 前端开发服务器是否正常运行（终端中应显示 Next.js 编译成功且无报错）；(2) 访问的 URL 路径是否正确——所有学生端功能页面位于 `(shell)` 路由组下，路径格式为 `/功能名`（如 `/profile`、`/path`、`/generate`）；(3) 教师端页面位于 `(teacher)` 路由组下，路径格式为 `/teacher/功能名`（如 `/teacher/dashboard`）；(4) 辅导模块页面位于 `/tutor/子页面`（如 `/tutor/workshop`、`/tutor/mcp-store`）；(5) 如果页面白屏但有 HTML 框架，可能是 API 请求失败导致数据加载不完整——打开浏览器开发者工具（F12）查看 Network 和 Console 面板排查具体错误。
