# SparkLearn 第三阶段：移动端多端展示适配说明

## 1. 本阶段目标

本阶段仅实现 **多端展示适配（Web 移动端）**，不扩展业务功能，不新增后端接口，不改动数据结构。

- 目标：让现有系统在手机端（390 基线，兼容 360-430）可用、可读、可导航。
- 范围：前端展示层与壳层导航适配。
- 不包含：功能新增、接口改造、业务流程重构。

## 2. 已完成改造

### 2.1 壳层与导航

已在 `frontend` 单仓内完成桌面/移动双壳层分支：

- 移动端壳层：
  - 顶部页头（支持主页面标题、子页面返回）
  - 底部 5 Tab 导航（工作台/画像/路径/资源/练习）
- 桌面端壳层保持原有结构（Sidebar + Topbar）
- 根据断点自动切换：
  - `< 768px`：移动壳层
  - `>= 768px`：桌面壳层

### 2.2 文案与映射统一

- 新增统一导航与页面元信息映射，消除壳层标题与导航文案不一致问题。
- 修复原侧栏/页头中的乱码文案（仅展示文案层面）。

## 4. 明确不做（本阶段边界）

- 不新增页面功能
- 不修改 API 协议与请求参数
- 不新增数据库字段
- 不改动业务规则（练习、资源生成、路径规划逻辑保持原样）

## 5. 验收标准（展示层）

1. 手机宽度下可正常进入底部 5 个主页面，并保持激活态高亮。
2. 子页面显示页头标题，支持返回上一级。
3. 页面内容无明显横向溢出、无关键按钮遮挡。
4. 桌面端布局与主导航行为不回归。
