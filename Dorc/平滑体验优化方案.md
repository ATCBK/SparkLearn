# SparkLearn 平滑体验优化方案

## 问题诊断

当前项目是**强目标形式** — 用户必须带着明确学习目标才能使用产品。核心矛盾：

> 用户打开 App → 看到空任务台 → 不知道干什么 → 退出

理想体验应该是：

> 用户打开 App → 看到有趣内容 → 随便点点 → 发现自己的薄弱点 → 自然进入学习流程

---

## 改动方案（共 5 项）

### 方案一：Onboarding 增加"先逛逛"入口

**现状**：注册后强制 5 步问卷，无法跳过，用户还没看到 App 就被问了一堆问题。

**改动**：在 onboarding 页面顶部增加"先逛逛，稍后再填"的跳过按钮。

**涉及文件**：
- `frontend/src/app/onboarding/page.tsx`

**具体做法**：
1. 在步骤指示器下方添加"先逛逛"文字按钮（第 291 行附近）
2. 点击后直接跳转到首页，本地存储标记 `onboarding_skipped=true`
3. 首页检测到该标记时，显示引导提示条"完成学习画像，获得更精准的推荐"，可随时回到 onboarding

**效果**：用户注册后 3 秒内就能看到 App 全貌，降低流失率。

---

### 方案二：首页增加"探索区"模块

**现状**：首页只有任务列表和缺口推荐，没有目标的用户看到的是空状态。

**改动**：在首页任务列表下方增加一个"随便看看"的探索区。

**涉及文件**：
- `frontend/src/app/(shell)/page.tsx`

**具体做法**：
1. 在现有 `grid` 布局（第 148 行）下方添加新卡片区域
2. 探索区包含 3 类内容卡片：
   - **每日一题**：随机一道趣味编程题，零门槛即可开始
   - **热门知识**：展示 3-5 个热门学习话题（如"Python 能做什么"、"程序员的一天"）
   - **同学在学**：展示社区动态摘要（从 plaza 接口取数据）
3. 每个卡片可直接点击进入对应内容，不需要任何前置条件

**效果**：用户即使没有目标，也有东西可以点、可以看、可以做。

---

### 方案三：学习路径增加"自由浏览"模式

**现状**：路径是线性锁死结构，locked 节点无法查看，用户被强制按顺序走。

**改动**：允许用户点击任何节点查看内容预览，但练习/评测仍需按顺序完成。

**涉及文件**：
- `frontend/src/app/(shell)/path/page.tsx`

**具体做法**：
1. 将 `locked` 节点的样式从完全灰色不可点击，改为半透明可点击（第 60-63 行数据中 `locked` 状态的节点）
2. 点击 locked 节点时，弹出预览面板，展示该节点的学习内容简介（如"模块导入：学习如何在 Python 中导入和使用模块"）
3. 预览面板底部显示"完成前面的内容后解锁"的提示，但用户已经知道后面会学什么
4. 将节点的 `status` 逻辑调整：`locked` 改为 `locked-preview`，允许查看但不允许做练习

**效果**：用户能"看到"完整学习路径，产生期待感，而不是面对一堵灰色的墙。

---

### 方案四：Agent 对话支持自由聊天

**现状**：Agent 只有 3 个预设任务类型（搜索资料、文章摘要、对比搜索），像一个工具而非伙伴。

**改动**：在任务类型选择器中增加"随便聊聊"选项，支持开放式对话。

**涉及文件**：
- `frontend/src/components/agent/AgentChat.tsx`

**具体做法**：
1. 在 `TASK_TYPE_OPTIONS`（第 11-15 行）中增加第 4 个选项：
   ```ts
   { id: 'chat', label: '随便聊聊', icon: <MessageCircle className="h-3.5 w-3.5" />, placeholder: '随便说点什么...' }
   ```
2. 当 `taskType === 'chat'` 时，输入内容直接发送给后端 LLM，不走任务执行流程
3. 后端 `/api/agent/chat` 接口需要支持无 task_type 的自由对话模式（或新增一个轻量接口）
4. Agent 的回复不做结构化渲染，直接用 markdown 渲染文本

**效果**：用户可以像聊天一样和 AI 互动，降低使用门槛。

---

### 方案五：侧边栏增加"发现"入口 + 广场内嵌

**现状**：广场藏在侧边栏最底部，新标签页打开，几乎没人能找到。

**改动**：将广场提升为一级导航，且在 App 内直接展示。

**涉及文件**：
- `frontend/src/components/layout/Sidebar.tsx` — 添加导航项
- `frontend/src/app/(shell)/layout.tsx` — 移动端 BottomTabBar 考虑是否替换一个 tab
- `frontend/src/app/(shell)/discover/page.tsx` — 新建发现页（可选，或直接复用 plaza 内容）

**具体做法**：
1. 在 `NAV_ITEMS`（Sidebar.tsx 第 15-24 行）中，在"学习中心"分组下添加：
   ```ts
   { label: '发现', href: '/discover', icon: <Compass className="h-5 w-5" />, group: '学习中心' }
   ```
2. 新建 `/discover` 页面，将 plaza 的核心内容（资源分享、问答、同学动态）嵌入到 shell 布局内
3. 去掉 plaza 的 `target="_blank"` 新标签页行为
4. 广场底部的入口卡片保留，但改为"进入完整广场"的补充入口

**效果**：用户在侧边栏就能看到"发现"，随时可以浏览社区内容，不再需要明确目标。

---

## 实施优先级

| 优先级 | 方案 | 理由 |
|--------|------|------|
| P0 | 方案一：Onboarding 跳过 | 最快见效，改 1 个文件，直接影响注册转化率 |
| P0 | 方案二：首页探索区 | 改 1 个文件，解决"打开不知道干嘛"的核心问题 |
| P1 | 方案四：Agent 自由聊天 | 改 1 个前端文件 + 1 个后端接口，让 AI 从工具变成伙伴 |
| P2 | 方案三：路径自由浏览 | 改 1 个文件，改善但非关键 |
| P2 | 方案五：发现入口 | 需要新建页面，工作量最大，可后续迭代 |

## 验证方式

1. 新用户注册后，能在 5 秒内不填问卷直接看到首页内容
2. 首页在没有学习任务时，仍有可点击、可浏览的内容
3. 学习路径中，所有节点都能点击查看预览
4. Agent 对话中，用户可以输入任意内容并得到回复
5. 侧边栏有明确的"发现"入口，点击后在 App 内展示社区内容
