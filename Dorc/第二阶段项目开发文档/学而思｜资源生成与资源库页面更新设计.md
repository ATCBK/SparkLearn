# 学而思｜资源生成与资源库页面更新设计

## 1. 文档目的

本文档用于定义 SparkLearn 第二阶段资源相关页面的职责边界和交互更新方案，重点覆盖 `/generate`、`/resources`、`/feed`、`/video` 以及首页资源推荐模块。本文档是在资源生成页交互讨论、SOP 流式页面原型和功能层级改造方案基础上形成的页面设计文档。

本文档聚焦五个问题：

第一，`/generate` 如何从普通 AI 生成表单改造成按步骤执行的资源生成 SOP 页面。

第二，资源推送为什么应放到首页工作台，而不是继续作为一级导航或堆在资源生成页中。

第三，资源库为什么应作为独立页面，并支持搜索、筛选、保存资源管理和点击预览。

第四，视频/动画为什么应作为资源生成类型出现，而不是独立视频学习模块。

第五，资源生成完成后如何进入保存、学习、练习、AI 精灵讲解和下载等闭环动作。

本文档是前端页面设计文档，尚不代表代码已经完成。`/generate` SOP 化、资源库预览抽屉、首页推荐资源模块、`/feed` 下沉入口、`/video` 多模态预览入口、视频/动画资源类型支持仍待实现或验证。

## 2. 结论

资源相关页面需要从“多个平级入口”调整为“主流程 + 子页面”的结构。

最终页面职责为：

```text
/                 学习工作台：承接资源推送、今日任务、路径节点、错题补弱
/generate         资源生成：只做 SOP 生成流程
/resources        资源库：保存资源、管理资源、点击预览
/feed             推荐资源详情页：从首页进入，不做一级导航
/video            视频预览页：从资源库或资源详情进入，不做一级导航
```

`/generate` 不再承担完整资源库和大面积推荐列表。它只负责完成一次资源生成任务：

```text
确认学习上下文
-> 选择资源类型
-> 配置生成需求
-> 生成与预览
-> 保存到资源库 / 开始学习
```

---

## 3. 改造目标

### 3.1 产品目标

1. 让 `/generate` 成为清晰的资源生成 SOP，而不是复杂功能堆叠页。
2. 让资源推送回到学习工作台，服务“今天该学什么”的首页任务流。
3. 让资源库成为独立页面，承担资源保存、管理和预览。
4. 让视频/动画成为资源类型之一，而不是一级导航或独立业务模块。
5. 让资源生成完成后自然进入学习、练习、精灵讲解和报告回流。

### 3.2 工程目标

1. 优先改前端结构和页面交互，不强依赖后端立即新增接口。
2. 保留现有 `/generate`、`/resources`、`/feed`、`/video` 路由。
3. 页面请求仍统一通过 `frontend/src/lib/api/real.ts`。
4. 新增预览、保存、推荐来源等能力时，字段映射不得散落在页面组件中。

---

## 4. 当前问题

当前资源相关能力分散在：

```text
/generate
/resources
/feed
/video
```

主要问题：

1. `/generate` 像普通 AI 生成表单，缺少学习上下文。
2. 资源类型入口平铺，用户先看到“生成什么格式”，而不是“为什么生成”。
3. `/resources` 只有资源清单，查看按钮没有形成清晰预览。
4. `/feed` 是独立页面，和首页、错题、路径节点的关系不明显。
5. `/video` 像独立视频学习模块，削弱“视频是多模态资源类型”的定位。
6. 生成完成后的下一步动作不足，闭环断在“下载/打开链接”。

## 5. 页面职责重新定义

### 5.1 学习工作台 `/`

首页工作台承接资源推送。

建议模块：

```text
今日学习任务
当前路径节点
错题补弱提醒
今日推荐资源
最近学习反馈
```

首页中的“今日推荐资源”只展示 3 到 4 条，避免首页变成资源列表页。

每条推荐资源展示：

```text
资源名称
资源类型
推荐理由
推荐来源：路径节点 / 错题补弱 / 报告调整 / 今日复习
操作：查看 / 生成同类 / 开始学习
```

完整推荐列表通过以下入口进入：

```text
查看全部推荐 -> /feed
```

### 5.2 资源生成 `/generate`

资源生成页只负责一次资源生成 SOP。

主流程：

```text
第 1 步：确认学习上下文
第 2 步：选择资源类型
第 3 步：配置生成需求
第 4 步：生成与预览
```

生成完成后提供闭环动作：

```text
保存到资源库
开始学习
生成配套练习
让 AI 精灵讲解
下载
```

如果后端生成接口已经自动写入 `resources_index.json`，前端应显示：

```text
已自动保存到资源库
```

而不是让用户误以为资源仍是临时内容。

### 5.3 资源库 `/resources`

资源库是独立页面，负责管理已保存资源。

必须支持：

```text
搜索
资源类型筛选
来源筛选
状态筛选
点击预览
下载
删除
继续学习
生成配套练习
```

资源库中的资源来源建议使用以下枚举：

```text
路径节点生成
错题补弱生成
首页推荐保存
报告调整建议
手动生成
```

### 5.4 推荐资源详情 `/feed`

`/feed` 保留为推荐资源详情页，但不进入一级导航。

进入方式：

```text
首页今日推荐资源 -> 查看全部推荐
学习路径节点 -> 查看节点推荐资源
练习评估页 -> 查看补弱推荐
学习报告页 -> 查看后续推荐
```

页面职责：

```text
集中展示推荐资源
展示推荐理由
展示推荐来源
支持查看、生成同类、开始学习
```

### 5.5 视频预览 `/video`

`/video` 保留为视频资源预览页，但不进入一级导航。

进入方式：

```text
资源库中的视频/动画资源
资源生成完成后的多模态预览
推荐资源中的视频资源
```

页面定位：

```text
多模态资源预览页
```

不是独立视频学习平台。

## 6. `/generate` SOP 页面设计

### 6.1 顶部区域

顶部展示：

```text
资源生成 SOP 标题
一句话说明
当前闭环状态
当前知识点摘要
掌握度
错题数量
视频/动画已纳入资源类型
```

闭环状态建议展示为：

```text
学习画像 -> 路径节点 -> 资源生成 -> 练习评估 -> 报告回流
```

当前激活：

```text
路径节点
资源生成
```

### 6.2 左侧流程导航

左侧固定 SOP 步骤：

```text
1. 确认学习上下文
2. 选择资源类型
3. 配置生成需求
4. 生成与预览
```

每一步需要显示：

```text
步骤状态：已完成 / 当前 / 待操作
步骤标题
简短说明
```

### 6.3 第 1 步：确认学习上下文

展示内容：

```text
当前知识点
当前学习阶段
掌握度
相关错题数量
建议学习时长
薄弱来源
推荐生成目标
```

操作：

```text
使用当前上下文
切换知识点
手动生成
```

目标：让用户知道本次资源生成不是凭空生成，而是服务具体学习问题。

### 6.4 第 2 步：选择资源类型

资源类型按学习目的分组：

```text
讲解理解
├─ 课程文档
├─ PPT
├─ 思维导图
└─ 视频/动画

练习巩固
├─ 练习题
└─ 代码案例

拓展提升
└─ 拓展阅读
```

`视频/动画` 在当前阶段定义为：

```text
视频脚本 / 分镜 / 字幕 / 动画讲解方案
```

不得在后端未支持真实视频合成前写成“已支持 MP4 自动生成”。

### 6.5 第 3 步：配置生成需求

主要字段：

```text
生成目标
难度
学习时长
生成风格
```

快捷模板：

```text
生成补弱讲义
生成配套练习
生成视频讲解脚本
让精灵优化需求
```

高级设置默认收起，可包含：

```text
是否基于当前错题
是否基于当前画像
是否基于当前路径节点
是否生成配套练习
是否启用内容审核
```

### 6.6 第 4 步：生成与预览

生成过程展示：

```text
画像分析
路径对齐
资源生成
审核入库
```

每一步显示：

```text
状态
进度
简短说明
错误 / 降级提示
```

预览区域按资源类型渲染：

| 资源类型 | 预览方式 |
|---|---|
| 课程文档 | Markdown / HTML 预览 |
| PPT | iframe / PPT schema 预览 |
| 思维导图 | 图片或结构化导图预览 |
| 练习题 | 题目列表、答案解析折叠 |
| 拓展阅读 | Markdown / HTML 预览 |
| 代码案例 | 代码块 + 讲解 |
| 视频/动画 | 脚本、分镜、字幕、动画说明 |

生成完成后的动作：

```text
保存到资源库
开始学习
生成配套练习
让 AI 精灵讲解
下载
```

## 7. `/resources` 资源库页面设计

### 7.1 页面结构

建议采用：

```text
顶部：资源库标题 + 搜索
左侧/顶部：筛选条件
中间：资源列表
右侧：预览抽屉
```

资源库不建议只做普通表格。可以使用：

```text
列表 + 右侧预览抽屉
```

这样更适合学习资源查看。

### 7.2 资源列表字段

每条资源展示：

```text
资源名称
资源类型
状态
关联知识点
来源
创建时间
最近使用
```

状态：

```text
generating
completed
failed
```

可扩展：

```text
review_failed
fallback
```

### 7.3 预览抽屉

点击资源卡片或查看按钮后，打开右侧预览抽屉。

抽屉展示：

```text
资源标题
资源类型
来源
关联知识点
创建时间
预览内容
```

操作：

```text
下载
继续学习
生成配套练习
让 AI 精灵讲解
删除
```

### 7.4 空状态

资源库为空时：

```text
暂无已保存资源
```

操作：

```text
去生成资源 -> /generate
查看今日推荐 -> /feed
```

## 8. 首页资源推送设计

### 8.1 首页推荐资源模块

模块名称建议：

```text
今日推荐资源
```

或：

```text
基于画像的资源推荐
```

展示 3 到 4 条资源。

每条资源展示：

```text
资源名称
资源类型
推荐理由
推荐来源
关联知识点
```

操作：

```text
查看
生成同类
开始学习
```

### 8.2 推荐来源

推荐来源必须显式展示：

```text
路径节点
错题补弱
报告调整
今日复习
画像偏好
```

原因：资源推送的价值在于解释“为什么推荐”，而不是只展示资源卡片。

## 9. 视频/动画资源设计

### 9.1 当前阶段定义

当前阶段 `视频/动画` 作为资源类型纳入 `/generate`。

能力定义：

```text
视频脚本
分镜结构
字幕稿
动画讲解方案
```

### 9.2 后续升级方向

后续可升级为：

```text
HTML 动画预览
TTS 音频
字幕轨道
MP4 合成
```

在后端真实支持前，前端不得写成：

```text
一键生成真实视频
```

## 10. 数据与接口建议

### 10.1 当前可复用接口

当前可先复用：

```text
GET /api/resources
POST /api/resources/generate
GET /api/resources/recommendations/list
GET /api/resources/{resource_id}/preview
GET /api/resources/{resource_id}/preview/html
GET /api/resources/{resource_id}/download
DELETE /api/resources/{resource_id}
```

### 10.2 后续建议补充字段

资源对象建议逐步补充：

```json
{
  "knowledge_point_id": "function_params",
  "knowledge_point_name": "Python 函数参数与返回值",
  "source_type": "wrong_question_remedial",
  "source_label": "错题补弱生成",
  "recommendation_reason": "最近 3 道错题都涉及默认参数和返回值",
  "last_used_at": "2026-05-07T10:00:00+08:00",
  "saved": true
}
```

前端消费字段继续转换为 camelCase。

### 10.3 保存逻辑

当前若生成接口已经写入资源索引，则页面应展示：

```text
已自动保存到资源库
```

如果后续改成手动保存，则建议新增：

```text
POST /api/resources/{resource_id}/save
```

## 11. 执行顺序

### 11.1 第一阶段：页面职责调整

1. `/generate` 改成 SOP 流程页。
2. `/resources` 改成资源库 + 预览抽屉。
3. 首页增加推荐资源模块。
4. `/feed` 从一级导航下沉为推荐详情页。
5. `/video` 从一级导航下沉为多模态预览页。

### 11.2 第二阶段：数据闭环增强

1. 推荐资源显示推荐来源和推荐理由。
2. 资源库资源显示关联知识点和来源。
3. 练习错题触发补弱资源推荐。
4. 学习报告输出后续推荐调整。

### 11.3 第三阶段：多模态增强

1. 视频/动画脚本生成。
2. HTML 动画预览。
3. TTS 和字幕。
4. MP4 合成。

## 12. 验收标准

### 12.1 `/generate`

1. 用户按 SOP 完成资源生成。
2. 页面展示当前生成上下文。
3. 资源类型包含 `视频/动画`。
4. 生成完成后可以保存、学习、练习、精灵讲解或下载。
5. 页面不再展示完整资源库。

### 12.2 `/resources`

1. 资源库能搜索和筛选。
2. 点击资源能预览。
3. 预览支持文档、PPT、思维导图、题库、代码、视频/动画脚本等类型。
4. 资源能下载、删除、继续学习和生成配套练习。

### 12.3 首页

1. 首页展示资源推送模块。
2. 推荐资源有推荐理由和来源。
3. 可进入推荐详情页。

### 12.4 层级

1. `资源推送` 不出现在一级导航。
2. `资源中心/资源库` 不作为主链路一级功能。
3. `视频播放` 不作为一级导航。
4. 视频/动画作为资源生成类型出现。
