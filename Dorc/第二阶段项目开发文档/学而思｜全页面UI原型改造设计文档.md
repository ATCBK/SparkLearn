# 学而思｜全页面 UI 原型改造设计文档

## 文档入口说明

本文档用于对照当前 UI 原型、既有产品功能设计和第二阶段学习闭环要求，梳理 SparkLearn 后续全页面改造方向。

当前状态：

1. 登录页已经单独形成 HTML 原型：`ui-prototypes/phase2-complete-ui/auth-login-prototype.html`。
2. 登录页视觉方向已确认：浅蓝白、纯净背景图、玻璃态卡片、品牌口径为“学而思 SparkLearn”。
3. 主学习闭环原型仍在：`ui-prototypes/phase2-complete-ui/index.html`。
4. 主原型页面功能基本覆盖，但多数页面仍偏“功能说明型原型”，尚未完全改造成当前确认的视觉风格和真实用户任务流。
5. 本文档只定义 UI 原型与产品交互改造，不直接规定后端接口字段和 Next.js 接入实现。

当前不改动范围：

1. 暂不接入 `frontend` 目录。
2. 暂不重构后端接口。
3. 暂不把静态演示包装成真实动态能力。

---

## 1. 当前对照结论

### 1.1 已经对齐的内容

当前原型已经完成以下关键方向：

1. 一级导航已收敛为学习工作台、学习画像、学习路径、资源工作台、资源库、练习评测、学习报告。
2. 资源推送、视频播放、智能辅导不再作为一级导航。
3. AI 辅导已作为全局悬浮助手出现。
4. 学习画像页已采用“中心头像 + 背景图 + 周围注释卡片”的方向。
5. 资源工作台已经从“空白 Prompt”向“带学习上下文的 SOP 生成流程”转变。
6. 练习评测页面已经表达错题、掌握度回写和补弱推荐。
7. 学习报告页面已经有下一步行动建议。

### 1.2 主要未对齐问题

当前原型最大问题不是页面缺失，而是“页面之间的因果关系还不够强”。

具体表现：

1. 登录页和主学习闭环没有明确连接关系。
2. 学习工作台仍像信息面板，不像“今天怎么学”的决策页面。
3. 学习画像页展示方向已对，但画像编辑、画像更新记录、对话建档入口不足。
4. 学习路径页有节点动作，但路径调整、节点达标条件、资源与练习回流关系不完整。
5. 资源工作台有 SOP 外观，但生成过程、保存、预览、开始学习之间的状态链不完整。
6. 资源库有预览和继续学习，但缺少搜索、筛选、学习进度、资源来源与关联知识点。
7. 练习评测表达了错题回流，但缺少完整作答后的题目解析、错题本、收藏题和补弱资源推荐区。
8. 学习报告有总结与行动按钮，但缺少阶段对比、资源使用情况、薄弱点证据和报告后回流动作。
9. 全局 AI 精灵目前是静态提示，缺少页面上下文、问题建议、深度会话入口和结果回写表达。
10. 当前主原型视觉风格与最新登录页风格不统一，卡片偏多、尺寸偏大、信息密度不稳定。

## 3. 目标学习闭环

后续所有页面应共同服务以下闭环：

```text
登录 / 注册
-> 画像建档或画像确认
-> 学习工作台生成今日任务
-> 学习路径确认当前节点
-> 推荐资源或生成资源
-> 资源库继续学习
-> 练习评测
-> 错题沉淀与掌握度回写
-> 学习报告
-> 更新画像、调整路径、推荐下一轮任务
```

页面之间必须能解释清楚：

1. 当前信息从哪里来。
2. 用户点击后去哪里。
3. 操作结果会保存到哪里。
4. 结果如何影响下一页或下一轮学习。

---

## 4. 视觉改造总原则

后续全页面应继承登录页已确认风格：

1. 品牌统一为“学而思 SparkLearn”。
2. 主色保持蓝白浅色系。
3. 背景和插图用于营造轻盈科技感，不做营销式大英雄页。
4. 卡片尺寸要克制，避免一屏堆满大卡。
5. 信息分层采用“主判断 + 关键证据 + 下一步动作”。
6. 每页首屏只突出一个核心任务。
7. 所有学习页面要像学生看得懂的学习工具，不像后台管理面板。
8. 按登录页比例控制组件尺寸，确保常见桌面一屏能完整看到主内容。

## 5. 页面逐项改造设计

### 5.1 登录页

当前状态：

1. 已使用 `auth-home-bg.png` 作为背景。
2. 品牌为“学而思 SparkLearn”。
3. 右侧登录卡片保留登录/注册、邮箱、密码、记住我、忘记密码、协议。
4. 已删除手机号登录、微信登录、机构账号。

需要继续确认：

1. 登录卡片最终位置和比例。
2. 左侧三张功能卡是否保留，还是进一步简化。
3. 登录成功后进入哪里。

建议改造：

1. 登录页首屏只承担入口职责，不再展示完整产品闭环。
2. 左侧宣传保留两行主张和最多三张轻量能力卡。
3. 登录后默认进入学习工作台。
4. 如果是新用户或画像缺失，应进入画像建档/画像确认。

闭环关系：

```text
登录成功
-> 判断是否已有画像
-> 有画像：进入学习工作台
-> 无画像：进入画像建档/画像确认
```

### 5.2 学习工作台

当前状态：

1. 有当前节点、今日建议、阶段进度。
2. 有四个统计卡：累计学习、任务完成率、练习正确率、连续学习。
3. 有今日任务和个性化资源推荐。

问题：

1. 首屏信息像统计面板，不像决策入口。
2. “今天先做什么”不够强。
3. 资源推荐和今日任务分离，学生还要自己判断优先级。
4. 缺最近学习。
5. 缺“为什么今天这么安排”的解释。

改造目标：

学习工作台要回答：

```text
我今天先做什么？
为什么要这么做？
做完会影响什么？
```

建议结构：

1. 顶部状态区：今天学习判断。
2. 主任务区：按顺序展示 1-3 个今日任务。
3. 原因区：来自错题、路径进度、画像变化的依据。
4. 最近学习：上次资源或练习继续入口。
5. 轻量统计条：不再做四个大卡。

关键交互：

```text
点击今日任务资源
-> 进入资源库预览或资源学习
-> 学完后进入练习
-> 练习结果回写画像和路径
```

### 5.3 学习画像

当前状态：

1. 已采用中心头像和周围注释卡片。
2. 展示已看资源、学习目标、薄弱点、已掌握、学习偏好、今天建议。
3. 有“和 AI 更新画像”和“生成补弱资源”按钮。

问题：

1. 展示已较好，但编辑能力不足。
2. 画像来源证据不足，学生不知道系统为什么这么判断。
3. 缺画像更新记录。
4. 缺对话式建档入口。
5. 页面代码中存在部分旧 JS 引用，但页面元素不完整。

改造目标：

学习画像页要回答：

```text
我现在是什么状态？
系统为什么这么判断？
我可以怎么修正画像？
画像会如何影响路径、资源和练习？
```

建议结构：

1. 中心画像图继续保留。
2. 周围注释卡增加“来源”信息，例如练习、资源、AI 对话。
3. 右侧或底部增加画像更新记录。
4. 增加“对话更新画像”展开面板。
5. 增加“确认画像后重新规划路径”的动作。

闭环关系：

```text
资源学习 / 练习结果 / AI 对话
-> 更新画像字段
-> 影响学习路径、资源推荐、练习难度
```

### 5.4 学习路径

当前状态：

1. 有路径节点图。
2. 有当前节点：函数与作用域。
3. 提供查看推荐资源、生成资源、进入练习、AI 解释节点。

问题：

1. 节点为什么当前、为什么下一步，不够清楚。
2. 缺每个节点的达标条件。
3. 缺路径调整记录。
4. 缺从报告/错题回流导致路径变化的表达。

改造目标：

学习路径页要回答：

```text
我按什么顺序学？
当前卡在哪里？
什么时候能进入下一节点？
每个节点可以做什么？
```

建议结构：

1. 左侧路径图保持轻量。
2. 右侧当前节点详情增加：
   - 掌握度
   - 薄弱来源
   - 达标条件
   - 推荐动作
3. 增加路径调整提示，例如“因错题回流，模块导入延后”。
4. 每个节点动作统一为：看资源、生成资源、做练习、问 AI。

闭环关系：

```text
练习掌握度变化
-> 路径节点状态变化
-> 工作台今日任务变化
```

### 5.5 资源工作台

当前状态：

1. 已有 SOP 步骤条。
2. 已有学习上下文标签。
3. 已有资源类型选择。
4. 已有生成要求和“开始生成”按钮。

问题：

1. 步骤条是静态展示，没有具体生成过程。
2. 缺生成中状态。
3. 缺生成结果预览。
4. 缺保存到资源库。
5. 缺开始学习入口的完成态。

改造目标：

资源工作台要回答：

```text
为什么要生成这个资源？
生成什么类型？
生成过程是否可信？
生成后保存在哪里？
学完后做什么？
```

建议结构：

1. 保留上下文确认区。
2. 资源类型改为更紧凑的图标卡。
3. 生成配置保留 AI 改写建议。
4. 增加生成过程区：
   - 分析画像
   - 读取路径节点
   - 生成大纲
   - 安全检查
   - 完成预览
5. 增加结果区：
   - 预览
   - 保存到资源库
   - 开始学习
   - 生成配套练习

闭环关系：

```text
路径节点 / 错题 / 画像
-> 生成资源
-> 保存资源库
-> 学习行为回写
-> 推荐练习
```

### 5.6 资源库

当前状态：

1. 有已保存资源列表。
2. 有资源预览。
3. 有继续学习、预览、下载、删除。

问题：

1. 缺搜索和筛选。
2. 缺资源学习进度。
3. 缺资源来源和关联知识点。
4. 资源预览太简略。
5. 删除、下载、继续学习没有状态区分。

改造目标：

资源库要回答：

```text
我有哪些资源？
哪些和当前薄弱点有关？
我学到哪里了？
下一步继续看还是去练习？
```

建议结构：

1. 顶部搜索和筛选。
2. 左侧资源列表按类型、来源、进度展示。
3. 右侧资源详情预览。
4. 每个资源标明：
   - 类型
   - 来源
   - 关联知识点
   - 学习进度
   - 推荐动作
5. 增加“学完去练习”和“让 AI 讲解”入口。

闭环关系：

```text
资源学习完成
-> 更新资源进度
-> 触发练习建议
-> 影响报告中的资源使用情况
```

### 5.7 练习评测

当前状态：

1. 有个性化题目。
2. 有提交判题和收藏按钮。
3. 有错题回流时间线。
4. 有一键生成补弱资源。

问题：

1. 缺提交后的正确/错误态。
2. 缺答案解析。
3. 缺错题本和收藏题列表入口。
4. 缺错因分析。
5. 缺补弱资源推荐区。

改造目标：

练习评测要回答：

```text
我做对了吗？
为什么错？
错题保存在哪里？
系统会怎么调整我的画像和路径？
我现在应该补什么？
```

建议结构：

1. 题目区。
2. 答案选择区。
3. 提交后解析区。
4. 错因分析区。
5. 错题回流区。
6. 补弱资源推荐区。
7. 进入下一题或生成补弱资源。

闭环关系：

```text
提交练习
-> 判题
-> 解析
-> 错题本
-> 掌握度回写
-> 推荐补弱资源
-> 更新工作台和画像
```

### 5.8 学习报告

当前状态：

1. 有学习统计。
2. 有行为柱状图。
3. 有 AI 总结和下一步按钮。

问题：

1. 缺资源使用情况。
2. 缺练习结果明细。
3. 缺薄弱点证据。
4. 缺阶段对比。
5. 缺报告触发路径和画像更新的表达。

改造目标：

学习报告要回答：

```text
这一阶段学得怎么样？
证据是什么？
哪里薄弱？
下一步做什么？
系统会如何调整我的路径？
```

建议结构：

1. 阶段总结。
2. 学习行为和资源使用。
3. 练习表现。
4. 薄弱点证据。
5. AI 总结。
6. 下一步行动。
7. 回流状态：画像已更新、路径待确认、资源已推荐。

闭环关系：

```text
报告生成
-> 总结薄弱点
-> 更新画像
-> 调整路径
-> 生成下一轮工作台任务
```

### 5.9 全局 AI 精灵助手

当前状态：

1. 有悬浮入口。
2. 能根据页面切换上下文文案。
3. 有快捷按钮。

问题：

1. 目前只是静态说明。
2. 缺真实对话态。
3. 缺页面上下文引用。
4. 缺把 AI 建议写回画像、路径、资源配置的表达。
5. 深度会话入口没有实际页面承接。

改造目标：

AI 精灵要回答：

```text
我现在能问什么？
AI 基于当前页面知道什么？
AI 的建议会影响页面内容吗？
```

建议结构：

1. 悬浮按钮保留。
2. 面板内显示当前页面上下文。
3. 快捷问题随页面变化。
4. 回答后提供可执行动作：
   - 更新画像
   - 调整路径
   - 优化生成要求
   - 讲解题目
   - 生成补弱资源
5. 深度会话作为二级入口，不作为一级导航。

## 6. 全页面改造优先级

### P0：必须先改

1. 学习工作台重做为“今日学习决策页”。
2. 学习画像补齐编辑、对话建档、更新记录。
3. 资源工作台补齐生成过程、预览、保存、开始学习。
4. 练习评测补齐判题后解析和错题本入口。

### P1：第二批改

1. 学习路径补齐达标条件和路径调整记录。

## 7. 建议的页面改造顺序

建议不要一次重做所有页面，按闭环顺序推进：

1. 登录页最终定稿。
2. 学习工作台重做。
3. 学习画像补齐建档和更新。
4. 学习路径补齐节点状态。
5. 资源工作台补齐生成闭环。
6. 资源库补齐学习入口。
7. 练习评测补齐判题与错题。
8. 学习报告补齐评估与回流。
9. 全局 AI 精灵统一增强。

## 8. 验收清单

每个页面改造完成后，至少检查：

1. 是否符合“学而思 SparkLearn”品牌口径。
2. 是否延续当前登录页的蓝白通透风格。
3. 是否一屏内有明确主任务。
4. 是否减少无意义卡片堆叠。
5. 是否能说明页面数据来源。
6. 是否能说明用户操作后的下一步。
7. 是否能说明操作结果如何影响画像、路径、资源、练习或报告。
8. 是否没有把资源推送、视频播放、智能辅导重新做成一级入口。
9. 是否没有把静态演示包装成真实动态闭环。
10. 是否在 HTML 原型确认前不改 `frontend` 目录。

## 9. 当前最关键的设计判断

当前原型已经具备页面框架，但还不够像一个真实学习产品。

下一步最重要的不是继续加页面，而是把页面之间的因果关系做清楚：

```text
为什么推荐这个任务？
为什么我薄弱？
为什么路径这么安排？
我学完之后系统怎么变化？
我做错之后系统怎么帮我补？
报告出来后下一轮怎么开始？
```

只要这些问题在 UI 上可见，学习闭环才算真正成立。
