# 学而思｜登录页改造设计

## 文档入口说明

本文档用于说明 SparkLearn 第二阶段中“登录 / 注册页”的改造目标、页面效果、交互状态和验收边界。

当前状态：本文档为设计草案，尚未进入代码实现。

适用页面：

```text
frontend/src/app/auth/page.tsx
```

参考 UI 方案：

```text
ui-prototypes/phase2-complete-ui/index.html
frontend/public/assets/generated/backgrounds/auth-hero-brand-portal-v1.png
```

本页改造不涉及后端鉴权、真实账号体系、数据库、Token、权限管理。当前产品仍保持点击登录 / 注册后进入 `onboarding` 的演示流程，后续若接入真实鉴权需另写接口与状态设计。

---

## 1. 当前页面现状

当前登录页为左右分栏结构：

左侧是深色品牌展示区，包含 SparkLearn 标识、口号和三组数据。

右侧是登录 / 注册表单，包含邮箱、密码、确认密码、记住我、忘记密码和提交按钮。

当前点击登录或注册后直接跳转到：

```text
/onboarding
```

当前页面优点：

1. 登录和注册入口清楚。
2. 表单字段数量少，用户不会有负担。
3. 密码可见性切换可用。
4. 测试用例已经覆盖基础显示、Tab 切换和跳转。

当前主要问题：

1. 左侧深色视觉与第二阶段主 UI 的浅色 Apple 风格割裂。
2. 页面只像普通账号入口，没有体现“进入学习闭环”的产品定位。
3. 口号“AI 驱动的个性化学习平台”较泛，不能告诉学生登录后要做什么。
4. “10w+ 学习用户、95% 提分率、50+ 课程覆盖”偏营销，且当前演示产品没有真实数据支撑。
5. 登录成功后进入建档流程，但页面没有提前告诉用户“下一步会创建学习画像”。
6. 忘记密码按钮目前没有反馈，真实用户会误以为功能损坏。

---

## 2. 改造目标

登录页不再只承担“输入账号”的职责，而是成为学习闭环的第一入口。

普通用户进入后应立即理解三件事：

1. 这是 SparkLearn。
2. 登录后会进入个性化学习建档 / 学习画像。
3. 系统会围绕画像、路径、资源、练习和报告帮助自己学习。

改造后的感受应与第二阶段 UI 原型一致：

1. 浅色背景。
2. 白色卡片。
3. 蓝色主色。
4. 柔和阴影。
5. 清爽、低压力。
6. 不使用强营销、强压迫或深色科技感。

本页需要明确具备“有背景、有层次、有图片化美化元素”的首页观感：

1. 页面底层使用完整背景图或大面积主视觉背景。
2. 页面内部不能只有表单和文字，还需要有可见的图片化视觉组件。
3. 图片化视觉组件要服务学习闭环，不做无意义装饰。
4. 视觉层次应包含背景、半透明浮层、白色登录卡片、学习状态预览卡片。
5. 整体仍保持清爽，不做营销官网式大字海报。

## 3. 页面定位

页面名称：

```text
登录 / 注册
```

用户任务：

```text
进入 SparkLearn，并开始建立自己的学习画像
```

在学习闭环中的位置：

```text
登录注册 -> 学习画像建档 -> 学习路径 -> 资源学习 -> 练习评测 -> 学习报告 -> 画像更新
```

页面不承载：

1. 真实学习数据展示。
2. 完整产品介绍。
3. 营销官网首屏。
4. 资源推送、视频播放、AI 辅导入口。

## 4. 改造后页面结构

### 4.1 整体布局

建议保留左右分栏，但视觉从“深色品牌区 + 白色表单区”改为“浅色沉浸背景 + 白色登录卡片”。

桌面端结构：

```text
页面背景
├─ 背景主视觉层
│  ├─ auth-hero-brand-portal-v1.png
│  ├─ 柔和蓝白光感
│  └─ 轻量模糊与遮罩，保证文字可读
│
├─ 左侧品牌引导区
│  ├─ SparkLearn 标识
│  ├─ 主标题：从学习画像开始，进入你的个性化学习路径
│  ├─ 说明文案：登录后先确认目标、基础、薄弱点和学习偏好
│  ├─ 三个闭环提示
│  │  ├─ 画像建档
│  │  ├─ 路径规划
│  │  └─ 练习回流
│  └─ 图片化学习状态预览区
│     ├─ 学习画像小卡
│     ├─ 路径节点小卡
│     └─ 今日建议小卡
│
└─ 右侧登录卡片
   ├─ 登录 / 注册 Tab
   ├─ 邮箱
   ├─ 密码
   ├─ 确认密码（注册时）
   ├─ 记住我
   ├─ 忘记密码
   ├─ 主按钮
   └─ 下一步提示
```

### 4.2 左侧品牌引导区

建议使用现有图片资产：

```text
frontend/public/assets/generated/backgrounds/auth-hero-brand-portal-v1.png
```

图片使用方式：

1. 作为左侧或全页背景的柔和视觉资产。
2. 右侧保留大面积留白，避免遮挡表单。
3. 不做深色遮罩。
4. 保持 SparkLearn 第二阶段浅色、蓝色、通透风格。

背景层建议：

```text
全屏背景：auth-hero-brand-portal-v1.png
位置：左侧 35% 到中间区域
尺寸：cover
叠加：白色半透明柔光层，避免视觉过重
```

背景不应只作为“漂亮图片”，而应表达“进入学习空间”的感觉：门、路径、书本、学习卡片等元素与登录后的建档流程呼应。

推荐文案：

```text
SparkLearn
从学习画像开始，进入你的个性化学习路径
```

辅助文案：

```text
登录后先确认你的目标、基础、薄弱点和学习偏好，系统会据此生成路径、推荐资源，并在练习后持续更新画像。
```

闭环提示卡片：

```text
画像建档：先了解你的目标和薄弱点
路径规划：按当前阶段安排学习顺序
练习回流：错题会影响后续推荐
```

避免文案：

```text
10w+ 学习用户
95% 提分率
50+ 课程覆盖
```

原因：这些数字如果没有真实来源，普通用户会觉得像营销页，也不服务当前第二阶段学习闭环。

### 4.3 图片化美化组件

除背景图外，页面内部需要增加 2 到 3 个图片化或半拟物的学习预览组件。它们不是可点击主功能，而是帮助用户理解登录后会发生什么。

建议组件一：学习画像预览卡

```text
标题：学习画像
状态：项目实践准备期
内容：函数返回值、变量作用域、模块导入
视觉：小头像 / 圆形轨道 / 蓝绿色标签
```

建议组件二：今日学习建议卡

```text
标题：今天建议
内容：先补齐“局部变量与返回值”
动作提示：登录后继续
视觉：白色浮层卡片 + 蓝色进度条
```

建议组件三：路径节点卡

```text
标题：当前路径
节点：Python 函数与作用域
进度：72%
视觉：三个节点连线，当前节点高亮
```

这些组件的设计原则：

1. 卡片要像产品真实片段，而不是宣传插画。
2. 内容要与第二阶段原型中的学习画像、工作台、路径页一致。
3. 不展示过多假数据，最多保留少量用于表达流程的演示数据。
4. 卡片应有轻微错层、玻璃感或柔和阴影，形成空间层次。
5. 不使用纯装饰 SVG 图案堆叠，不使用无意义渐变圆球。

### 4.4 右侧登录卡片

卡片标题：

```text
欢迎回来
```

登录态说明：

```text
继续你的学习画像和今日学习任务
```

注册态说明：

```text
创建账号后，我们会先帮你建立学习画像
```

Tab：

```text
登录
注册
```

字段：

登录：

```text
邮箱
密码
```

注册：

```text
邮箱
密码
确认密码
```

按钮：

```text
登录并继续
创建账号并建档
```

表单底部提示：

登录态：

```text
下一步：进入学习画像建档或今日学习工作台
```

注册态：

```text
下一步：用 3 分钟完成学习画像
```

## 5. 交互设计

### 5.1 登录 / 注册切换

用户点击 Tab 后：

1. 表单内容切换。
2. 卡片标题和说明文案同步切换。
3. 主按钮文案同步切换。
4. 不清空已输入的邮箱，减少重复输入。

### 5.2 密码显示与隐藏

保持现有眼睛图标交互。

要求：

1. 图标按钮要有可访问名称。
2. 点击后仅切换当前密码框。
3. 注册页的密码和确认密码分别控制。

### 5.3 忘记密码

当前没有真实功能时，不能点击无反馈。

建议临时处理：

点击后展示轻提示：

```text
密码找回暂未接入，请先使用演示账号或重新注册。
```

后续真实接入时再进入找回密码流程。

### 5.4 提交

当前演示阶段：

1. 点击主按钮后出现短暂 loading。
2. loading 文案为“正在进入你的学习空间”。
3. 结束后跳转到 `/onboarding`。

未来真实鉴权阶段：

1. 邮箱格式校验。
2. 密码必填校验。
3. 注册时确认密码一致校验。
4. 错误提示出现在字段下方。
5. 服务端失败时展示可读错误。

## 6. 状态设计

### 6.1 默认状态

默认显示登录 Tab。

### 6.2 输入状态

输入框获得焦点时：

1. 边框变为蓝色。
2. 保持背景为白色。
3. 不出现强烈阴影。

### 6.3 错误状态

错误文案使用普通人能理解的表达：

```text
请输入邮箱
请输入正确的邮箱
请输入密码
两次输入的密码不一致
```

避免：

```text
Invalid credentials
Auth error
Request failed
```

### 6.4 Loading 状态

主按钮进入 loading，不允许重复点击。

按钮文案：

```text
正在进入...
```

### 6.5 空功能状态

忘记密码未接入时必须给提示，不允许无反应。

## 7. 与当前产品的改造差异

| 模块 | 当前产品 | 改造后 |
|---|---|---|
| 整体视觉 | 左深右浅，偏登录工具页 | 全页浅色 Apple 风格，贴近第二阶段原型 |
| 左侧内容 | 品牌 + 营销数字 | 品牌 + 学习闭环引导 |
| 主标题 | SparkLearn | 从学习画像开始，进入个性化学习路径 |
| 表单说明 | 无明确下一步 | 明确登录后进入建档 / 学习空间 |
| 忘记密码 | 点击无反馈 | 未接入时给轻提示 |
| 登录按钮 | 登录 | 登录并继续 |
| 注册按钮 | 注册 | 创建账号并建档 |
| 数据呈现 | 10w+、95%、50+ | 画像建档、路径规划、练习回流 |

## 8. 验收标准

页面改造后至少满足：

1. 默认进入 `/auth` 时显示登录表单。
2. 登录 / 注册 Tab 可切换。
3. 登录态只显示邮箱和密码。
4. 注册态显示邮箱、密码、确认密码。
5. 密码可显示 / 隐藏。
6. 忘记密码点击后有提示。
7. 点击登录或注册后有 loading 状态。
8. 演示阶段提交后进入 `/onboarding`。
9. 页面视觉与第二阶段 HTML 原型风格一致。
10. 页面文案能让普通用户理解：登录后会先建立或继续学习画像。
11. 不再使用无来源营销数字作为主要信任背书。
12. 不出现资源推送、视频播放、AI 辅导等一级功能入口。
