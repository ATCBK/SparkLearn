# SparkLearn 当前功能树梳理

> 基于当前前端页面与 API 调用整理，反映的是代码里已经存在或已经预留的功能，不等同于产品最终规划。

## 0. 全局框架

- 应用布局
  - 顶层布局：全局字体、图标、页面元信息
  - Shell 布局：左侧导航 + 右侧页面容器
  - 特殊全屏页面：登录页、画像建档页不走 Shell
- 左侧导航
  - 首页总览：`/`
  - 学习路径：`/path`
  - 资源中心：`/resources`
  - 练习与错题：`/practice`
  - 资源推送：`/feed`
  - 资源生成：`/generate`
  - 智能辅导：`/tutor`
  - 学习报告：`/report`
  - 视频播放：`/video`
  - 个人信息入口目前只在侧栏底部展示用户信息，没有导航到 `/profile`
- 侧栏交互
  - 展开模式：显示 Logo、导航文字、名言、用户信息
  - 图标模式：只显示图标，hover 显示提示
  - 收起模式：隐藏侧栏，页面左侧显示展开按钮

## 13. 当前功能混乱点初步归纳

- 入口重复
  - 资源中心负责管理资源
  - 资源生成也同时承担资源列表、预览、下载
  - 资源推送也展示资源，但没有后续操作
- 占位功能较多
  - 登录注册没有鉴权
  - 画像建档没有保存
  - 资源中心查看无动作
  - 视频播放无真实播放器
  - 报告语音播报无动作
  - 智能辅导语音输入无动作
  - 个人信息安全设置无动作
- 数据写入不一致
  - 首页任务勾选只改本地前端状态
  - 练习、资源、导师会话是真实后端写入
  - 个人信息部分写后端，部分写 localStorage
- 功能边界不清
  - 资源查看 / 预览 / 下载主要在资源生成页完成，资源中心反而只管理列表和删除
  - 学习路径的推荐资源类型没有直接跳到资源生成或资源中心
  - 资源推送只推荐，不支持“开始学习 / 查看 / 收藏 / 生成同类资源”
- 页面导航缺口
  - `/profile` 页面存在，但侧栏用户区域没有链接到该页面
  - 首页最近学习有点击样式，但没有实际导航

## 14. 建议后续重构方向

- 明确三类资源页面职责
  - 资源中心：统一查看、搜索、筛选、预览、下载、删除
  - 资源生成：只负责创建资源和展示生成进度
  - 资源推送：负责推荐与“采纳推荐”动作
- 明确学习闭环
  - 学习路径节点 → 推荐资源 → 练习题 → 错题本 → 学习报告
  - 每一步都应有明确跳转或数据写入
