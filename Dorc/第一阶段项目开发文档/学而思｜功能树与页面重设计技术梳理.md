# SparkLearn 功能树与页面重设计技术梳理

> 位置：项目文档目录 `Dorc/`  
> 梳理对象：`frontend/src/app` 页面、`frontend/src/lib/api` 前端 API 封装、`backend/app/routes` 后端能力  
> 目的：作为后续重新设计每个页面功能、组合关系和技术实现细节的基础文档。

## 1. 项目功能总览

SparkLearn 当前是一个面向 Python 学习的个性化学习平台。产品主线可以抽象为：

```text
认证/画像建立
  -> 学习首页总览
  -> 学习路径规划
  -> 资源生成/资源管理/资源推荐
  -> 练习测评/错题收藏
  -> 智能辅导/多智能体研讨
  -> 学习报告/个人资料维护
```

当前实现里有两类页面：

- 入口页面：`/auth`、`/onboarding`
- 主应用页面：`/`、`/path`、`/resources`、`/practice`、`/feed`、`/generate`、`/tutor`、`/report`、`/video`、`/profile`

主应用页面共用 `frontend/src/app/(shell)/layout.tsx` 和 `Sidebar`。侧边栏支持展开、图标模式、完全收起三种状态。

前端调用：

- `api.getResources()` -> `GET /api/resources`
- `api.generateResource(type, prompt)` -> `POST /api/resources/generate` SSE
- `api.downloadResource(id)` -> `GET /api/resources/{id}/download/pdf`，失败回退 `download`
- `api.downloadResourceSource(id)` -> `GET /api/resources/{id}/download/source`

后端逻辑：

- `ppt`：走讯飞智文客户端，创建 PPT、轮询进度、保存 `source_url`。
- 其他资源：走 Coze 适配器流式生成文本，可解析 `SOURCE_URL`。
- 生成完成后写入 `resources_index.json`，并追加 `resource_usage.jsonl`。
- HTML/PDF 预览和下载使用 Playwright 渲染。

当前状态：

- 前端 `generateResource` 只等待 SSE 完成事件，未把中间 progress/text 事件直接反映到页面。
- 重试逻辑是前端模拟，不会重新调用后端生成。

重设计建议：

- 将资源生成拆为“生成表单 + 任务队列 + 资源详情预览”三块。
- SSE 应暴露实时文本、进度、阶段、错误并写入状态机。
- 生成完成后应允许关联知识点、学习路径节点、推荐场景。

- 研讨会总结解析依赖文本标题，较脆弱。

重设计建议：

- 智能辅导应成为跨页面能力：资源页、练习页、路径节点都能“带上下文问导师”。
- 文件问答需要补充解析链路：上传 -> 文本抽取 -> 索引/摘要 -> 聊天引用。
- 研讨会结果建议后端直接返回结构化字段，而不是前端解析自然语言。

## 8. 后续设计需要确认的问题

1. 是否保留单用户 Demo 模式，还是补真实登录认证？
2. 学习路径是否只服务 Python，还是要支持多课程/多知识库？
3. 资源生成是否需要异步任务队列，还是 SSE 同步生成即可？
4. 资源是否必须绑定知识点？如果绑定，资源推荐和报告会更容易闭环。
5. 智能辅导上传文件是否需要文档解析和向量检索？
6. 练习是否需要题目难度、题型比例、知识点覆盖率配置？
7. 报告是否需要导出 PDF 或分享给教师端？
8. 管理端 API 已有占位，是否需要教师/管理员页面？
