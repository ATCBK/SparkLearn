## 1. 设计哲学与理念

### 1.1 核心理念：安静的智能

SparkLearn 的界面设计服务于一个核心信念——**学习工具本身不应该消耗学习者的认知资源**。

传统教育类产品常犯的错误是"信息过载"：仪表盘塞满数字、按钮、图表、通知，用户需要先学会使用工具才能学习。我们的设计哲学相反——界面应该**安静地退居幕后**，让用户的心智完全投入在学习内容上。

这个理念源自 Apple 的设计哲学："真正的简约不是没有杂乱可添加，而是没有多余可删减。"（Jonathan Ive）。我们追求的不是极简主义的冰冷感，而是一种**温暖的克制**——足够的信息引导，但绝不过量。

### 1.2 设计原则

| 原则 | 说明 | 设计表达 |
|------|------|----------|
| **内容优先** | 界面是学习内容的框架，不是主角 | 大面积留白、字体层级清晰、视觉干扰最少化 |
| **渐进披露** | 信息按需呈现，不一次全暴露 | 折叠面板、卡片式摘要、"查看全部"链接 |
| **情感连接** | 学习是孤独的，界面应有温度 | 每日名言、学习 streak 激励、个性化问候 |
| **认知减负** | 减少用户的决策成本 | 智能推荐、默认排序、预设筛选条件 |
| **一致性** | 相似的功能用相似的方式呈现 | 统一的卡片、标签、按钮系统 |

### 1.3 为什么选择 Apple Pure 设计语言

经过四版设计方案迭代（Notion 温暖风、学术风、Framer 创意风、Apple 纯净风），最终选定 **Apple Pure** 作为设计语言，原因：

1. **可读性最优**：深色文字 + 浅色背景 + 大量留白，长时间阅读不疲劳
2. **信息层级最清晰**：通过字号梯度（48px → 32px → 17px → 14px → 13px → 11px）和字重（700 → 600 → 500 → 400）建立严格的视觉层级
3. **最利于扩展**：中性色系统可以承载任何未来功能，不受特定色彩主题限制
4. **评委接受度高**：Apple 设计语言是全球公认的企业级设计标杆，传达专业感和品质感

## 2. 信息架构

### 2.1 全局导航结构

```
学而思 SparkLearn
├── 学习模块
│   ├── 01 首页学习总览      ← 入口页，全局概览
│   ├── 02 学习路径          ← 知识图谱 + 阶段规划
│   ├── 03 资源中心          ← 搜索 + 分类浏览
│   ├── 04 练习与错题        ← 习题训练 + 错题本
│   ├── 05 资源推送          ← AI 推荐 + 个性化内容
│   └── 06 资源生成与资源详情  ← AI 生成资源 + 在线预览 + 保存入库
├── 工具模块
│   ├── 07 智能辅导          ← AI 对话式辅导
│   ├── 08 学习报告          ← 数据分析 + 薄弱点
│   └── 09 视频播放          ← 课程视频 + AI 笔记
├── 个人模块
│   ├── 10 个人信息          ← 账户设置
│   └── 11 学习画像建档      ← 建档流程
└── 系统模块
    └── 12 登录/注册         ← 身份验证
```

### 2.2 信息层级模型

设计中严格遵循四层信息层级：

| 层级 | 名称 | 作用 | 典型元素 |
|------|------|------|----------|
| **L1** | 导航层 | 告诉用户"我在哪" | 侧栏导航、面包屑、页面标题 |
| **L2** | 概览层 | 告诉用户"整体状态" | Hero 标题、统计卡片、进度环 |
| **L3** | 内容层 | 用户实际操作 | 任务列表、资源列表、习题区域 |
| **L4** | 详情层 | 深入信息 | 弹窗、抽屉、展开面板 |

**设计规则**：任何页面同屏可见不超过 L1-L3。L4 通过交互触发，避免信息过载。

### 2.3 用户任务流

```
首次登录 → 学习画像建档 → 首页总览 → 选择学习路径
                                          ↓
                              开始学习 → 视频/讲义/练习
                                          ↓
                              完成任务 → 查看报告 → AI 推荐下一步
                                          ↓
                              循环 ↑ —— 每日任务驱动学习
```

核心设计目标：**减少从"想学习"到"开始学习"的点击次数**。从首页到开始学习内容，目标路径 ≤ 2 次点击。

## 3. 视觉设计系统

### 3.1 色彩系统

#### 3.1.1 主色板

| 用途 | 色值 | 名称 | 应用场景 |
|------|------|------|----------|
| 主文字 | `#1d1d1f` | Apple Black | 标题、正文 |
| 次要文字 | `#6e6e73` | Apple Gray 1 | 描述文字、副标题 |
| 三级文字 | `#86868b` | Apple Gray 2 | 辅助说明 |
| 四级文字 | `#aeaeb2` | Apple Gray 3 | 占位文字、禁用态 |
| 主操作蓝 | `#0071e3` | Apple Blue | 按钮、链接、激活态 |
| 背景 | `#fbfbfd` | Snow White | 页面背景 |
| 卡片背景 | `#ffffff` | Pure White | 卡片、面板 |
| 悬浮背景 | `#f5f5f7` | Light Gray | 悬浮态、选中态 |

#### 3.1.2 语义色板

| 语义 | 色值 | 使用场景 |
|------|------|----------|
| 成功 | `#34c759` | 正确答案、完成状态、正向指标 |
| 警告 | `#ff9500` | 需要注意、中等掌握度 |
| 危险 | `#ff3b30` | 错误、低掌握度、未完成 |
| 信息 | `#5ac8fa` | 高亮渐变终点、装饰性强调 |
| 特殊 | `#af52de` | AI 功能标识、智能提示 |

#### 3.1.3 色彩使用规则

- **面积比**：白色/浅灰 85%，主色 10%，强调色 5%
- **文字对比度**：主文字在白色背景上对比度 ≥ 12:1（WCAG AAA）
- **渐变使用**：仅用于 Hero 标题高亮和特色卡片装饰，不作为功能性颜色
- **深色模式**：当前版本暂不支持，预留 CSS 变量结构以便未来扩展

### 3.2 字体系统

#### 3.2.1 字体选型

| 字体 | 用途 | 选择理由 |
|------|------|----------|
| **DM Sans** | 英文标题与正文 | 几何感强、可读性高、现代感，避免 Inter/Roboto 等过度使用的字体 |
| **Noto Sans SC** | 中文标题与正文 | Google 出品、覆盖全、与 DM Sans 风格协调 |
| **Lucide Icons** | 图标系统 | 开源 SVG 图标库，风格统一、可定制、无 emoji 模糊问题 |

#### 3.2.2 字号梯度

| 级别 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Display | 48px | 700 | 页面 Hero 标题 |
| H1 | 32px | 700 | 卡片标题、特色卡片标题 |
| H2 | 21px | 400 | Hero 副标题 |
| H3 | 17px | 700 | 卡片内标题 |
| Body | 15px | 500 | 任务标题、列表项 |
| Small | 14px | 500 | 导航项、标签 |
| Caption | 13px | 400-600 | 元数据、辅助文字 |
| Micro | 11px | 400-600 | 徽标、极小标注 |

**行高**：统一 1.5（正文）/ 1.1（Display 标题）/ 1.4（副标题）

### 3.3 间距系统

基于 4px 基数的间距系统：

| Token | 值 | 应用 |
|------|------|------|
| `--space-xs` | 4px | 图标与文字间距 |
| `--space-sm` | 8px | 标签内间距、紧凑元素间距 |
| `--space-md` | 12px | 列表项间距 |
| `--space-lg` | 16px | 卡片间距、网格间距 |
| `--space-xl` | 20px | 卡片内边距（水平） |
| `--space-2xl` | 24px | 卡片内边距（垂直）、区段间距 |
| `--space-3xl` | 32px | 大卡片内边距、主要区段分隔 |
| `--space-4xl` | 40px | 页面主内容区 padding |
| `--space-5xl` | 48px | Hero 区域 padding、特色卡片内边距 |

### 3.4 圆角系统

| Token | 值 | 应用 |
|------|------|------|
| `--radius-sm` | 12px | 按钮、小卡片、图标容器 |
| `--radius-md` | 16px | 中等卡片、输入框 |
| `--radius-lg` | 20px | 内容卡片 |
| `--radius-xl` | 28px | 特色卡片（Featured Card） |
| `--radius-pill` | 100px | 标签、阶段徽章、筛选按钮 |

### 3.5 阴影系统

| Token | 值 | 应用 |
|------|------|------|
| `--shadow-sm` | `0 2px 8px rgba(0,0,0,0.04)` + `0 0 0 1px rgba(0,0,0,0.02)` | 默认卡片 |
| `--shadow-md` | `0 4px 20px rgba(0,0,0,0.06)` + `0 0 0 1px rgba(0,0,0,0.02)` | 悬浮态 |
| `--shadow-lg` | `0 8px 40px rgba(0,0,0,0.08)` | 弹窗 |
| `--shadow-xl` | `0 20px 60px rgba(0,0,0,0.1)` | 特色卡片 |

**阴影设计逻辑**：所有阴影使用黑色透明度而非彩色，保持 Apple 式的"无色感"。阴影层次对应卡片的"海拔"，海拔越高（如特色卡片），阴影越深。

└──────────────────────────────────────────────────────────┘

### 4.2 侧栏设计

**固定定位 + 毛玻璃效果**：侧栏使用 `position: fixed` 固定在左侧，配合 `backdrop-filter: blur(60px) saturate(200%)` 实现半透明毛玻璃效果。

**设计意图**：
- 毛玻璃保持上下文感知——滚动时用户仍能看到主内容区的色彩透出
- 宽度 260px 兼顾导航可读性和内容空间
- 底部名言区增加情感温度，避免工具化的冰冷感

### 4.3 内容区栅格

**主内容区**：`max-width: 1200px`，居左对齐（非居中），留出右侧空间避免大屏过宽。

**双列布局**：`grid-template-columns: 1fr 360px`

| 左列 | 右列 |
|------|------|
| 弹性宽度，占据剩余空间 | 固定 360px，信息聚合区 |
| 承载主要操作内容（任务、资源） | 承载辅助信息（阶段、画像、掌握度） |
| 信息密度较高 | 信息密度适中，阅读优先 |

**设计意图**：右列固定宽度确保辅助信息在任何屏幕宽度下保持一致的阅读体验。

## 5. 交互设计模式

### 5.1 卡片交互

| 状态 | 阴影 | 变换 | 说明 |
|------|------|------|------|
| 默认 | `--shadow-sm` | none | 安静地存在 |
| Hover | `--shadow-md` | `translateY(-3px)` | 微妙浮起，暗示可交互 |
| Active | `--shadow-sm` | `translateY(0)` | 按下回弹 |

**设计理念**：卡片的"浮起"效果模拟物理世界的纸张抬起，利用空间隐喻（Z轴移动）传达交互可能性，而非依赖颜色变化或边框。

### 5.2 动画系统

#### 5.2.1 页面加载动画

使用 `fadeInUp` + `scaleIn` 交错延迟：

```css
.delay-1 { animation-delay: 0.05s; }
.delay-2 { animation-delay: 0.1s; }
.delay-3 { animation-delay: 0.15s; }
/* ... 最多 delay-8 */
```

**设计意图**：内容从下往上逐个淡入，模拟"页面从屏幕底部生长"的感觉。每个元素的延迟差 50ms，足够感知到顺序但不会觉得慢。

#### 5.2.2 微交互

| 交互 | 动画 | 时长 | 缓动函数 |
|------|------|------|----------|
| 页面加载 | fadeInUp + scaleIn | 500ms | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 卡片悬浮 | translateY + shadow | 250ms | ease |
| 进度条填充 | width transition | 800ms | ease |
| 导航切换 | background + color | 200ms | ease |
| 脉冲指示器 | scale + opacity | 2000ms | infinite |

**缓动函数选择**：统一使用 `ease` 或 Apple 标准曲线 `cubic-bezier(0.16, 1, 0.3, 1)`，避免线性动画的生硬感。

### 5.3 导航模式

**侧栏导航**：
- 点击高亮项显示蓝色指示条（左侧 3px 竖线）
- 选中态使用 `--blue-light` 背景色，保持轻量
- 徽标使用圆角胶囊形（`border-radius: 10px`），显示未读数

**页面间跳转**：
- 使用 Next.js 客户端路由，避免整页刷新
- 加载过程中显示骨架屏（Skeleton），保持连续性

### 5.4 数据可视化交互

| 组件 | 交互方式 |
|------|----------|
| 进度环 | 加载时有填充动画（0% → 实际值），使用 SVG `stroke-dashoffset` |
| 进度条 | 宽度从 0 过渡到目标值，800ms ease |
| 热力图 | 单元格 hover 显示详细数据 tooltip |
| 掌握度条 | 渐变色填充（绿→橙→红），语义化表达掌握程度 |

## 6. 页面设计详解

### 6.1 首页学习总览（Dashboard）

#### 设计目标
让用户在 3 秒内了解：我是谁、我的学习状态、我下一步该做什么。

#### 信息层级拆解

**L1 — 导航层**：
- 侧栏"首页总览"高亮，用户明确当前位置

**L2 — 状态概览**：
- Hero 区域：日期 + 个性化问候 + 进度摘要 + 学习连续天数
- 特色卡片（Featured Card）：当前学习路径 + 圆环进度 + 阶段步骤
- 统计卡片行（Stats Row）：四个关键指标一目了然

**L3 — 操作内容**：
- 今日任务列表（可勾选完成）
- 最近学习资源（可点击进入）
- 右侧信息栏：学习阶段、画像标签、知识掌握度、热力图

#### 关键设计决策

1. **Featured Card 使用深色背景**
   - 原因：在大量白色卡片中制造视觉锚点，引导注意力优先聚焦
   - 风险评估：深色可能显得沉重——通过渐变（#1d1d1f → #2c2c2e）+ 径向光晕（蓝色 radial-gradient）+ 半透明白色文字来减轻重量感
   - 类比：类似 Apple 官网的产品卡片——深色背景让产品信息更醒目

2. **Hero 标题 48px 大号字体**
   - 原因：大标题在打开页面瞬间建立情感基调，"继续你的学习之旅"比一排数据更有温度
   - 关键字使用渐变高亮（蓝→青），在保持整体克制的同时增加视觉焦点

3. **学习连续天数使用 Pill 标签**
   - 原因：Streak 是核心激励机制，用绿色药丸标签让它像一个"成就徽章"而非普通数据
   - 位置：紧跟副标题下方，不占用统计数据行的空间

4. **统计卡片四列等宽**
   - 原因：四个指标（时长、完成率、正确率、天数）同等重要，等宽传达平等感
   - 每张卡片顶部有对应颜色的半透明背景（蓝、绿、橙、紫），建立色彩编码系统

### 6.2 学习路径

#### 设计目标
展示学习者的知识地图——已经走过哪些路、当前在哪、前方还有什么。

#### 信息层级

- **纵向步骤指示器**：当前阶段高亮，已完成打勾，未完成灰色——用户一眼看到"进度"
- **知识树**：用父子节点关系展示知识体系，已掌握/正在学/待学三种状态用颜色区分
- **章节详情面板**：点击树节点后展开，显示具体知识点列表和掌握度

#### 设计决策

1. **选择树形结构而非扁平列表**：因为知识本身有层级关系（Python → 函数 → 闭包），树形结构直观映射认知模型
2. **三种状态色彩**：绿色=已掌握、蓝色=学习中、灰色=待学——与掌握度条的颜色系统一致

### 6.3 资源中心

#### 设计目标
让用户快速找到需要的学习资源，核心动作是"搜索"和"筛选"。

#### 信息层级

- **搜索栏**：页面最顶部，大尺寸，带快捷键提示（⌘K）
- **类型筛选**：标签式按钮（全部/讲义/视频/练习/PPT/代码）
- **资源列表**：表格视图，显示名称、类型标签、上传时间、操作按钮

#### 设计决策

1. **表格视图替代卡片视图**：资源数量可能很多（50+），表格的信息密度更高，更适合批量浏览
2. **搜索栏带 ⌘K 提示**：培养键盘快捷操作习惯，提升高级用户效率
3. **类型标签使用颜色编码**：讲义=蓝色、视频=橙色、练习=绿色、PPT=紫色——与首页资源列表一致

### 6.4 练习与错题

#### 设计目标
核心体验是"做题"——界面应该让出空间给题目本身。

#### 信息层级

- **Tab 切换**：练习/错题/收藏 三个标签
- **题目区域**：题干 + 选项，选项使用圆形单选按钮，选中时高亮
- **右侧辅助**：正确率环形图 + 题号导航网格（快速跳转）

#### 设计决策

1. **题目区域占主宽度 70%**：做题时需要阅读和思考，题目区域应该足够宽敞
2. **题号网格用颜色标记状态**：正确=绿色、错误=红色、未做=灰色——做完后能快速回顾
3. **错题模式自动加载上次错误答案**：方便对比学习

### 6.5 资源推送

#### 设计目标
展示 AI 个性化推荐，让用户理解"为什么推荐这个"。

#### 信息层级

- **AI 推荐横幅**：顶部提示"基于你的学习画像，AI 为你精选以下资源"
- **资源卡片网格**：3 列布局，每张卡片含缩略图、标题、类型标签、推荐理由

#### 设计决策

1. **推荐理由显式展示**：每个推荐附带一句话说明（如"因为你函数章节掌握度较低"），建立 AI 推荐的透明度和可信度
2. **卡片缩略图**：视觉化资源类型（视频显示播放按钮图标、讲义显示文档图标），比纯文字更直观

### 6.6 资源生成与资源详情

#### 设计目标
让学生通过自然语言描述需求，AI 自动生成六类学习资源，并支持在线预览和一键保存到资源中心。

#### 信息层级

- **资源类型选择器**：页面顶部，六宫格布局（课程讲解文档 / PPT / 思维导图 / 练习题 / 拓展阅读材料 / 代码实操案例），选中高亮蓝色
- **需求描述区**：自然语言输入框 + 知识点标签快捷选择（如 Python / 基础巩固 / 函数），点击"生成"触发 AI 生成
- **已生成资源列表**：左侧，显示资源名称、类型图标、状态标签（完成 / 生成中），当前选中项高亮
- **资源预览面板**：右侧固定 320px，支持文档/PPT 渲染预览、视频播放、思维导图查看，底部有"下载"和"保存到资源中心"操作按钮

#### 设计决策

1. **六宫格类型选择器**：用卡片图标+文字而非下拉菜单，降低选择成本，视觉上更直观
2. **需求描述支持标签辅助**：输入框下方自动携带当前学习上下文的标签（Python / 章节名），减少重复输入
3. **左右分栏列表+预览**：左侧资源列表供快速切换，右侧实时预览——类似邮件客户端的阅读体验
4. **生成状态实时反馈**：正在生成的资源显示橙色"生成中"标签 + 进度指示，完成变为绿色"完成"标签
5. **一键保存到资源中心**：预览面板底部的主操作按钮，生成的资源可在资源中心统一管理和检索

### 6.7 智能辅导

#### 设计目标
模拟一对一辅导对话，核心是对话流的自然感。

#### 信息层级

- **对话区域**：用户消息右对齐（浅蓝背景）、AI 消息左对齐（白色背景）
- **代码块**：AI 回复中的代码使用等宽字体 + 浅灰背景，可复制
- **输入栏**：底部固定，带发送按钮和附件按钮

#### 设计决策

1. **气泡式对话**：比传统聊天框更有对话感，圆角气泡增加亲和力
2. **AI 消息中的代码块高亮**：学习编程时代码是核心内容，需要特殊视觉处理
3. **输入栏固定在底部**：随时可输入，不需要滚动到页面末尾

### 6.8 学习报告

#### 设计目标
将学习数据转化为可行动的洞察——不仅告诉用户"发生了什么"，还告诉用户"该怎么做"。

#### 信息层级

- **时间筛选**：本周/本月/自定义
- **数据可视化**：柱状图（每日学习时长）+ 环形图（时间分配）+ 薄弱点列表
- **AI 分析总结**：底部区域，用卡片展示 AI 对学习数据的解读和建议

#### 设计决策

1. **薄弱点使用红色/橙色标记**：视觉紧迫感促使用户关注薄弱环节
2. **AI 分析总结放在底部**：先看数据，再看分析——让用户先形成自己的判断，再参考 AI 建议

### 6.9 视频播放

#### 设计目标
沉浸式学习体验，视频是主角。

#### 信息层级

- **16:9 播放器**：居中，占据主要内容区
- **AI 自动笔记**：视频下方，实时生成学习笔记
- **章节列表**：右侧侧栏，显示视频章节，可跳转

#### 设计决策

1. **播放器使用深色控件**：半透明白色控件叠在视频上，不干扰画面
2. **AI 笔记实时生成**：这是核心 AI 功能——观看视频时自动生成结构化笔记，降低手动记笔记的负担

### 6.10 个人信息

#### 设计目标
简洁的个人中心，不分散学习注意力。

#### 布局

- 左侧：头像 + 基本信息表单
- 右侧：学习统计摘要 + 安全设置

### 6.11 学习画像建档

#### 设计目标
首次使用时快速收集用户画像，为 AI 推荐提供数据基础。

#### 设计决策

1. **5 步流程指示器**：让用户知道总共有多少步、当前在哪步
2. **卡片选择替代下拉菜单**：可视化选择（如学习目标卡片"期末提分"/"竞赛准备"/"兴趣探索"）降低决策难度
3. **多选标签**：知识薄弱项用多选标签而非复选框，更自然

### 6.12 登录/注册

#### 设计目标
简洁、专业、值得信赖。

#### 布局

- **左右分割**：左侧深色品牌区（品牌 Logo + Slogan + 装饰图形），右侧表单区
- **Tab 切换**：登录/注册共用一个表单区，通过 Tab 切换

#### 设计决策

1. **左侧深色区传达专业感**：类似 Notion、Linear 等企业级产品的登录页设计
2. **社交登录选项**：降低注册门槛，但不作为主要路径

## 7. 响应式与适配策略

### 7.1 断点系统

| 断点 | 宽度 | 适配策略 |
|------|------|----------|
| Desktop | ≥ 1200px | 完整侧栏 + 双列内容 |
| Tablet | 768px - 1199px | 侧栏折叠为图标栏（64px）+ 单列内容 |
| Mobile | < 768px | 侧栏隐藏（汉堡菜单触发）+ 单列堆叠 |

### 7.2 适配原则

- **内容优先级不变**：无论屏幕大小，信息层级（L1-L4）保持一致
- **统计卡片**：桌面 4 列 → 平板 2 列 → 移动 1 列
- **内容网格**：桌面双列 → 平板/移动单列（右栏堆叠到左栏下方）

## 8. 无障碍与可用性

### 8.1 色彩无障碍

- 所有文字在背景上的对比度 ≥ 4.5:1（WCAG AA）
- 主文字（#1d1d1f on #ffffff）对比度 ≈ 16:1（WCAG AAA）
- 不依赖颜色传达信息——掌握度同时使用颜色和百分比数字

### 8.2 键盘可访问性

- 所有交互元素可通过 Tab 键聚焦
- 导航项支持 Enter/Space 激活
- 搜索栏支持 ⌘K 快捷键

### 8.3 图标系统

- 使用 Lucide SVG 图标，支持 `aria-label` 和 `title` 属性
- 图标始终伴随文字标签（或有 tooltip 补充），不单独依赖图标传达含义
- 拒绝使用 emoji 作为功能性图标——emoji 在不同平台渲染不一致，且缺乏无障碍支持

## 10. 设计决策记录

### DR-001: 为什么不用 Tailwind 的 utility-first 方案而保留 CSS 变量

**决策**：原型阶段使用 CSS 自定义属性（`--blue`、`--shadow-sm`）定义设计 Token，实际开发中迁移到 Tailwind CSS 的 `theme` 配置。

**原因**：CSS 变量在原型阶段更直观，可直接在浏览器 DevTools 中调试。Tailwind 主题配置是这些 Token 的运行时等价物。

### DR-002: 为什么统计卡片是 4 列而非其他

**决策**：固定 4 列（学习时长、任务完成率、正确率、连续天数）。

**原因**：
- 4 是偶数，视觉平衡
- 这四个指标覆盖了学习评估的四个维度：投入量（时长）、执行力（完成率）、理解度（正确率）、坚持度（天数）
- 不超过 4 个避免 Miller's Law（7±2）的上限压力

### DR-003: 为什么右侧栏宽度固定 360px

**决策**：右列不随窗口缩放。

**原因**：右列承载的是信息阅读型内容（进度条、标签、热力图），固定宽度确保这些元素的视觉比例在任何屏幕下一致。左列承载操作型内容（任务、资源），弹性宽度更灵活。

### DR-004: 为什么 Featured Card 用深色而非白色

**决策**：当前学习路径卡片使用深色渐变背景。

**原因**：
- 在白色为主的页面中制造视觉锚点，防止"白茫茫一片"
- 深色让进度环的发光效果（`drop-shadow`）更显著
- 类比 Apple 产品页的 hero 区域——深色传达"核心"、"重要"的视觉语义

### DR-005: 为什么选择 Lucide 而非 emoji 或 Iconify

**决策**：默认使用 Lucide React 图标，缺少对应图标时才用 Iconify 补充。

**原因**：
- emoji 在不同操作系统上渲染不一致（Windows vs macOS vs Android）
- emoji 无法自定义颜色、大小、描边粗细
- Lucide 是 SVG 图标，支持 `stroke-width`、`color` 等 CSS 属性，风格统一
- Iconify 作为 fallback 但不混用，避免视觉风格不一致

### DR-006: 为什么不使用深色模式

**决策**：v1.0 不实现深色模式，但 CSS 变量结构预留扩展。

**原因**：
- 教育场景以白天使用为主
- 深色模式需要重新设计所有颜色语义，增加 40% 设计和开发工作量
- Apple Pure 设计语言的核心是"明亮、干净、留白"，深色模式会削弱这一特质
