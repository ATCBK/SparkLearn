## 2. 项目成功总览

- 画像构建：通过对话与行为采集形成动态学习画像。

系统已成功打通以下能力：

1. 讯飞模型与语音能力（ASR/TTS）双通道接入。

2. CrewAI 与 Coze 低代码平台混合多智能体架构。

3. 资源实时预览 + 一键下载 + URL 到 PDF 导出完整链路。

4. 无限闯关、错题本、收藏与画像回流联动机制。

8. Vibecoding 到 MP4 的代码驱动视频生成路线。

## 3. 系统架构与技术实现

### 3.1 技术栈与实现框架

系统采用以下技术栈完成整体实现：

- 前端：Next.js + React + TypeScript。
- 后端：FastAPI（Python）+ StreamingResponse（SSE 流式输出）。
- 数据层：SQLite + JSON + JSONL。
- 检索增强：LangChain TextSplitter + ChromaDB。
- 多智能体：CrewAI（高代码编排）+ Coze（低代码 Agent）。
- 语音能力：讯飞 ASR（输入）+ 讯飞 TTS（输出）。
- 文档导出：浏览器渲染打印链路（URL -> PDF）。

### 3.2 模块化架构

系统采用模块化与中间层解耦设计，关键模块如下：

1. 用户画像模块  
负责画像初始化、动态更新、画像读写、画像版本化。

2. 智能导师模块  
负责会话管理、角色系统、文件上下文、多轮对话、导师协同。

3. 资源生成模块  
负责课程文档、思维导图、练习题、拓展阅读、代码案例、PPT/视频内容链路。

4. 练习与评测模块  
负责无限闯关、判题、错题沉淀、收藏、掌握度更新。

5. 学习路径与图谱模块  
负责知识节点管理、依赖关系计算、路径动态规划、AI 节点建议。

6. RAG 检索模块  
负责知识切分、向量入库、语义检索、上下文拼装。

7. Coze 中间层模块  
负责 bot_id 路由、鉴权统一、消息结构解析、异常标准化。

8. 导出与交付模块  
负责实时预览、链接抓取、PDF 渲染导出与下载分发。

### 3.3 统一中间层能力（Coze + 本地系统）

为保障系统扩展性，项目落地了统一中间层：

- 支持不同资源类型映射不同 bot_id。
- 支持环境变量热替换 bot_id，免改业务代码。
- 统一处理返回结构（文本、链接、事件流、异常）。
- 标准化“预览链接提取 -> 内容渲染 -> 导出下载”链路。

该中间层让资源 Agent 能力具备可插拔特性，后续替换不同 Agent 只需改配置不改主流程。

## 4. 核心竞争力

### 4.1 讯飞全生态深度融合能力

项目实现了“模型 + 语音 + 多模态场景”的统一接入：

- 模型能力用于教学问答、资源生成、意图提取、内容重写。
- 语音能力用于语音提问与语音播报，提升交互可达性。
- 最终形成文本、语音、图文资源、视频输出的多模态学习体验。

这是系统在教育场景中的关键差异化能力。

### 4.2 双主流 Agent 模式混合工程化

项目创新采用两种主流 Agent 开发模式协同：

- CrewAI：负责复杂工作流编排、任务分工、决策流程控制。
- Coze：负责资源型 Agent 快速上线、快速替换、快速调优。

通过统一中间层打通后，系统实现了“高代码深度 + 低代码速度”的工程平衡，形成稳定且高效的能力底座。

### 4.3 数据飞轮驱动个性化

项目并非静态推荐系统，而是持续进化系统：

- 用户每次对话、答题、资源使用都写入行为事件。
- 行为事件驱动画像更新与掌握度变化。
- 更新后的画像反向驱动路径规划、资源推荐与辅导策略。

这一闭环使系统个性化能力随使用持续增强。

### 4.4 可交付链路能力

系统把“生成”升级为“可用交付”：

- 生成后即时预览。
- 文档、图片、链接类资源统一下载入口。
- URL 资源可直接转 PDF，并保持真实网页渲染效果。

这解决了教学场景“能不能直接用”的核心问题。

### 4.5 低成本多模态内容生产能力

项目采用 Vibecoding 到 MP4 路线：

- 内容先结构化（JSON/DSL）再模板渲染（HTML 幻灯片）。
- 讲解文案与语音时间轴对齐。
- 前端可播放、可翻页、可录制，最终产出教学视频。

该能力在保证质量的同时显著降低制作成本。

## 5. 核心创新点（技术版）

### 5.1 无限闯关 + 实时诊断 + 画像回流

系统把练习能力做成持续进化机制：

- 用户完成当前题目后可继续生成下一批题目。
- 判题结果即时更新到错题与收藏体系。
- 错题与低掌握节点反向回流画像与路径系统。
- 形成“练-测-改-推”连续闭环。

### 5.2 多智能体导师协同链路

导师系统支持：

- 会话管理（新建、重命名、删除、历史追溯）。
- 角色系统（角色创建、编辑、启停、删除）。
- 文件上下文（上传资料后对话联动）。
- 多角色协同辅导（角色分工回答与风格控制）。

在教学体验上实现了从单助手到多导师协同的升级。

### 5.3 知识图谱化学习路径

路径系统采用图谱化表达：

- 节点包含学习主题、掌握度、状态与学习内容细分。
- 边表示先修依赖关系。
- 全图初始可见，支持缩放、拖拽、节点查看。
- 点击节点触发 AI 分析卡，输出个性化建议与通俗解释。

系统统一了“结构图谱 + 路径规划 + 节点建议”三层能力。

### 5.4 资源生成智能体解耦

资源生成已按能力类型解耦：

- 课程文档、练习题、拓展阅读、代码案例可复用同类 Agent。
- 思维导图独立 bot_id，返回图片链接直接预览与下载。
- PPT/视频走专门链路，保持多模态质量与可扩展性。

解耦后系统支持按资源类型独立替换 Agent，不影响整体流程。

### 5.5 URL 到 PDF 导出技术标准

系统沉淀了统一导出标准：

- 前端点击下载触发后端导出接口。
- 后端基于浏览器内核打开 URL，等待渲染完成。
- 注入打印样式，执行页面打印生成 PDF。
- 返回文件流并自动下载。

该标准可直接迁移到其他项目复用。

## 6. 数据与画像流转机制

### 6.1 数据分层设计

系统采用三层数据设计：

1. 实时状态层（SQLite）  
存储用户画像、会话、角色、消息、练习状态等结构化数据。

2. 事件沉淀层（JSONL）  
存储学习行为、资源使用、生成事件、评估事件等时间序列数据。

3. 资源索引层（JSON）  
存储资源索引、预览链接、导出状态、展示视图数据。

### 6.2 画像更新策略

画像更新触发源包括：

- 对话信息（目标、偏好、困难点）。
- 练习结果（正确率、错误类型、难度适配）。
- 资源行为（查看时长、下载记录、复习频次）。
- 导师交互（追问路径、理解障碍点）。

更新后画像立即影响：

- 路径排序。
- 资源推荐优先级。
- 辅导策略与讲解深度。
- 后续出题策略。

### 6.3 长期数据价值

该机制使系统拥有持续累积的数据资产：

- 可复盘。
- 可分析。
- 可解释。
- 可迭代优化。

这构成项目长期壁垒之一。

## 7. RAG 与知识底座实现

### 7.1 轻量级 RAG 结构

系统采用轻量 RAG 结构：

- TextSplitter 完成课程内容分片。
- ChromaDB 存储向量索引。
- 检索时按语义召回相关片段。
- 召回片段与用户画像共同作为上下文输入。

### 7.2 RAG 在业务中的使用点

RAG 能力贯穿多个模块：

- 智能导师问答。
- 课程文档生成。
- 题目生成与解析生成。
- 路径建议与节点解释。

该设计保证“答得像老师”而不是“答得像闲聊机器人”。
