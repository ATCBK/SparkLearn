## 1. 文档目的

本文档用于将当前已确认的产品想法落实为可执行的技术方案，作为《学而思｜技术产品方案 V2》之后的实施补充稿。本文档聚焦五个核心问题：

第一，智能辅导如何以拓扑图与 Workflow 形式展示多智能体协同。

第二，多模态资源生成如何封装为 Tools 并由智能体编排调用。

第三，Coze 作为中间层返回链接后，前端如何统一预览与下载。

第四，个性化学习路径如何升级为知识图谱并形成动态推荐闭环。

第五，视频资源如何通过“HTML-PPT + 讯飞语音 + 字幕对齐”链路稳定产出。

本文档作为本阶段落地依据，后续开发、联调、演示均以本文档为准。

***

## 2. 范围与目标

### 2.1 本阶段目标

本阶段目标不是新增零散功能，而是把已有能力统一进入“多智能体可视化编排”框架，使评审可清晰看到系统不是单模型调用，而是有角色分工、步骤协作、过程可观测、结果可回放的完整体系。

### 2.2 本阶段包含内容

包含以下五项：

1. 智能辅导拓扑化（角色绑定 Workflow、执行可视化）。
2. 资源生成 Tool 化（按类型路由到不同工具执行）。
3. Coze 链接资源统一渲染（PDF/HTML/图片预览）。
4. 学习路径知识图谱化（节点、依赖、掌握度驱动）。
5. 视频合成链路（HTML-PPT + TTS + 字幕对齐 + 导出视频）。

### 2.3 本阶段不包含内容

不包含多租户用户体系改造，不包含生产级云部署，不包含复杂权限系统。

***

## 3. 多智能体总体架构

### 3.1 编排核心

系统新增统一编排层 `Orchestrator`，所有智能辅导与资源生成请求必须进入编排层，不允许直接绕过编排层调用模型。

编排层负责：

1. 意图识别与任务拆解。
2. Workflow 路由（辅导流/资源流）。
3. Agent 与 Tools 调度。
4. 执行日志采集与回放。
5. 失败重试与降级策略。

### 3.2 Agent 角色（本阶段）

1. `IntentAgent`：识别用户意图、任务类型、约束条件。
2. `ProfileAgent`：读取并更新学生画像信号。
3. `PlannerAgent`：将目标拆解为可执行步骤。
4. `TutorAgent`：负责辅导问答文本生成。
5. `ResourceAgent`：负责资源任务分发与汇总。
6. `ReviewerAgent`：统一做内容校验、格式标准化与风险提示。

### 3.3 运行证据要求

每次请求必须输出可视化执行证据，包括：

- 节点名称
- 执行状态（pending/running/success/failed）
- 开始与结束时间
- 耗时
- 中间结果摘要
- 最终产物引用

## 4. 智能辅导拓扑与角色绑定

### 4.1 页面形态

导师页支持“角色 + 拓扑模板”双选择：

1. 先选择导师角色（如通用导师、严谨导师、项目导师）。
2. 再自动绑定该角色对应 Workflow 模板。
3. 用户发送消息后，页面实时高亮当前执行节点。

### 4.2 辅导 Workflow（默认模板）

`IntentAgent -> ProfileAgent -> PlannerAgent -> TutorAgent -> ReviewerAgent`

其中 `ProfileAgent` 与 `PlannerAgent` 可并行读取上下文；`ReviewerAgent` 必须串行收口。

### 4.3 前端展示要求

1. 左侧显示拓扑节点（可折叠）。
2. 中部显示对话内容（流式输出）。
3. 右侧显示执行事件流（步骤、耗时、状态变化）。

***

## 5. 多模态资源 Tool 化方案

### 5.1 Tool 清单

资源生成由 `ResourceAgent` 调用以下 Tools：

1. `doc_tool`：课程讲解文档。
2. `ppt_tool`：PPT 或 HTML-PPT。
3. `quiz_tool`：练习题与判题辅助信息。
4. `reading_tool`：拓展阅读。
5. `code_tool`：代码案例。
6. `video_tool`：视频脚本/分镜/成片链接（阶段化实现）。

### 5.2 Tool 统一返回协议

每个 Tool 统一返回：

- `tool_name`
- `status`
- `summary`
- `artifact_type`
- `artifact_url` 或 `artifact_content`
- `cost_ms`
- `trace_id`

### 5.3 资源 Workflow

`IntentAgent -> PlannerAgent -> ResourceAgent(并行调用 Tools) -> ReviewerAgent`

并行上限默认 3，防止资源争抢导致整体超时。

## 6. Coze 中间层与统一预览

### 6.1 Coze 返回产物约定

Coze Agent 返回结构统一为：

- `title`
- `url`
- `mime_type`（可选）
- `expires_at`（可选）
- `source`（coze agent id）

### 6.2 后端预览代理

后端新增预览代理层，前端不直接消费外链：

1. 后端判断内容类型（PDF/HTML/图片/其他）。
2. 返回安全预览地址与下载地址。
3. 统一处理 CORS、签名过期、重定向等问题。

### 6.3 前端渲染规则

1. 图片：`img` 预览。
2. PDF：`iframe` 或 `pdf.js` 预览。
3. HTML：`iframe sandbox` 预览。
4. 其他：显示下载按钮与无法预览提示。

***

## 7. 学习路径知识图谱化

### 7.1 图谱模型

路径升级为知识图谱：

1. 节点：知识点。
2. 边：前置依赖关系。
3. 状态：未学/学习中/已掌握。
4. 信号：掌握度、错题频次、学习偏好。

### 7.2 动态推荐机制

系统根据图谱状态自动计算：

1. 当前最优下一跳节点。
2. 回补路径（薄弱点反推前置知识）。
3. 每节点推荐资源（文档、题目、代码、视频）。

### 7.3 页面要求

学习路径页支持图谱可视化，节点颜色反映掌握度，点击节点展示推荐资源与学习动作。

## 8. 视频生成链路（不依赖视频生成 API）

### 8.1 方案定义

视频产出采用“讲解内容驱动”链路：

1. 大模型生成 HTML-PPT（页面结构化）。
2. 生成分段讲稿（按页/按段）。
3. 调用讯飞 TTS 生成分段音频。
4. 基于音频时长生成字幕时间轴。
5. 将 PPT 画面、语音、字幕合成为 MP4。

### 8.2 分段与对齐规则

1. 最小粒度为“页内段落”，避免整页过长。
2. 每段有唯一 `segment_id`。
3. 时间轴以毫秒为单位统一管理。
4. 支持断点续跑（单段失败可重试）。

### 8.3 产物清单

1. `slides.html`
2. `script.json`
3. `audio_segments/*.wav`
4. `subtitle.srt`
5. `timeline.json`
6. `output.mp4`

## 12. 风险与应对

1. 风险：多工具并发导致超时。  
应对：并发上限、超时重试、降级模板。

2. 风险：Coze 外链预览失败。  
应对：后端代理、到期检测、失败降级下载。

3. 风险：视频音画不同步。  
应对：统一 timeline、分段重算、单段重试。

4. 风险：多智能体被质疑“伪协同”。  
应对：展示完整执行拓扑、步骤日志、中间产物。

***

## 13. 验收标准

1. 辅导与资源两条主链路均可展示拓扑执行过程。
2. 资源生成至少 5 种类型可稳定返回并前端可预览。
3. Coze 产物支持 PDF/HTML/图片三类在线预览。
4. 学习路径可视化为知识图谱并能动态更新节点状态。
5. 视频链路可生成可播放 MP4，字幕与音频对齐。
6. 全链路具备可回放执行日志。

***

## 14. 与 V2 的关系

本文档不是替代 V2，而是 V2 的执行增强版。V2 负责总体选型与模块边界，本文档负责把“多智能体可视化编排 + Tool 化生成 + 知识图谱 + 视频链路”落实为可开发、可联调、可演示的实施方案。
