# 答题状态恢复测试场景

## 测试目标
验证用户点击之前做过的题目时，答题状态能否正确恢复，特别是对于多选题。

## 测试场景 1: 单选题状态恢复

### 步骤
1. 初始化系统，加载 4 道题目
2. 第 1 题是单选题，选择选项 "A"
3. 点击"提交判题"按钮
4. 观察题号按钮显示状态（✓ 或 ✗）
5. 点击第 2 题
6. 再点击第 1 题的题号按钮
7. 验证选项 "A" 是否被恢复选中

### 预期结果
- ✅ 选项 "A" 应该显示为蓝色（已选中状态）
- ✅ 提交按钮应该显示为"已提交"
- ✅ 结果卡片应该显示判题结果和解析

---

## 测试场景 2: 多选题状态恢复

### 步骤
1. 初始化系统，加载题目
2. 找到一道多选题（题型为 "multiple"）
3. 选择多个选项，例如 "A"、"C"、"D"
4. 点击"提交判题"按钮
5. 观察题号按钮显示状态
6. 点击其他题目
7. 再点击该多选题的题号按钮
8. 验证所有选中的选项是否都被恢复

### 预期结果
- ✅ 所有之前选中的选项（"A"、"C"、"D"）都应该显示为蓝色
- ✅ 未选中的选项应该显示为白色
- ✅ 提交按钮应该显示为"已提交"
- ✅ 结果卡片应该显示判题结果

### 关键验证点
```
选项状态对比：
- 选项 "A": 蓝色背景 ✓
- 选项 "B": 白色背景 ✓
- 选项 "C": 蓝色背景 ✓
- 选项 "D": 蓝色背景 ✓
```

---

## 测试场景 3: 填空题状态恢复

### 步骤
1. 初始化系统，加载题目
2. 找到一道填空题（题型为 "fill_blank"）
3. 选择一个选项（如果有的话）或输入答案
4. 点击"提交判题"按钮
5. 点击其他题目
6. 再点击该填空题的题号按钮
7. 验证答案是否被恢复

### 预期结果
- ✅ 之前选中的答案应该显示为蓝色
- ✅ 提交按钮应该显示为"已提交"
- ✅ 结果卡片应该显示判题结果

---

## 测试场景 4: 混合题型导航

### 步骤
1. 初始化系统，加载 8 道题目（混合单选、多选、填空）
2. 依次做题：
   - 第 1 题（单选）：选择 "B"，提交
   - 第 2 题（多选）：选择 "A"、"C"，提交
   - 第 3 题（填空）：选择答案，提交
   - 第 4 题（单选）：选择 "D"，提交
3. 使用题号按钮随机导航：
   - 点击第 2 题 → 验证多选答案恢复
   - 点击第 1 题 → 验证单选答案恢复
   - 点击第 3 题 → 验证填空答案恢复
   - 点击第 4 题 → 验证单选答案恢复

### 预期结果
- ✅ 每次导航时，对应题目的答案都应该正确恢复
- ✅ 题号按钮的状态指示符（✓ 或 ✗）应该正确显示
- ✅ 提交按钮状态应该正确反映

---

## 测试场景 5: 上一题/下一题导航

### 步骤
1. 初始化系统，加载 4 道题目
2. 第 1 题：选择答案，提交
3. 点击"下一题"按钮 → 进入第 2 题
4. 验证第 2 题的状态是否清空（selectedAnswer = null）
5. 第 2 题：选择答案，提交
6. 点击"上一题"按钮 → 回到第 1 题
7. 验证第 1 题的答案是否被恢复

### 预期结果
- ✅ 使用上一题/下一题时，新题目的状态应该清空
- ✅ 使用题号按钮返回已答题目时，答案应该被恢复
- ✅ 提交按钮状态应该正确反映

---

## 测试场景 6: 生成新题目后的状态

### 步骤
1. 初始化系统，加载 4 道题目
2. 做完第 1 题，提交
3. 点击"生成练习题"按钮，生成 4 道新题目
4. 验证题目列表是否追加（应该有 8 道题目）
5. 点击第 1 题的题号按钮
6. 验证第 1 题的答案是否被恢复

### 预期结果
- ✅ 新题目应该追加到列表中
- ✅ 之前做过的题目的答案应该仍然被保存
- ✅ 点击之前做过的题目时，答案应该被恢复

---

## 测试场景 7: 题号按钮状态指示符

### 步骤
1. 初始化系统，加载 8 道题目
2. 做完 4 道题目：
   - 第 1 题：正确 ✓
   - 第 2 题：错误 ✗
   - 第 3 题：正确 ✓
   - 第 4 题：错误 ✗
3. 观察题号按钮的显示：
   - 第 1 题按钮：绿色背景 + ✓ 图标
   - 第 2 题按钮：红色背景 + ✗ 图标
   - 第 3 题按钮：绿色背景 + ✓ 图标
   - 第 4 题按钮：红色背景 + ✗ 图标
   - 第 5-8 题按钮：白色背景（未答）

### 预期结果
- ✅ 正确的题目显示绿色 + ✓
- ✅ 错误的题目显示红色 + ✗
- ✅ 未答的题目显示白色
- ✅ 当前题目显示蓝色

## 调试技巧

### 检查 answerRecords 状态
在浏览器控制台中运行：
```javascript
// 查看所有答题记录
console.log('Answer Records:', answerRecords)

// 查看特定题目的记录
console.log('Question 1 Record:', answerRecords.find(r => r.quizId === 'q1'))
```

### 检查 selectedAnswer 状态
```javascript
// 查看当前选中的答案
console.log('Selected Answer:', selectedAnswer)
console.log('Selected Answer Type:', typeof selectedAnswer)
console.log('Is Array:', Array.isArray(selectedAnswer))
```

### 检查题目类型
```javascript
// 查看当前题目的类型
console.log('Current Question Type:', currentQuestion?.type)
console.log('Current Question:', currentQuestion)
```

## 已知限制

1. **填空题的 LLM 辅助判题**：如果后端启用了 LLM 辅助判题，可能会影响判题结果
2. **多选题的顺序**：多选题的答案顺序不影响判题结果，但在恢复时会保持原始顺序
3. **性能**：大量题目（> 100）时，状态恢复可能会有轻微延迟
