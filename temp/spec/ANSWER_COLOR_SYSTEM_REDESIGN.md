# 答题颜色系统重新设计

## 问题描述
之前的颜色系统存在以下问题：
1. 选中状态和提交状态的逻辑混乱
2. 无法清晰体现答题的对错
3. 用户选中的错误答案没有明显标记

## 新的颜色系统设计

### 核心原则
- **未提交状态**：显示用户的选择（蓝色）
- **已提交状态**：显示答题结果（绿色=正确，红色=错误）
- **清晰反馈**：用图标（✓ 或 ✗）强化对错信息

### 状态转换流程

```
未提交状态
├─ 未选中：白色背景 + 灰色边框
└─ 已选中：蓝色背景 + 蓝色边框 + 蓝色圆形/方形

        ↓ 用户点击"提交判题"

已提交状态
├─ 正确答案：绿色背景 + 绿色边框 + ✓ 图标
├─ 错误答案（已选中）：红色背景 + 红色边框 + ✗ 图标
└─ 其他答案（未选中）：白色背景 + 灰色边框
```

## 题型详细设计

### 1. 单选题

#### 未提交状态
```
┌─────────────────────────────────┐
│ ○ 选项 A                        │  ← 白色背景，灰色边框
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ ● 选项 B (已选中)               │  ← 蓝色背景，蓝色边框，蓝色圆形
└─────────────────────────────────┘
```

#### 已提交状态（假设B正确，用户选了B）
```
┌─────────────────────────────────┐
│ ○ 选项 A                        │  ← 白色背景，灰色边框
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ ● 选项 B (正确)              ✓  │  ← 绿色背景，绿色边框，✓ 图标
└─────────────────────────────────┘
```

#### 已提交状态（假设B正确，用户选了A）
```
┌─────────────────────────────────┐
│ ○ 选项 A (错误)              ✗  │  ← 红色背景，红色边框，✗ 图标
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ ● 选项 B (正确)              ✓  │  ← 绿色背景，绿色边框，✓ 图标
└─────────────────────────────────┘
```

### 2. 多选题

#### 未提交状态
```
┌──────────────────┐  ┌──────────────────┐
│ ☐ 选项 A        │  │ ☐ 选项 B        │  ← 白色背景，灰色边框
└──────────────────┘  └──────────────────┘
┌──────────────────┐  ┌──────────────────┐
│ ☑ 选项 C (已选) │  │ ☐ 选项 D        │  ← 蓝色背景，蓝色边框，蓝色方形
└──────────────────┘  └──────────────────┘
```

#### 已提交状态（假设A、B正确，用户选了A、C）
```
┌──────────────────┐  ┌──────────────────┐
│ ☑ 选项 A (正确) │  │ ☐ 选项 B (正确) │  ← 绿色背景，绿色边框，✓ 图标
│              ✓  │  │              ✓  │
└──────────────────┘  └──────────────────┘
┌──────────────────┐  ┌──────────────────┐
│ ☑ 选项 C (错误) │  │ ☐ 选项 D        │  ← 红色背景，红色边框，✗ 图标
│              ✗  │  │                 │
└──────────────────┘  └──────────────────┘
```

### 3. 填空题

#### 未提交状态
```
请输入答案：
┌─────────────────────────────────┐
│ 输入你的答案...                 │  ← 文本输入框
└─────────────────────────────────┘

参考答案：
┌─────────────────────────────────┐
│ import                          │  ← 白色背景，灰色边框
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ from (已选中)                   │  ← 蓝色背景，蓝色边框
└─────────────────────────────────┘
```

#### 已提交状态（假设import正确，用户选了from）
```
参考答案：
┌─────────────────────────────────┐
│ import (正确)                ✓  │  ← 绿色背景，绿色边框，✓ 图标
└─────────────────────────────────┘
┌─────────────────────────────────┐
│ from (错误)                  ✗  │  ← 红色背景，红色边框，✗ 图标
└─────────────────────────────────┘
```

## 代码实现逻辑

### 单选题
```typescript
const isSelected = selectedAnswer === option
const isCorrect = option === currentQuestion.correctAnswer

// 样式逻辑
submitted
  ? isCorrect
    ? 'border-green bg-green-light'      // 正确答案
    : isSelected
    ? 'border-red bg-red-light'          // 错误答案（用户选中）
    : 'border-line bg-white'             // 其他答案
  : isSelected
  ? 'border-blue bg-blue-light'          // 用户选中
  : 'border-line bg-white'               // 未选中
```

### 多选题

## 图标显示规则

### 提交前
- 无图标显示

### 提交后
- **正确答案**：显示 ✓ 图标（绿色）
- **错误答案（用户选中）**：显示 ✗ 图标（红色）
- **其他答案**：无图标

## 用户体验改进

1. **清晰的选择反馈**：蓝色清晰表示用户的选择
2. **明确的对错提示**：绿色/红色 + 图标强化对错信息
3. **一致的设计语言**：所有题型遵循相同的颜色逻辑
4. **易于理解**：颜色和图标的组合直观易懂
